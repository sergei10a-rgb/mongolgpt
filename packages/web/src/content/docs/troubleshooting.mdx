Асуудлыг засахаас өмнө алдааны яг текст, MongolGPT-ийн хувилбар, үйлдлийн систем, ажиллуулсан командыг тэмдэглэж ав. API түлхүүр, token, нууц үг, хувийн кодыг issue эсвэл логт бүү нийтэл.

## Шуурхай шалгалт

Дараах командууд ихэнх асуудлын шалтгааныг хурдан ялгана.

```bash
mongolgpt --version
mongolgpt --help
mongolgpt debug config
mongolgpt models
```

Илүү дэлгэрэнгүй логийг терминалд харах:

```bash
mongolgpt --print-logs --log-level DEBUG
```

1. Команд олдож байгаа эсэхийг шалга.
2. Тохиргооны алдаа байгаа эсэхийг `mongolgpt debug config`-оор шалга.
3. Provider-ийн эрх болон model жагсаалтыг шалга.
4. Асуудлыг хамгийн бага тохиргоотой шинэ хоосон хавтаст давтаж үз.
5. Хамгийн сүүлийн тогтвортой хувилбарыг [GitHub Releases](https://github.com/sergei10a-rgb/mongolgpt/releases) хуудастай харьцуул.

## Лог ба өгөгдлийн байршил

Лог файлууд:

- **macOS, Linux, WSL:** `~/.local/share/mongolgpt/log/`
- **Windows:** `%USERPROFILE%\.local\share\mongolgpt\log\`

Програмын өгөгдөл:

- **macOS, Linux, WSL:** `~/.local/share/mongolgpt/`
- **Windows:** `%USERPROFILE%\.local\share\mongolgpt\`

Энд ихэвчлэн дараах зүйлс байна.

- `auth.json`: provider API түлхүүр болон OAuth token
- `log/`: ажилласан үеийн лог
- `project/`: сесс, мессеж, төслийн төлөв

:::caution
`auth.json` болон логт нууц мэдээлэл байж болно. Хуваалцахын өмнө API түлхүүр, token, URL query, төслийн зам, хэрэглэгчийн нэрийг арилга.
:::

## `mongolgpt` команд олдохгүй

### Windows

Архиваас задалсан хавтаст:

```powershell
.\mongolgpt.exe --version
```

Энэ команд ажиллавал binary зөв боловч хавтас `PATH`-д ороогүй байна. Binary-г байнгын хавтаст байрлуулаад тэр хавтсыг хэрэглэгчийн `PATH`-д нэмнэ. Шинэ PowerShell цонх нээсний дараа дахин шалга.

```powershell
Get-Command mongolgpt -ErrorAction SilentlyContinue
```

### macOS, Linux, WSL

```bash
chmod +x mongolgpt
./mongolgpt --version
command -v mongolgpt
```

`./mongolgpt` ажиллаад `mongolgpt` ажиллахгүй бол binary байгаа хавтсаа `PATH`-д нэмнэ.

## CLI эхлэхгүй эсвэл шууд хаагдана

1. `mongolgpt --print-logs --log-level DEBUG` ажиллуулж эхний алдааг ол.
2. `mongolgpt debug config` ажиллуулж буруу JSON, үл таних config key, байхгүй файл байгаа эсэхийг шалга.
3. Төслийн `.mongolgpt/` болон глобал `~/.config/mongolgpt/` тохиргоог тус тус шалга.
4. Plugin ашигладаг бол түр идэвхгүй болгож дахин ажиллуул.
5. Binary-г зөвхөн [GitHub Releases](https://github.com/sergei10a-rgb/mongolgpt/releases)-ээс дахин татаж шалга.

## Provider-т холбогдохгүй

1. TUI-д `/connect` ажиллуулж provider-ээ дахин холбоно.
2. API түлхүүрийн хугацаа дууссан, эрх цуцлагдсан, billing идэвхгүй болсон эсэхийг provider-ийн өөрийн console дээр шалга.
3. Байгууллагын proxy, VPN, firewall тухайн provider-ийн API-г хааж байгаа эсэхийг шалга.
4. Системийн цаг зөв эсэхийг шалга. OAuth болон гарын үсэгтэй хүсэлт буруу цагтай үед бүтэлгүйтэж болно.
5. `~/.local/share/mongolgpt/auth.json` файлыг шууд засахын оронд `/connect` эсвэл `mongolgpt auth login` ашигла.

`401` эсвэл `403` нь ихэвчлэн эрх, API түлхүүр, subscription-ийн асуудал. `429` нь rate limit эсвэл quota. `5xx` нь provider талын түр алдаа байх боломжтой.

## Загвар харагдахгүй

```bash
mongolgpt models
mongolgpt models <provider-id>
```

Дараахыг шалгана.

1. Provider амжилттай холбогдсон эсэх.
2. Model ID нь яг `provider-id/model-id` хэлбэртэй эсэх.
3. `blacklist` эсвэл `whitelist` тохиргоо model-ийг нуусан эсэх.
4. Тухайн account model-д хандах эрхтэй эсэх.

Жишээ ID:

- `openai/gpt-4.1`
- `openrouter/google/gemini-2.5-flash`

Provider болон model нэрийг орчуулж өөрчилж болохгүй. Provider-ийн жагсаалт, model ID-г `/models` эсвэл `mongolgpt models`-оос яг хэвээр нь хуулна.

## `ProviderInitError`

Энэ алдаа нь ихэвчлэн provider-ийн буруу `baseURL`, дутуу орчны хувьсагч, буруу model ID, эсвэл plugin-ийн эхлүүлэлтийн алдааг заана.

1. [Provider тохируулах зааврыг](../providers) дахин шалга.
2. `mongolgpt debug config` гаралтаас аль config файл утга өгснийг хар.
3. Захиалгат `baseURL` болон proxy-г түр авч дахин турш.
4. Plugin-уудыг түр идэвхгүй болго.
5. `/connect` командаар provider-ээ дахин холбо.

Бүх програмын өгөгдлийг шууд устгах хэрэггүй. Эхлээд `auth.json` болон config файлаа тусдаа backup болгон хуулж, аль нэг өөрчлөлтийг нэг удаад турш.

## `AI_APICallError`

Алдааны status болон provider-ийн response хэсгийг шалгана.

- **Timeout:** сүлжээ, proxy, VPN, provider status-аа шалгаад дахин оролд.
- **Context length:** prompt, хавсаргасан файл, өмнөх сессийн хэмжээг багасга.
- **Unsupported parameter:** model-д тохироогүй reasoning, temperature, output сонголтыг авч үз.
- **Package/cache:** MongolGPT-ийг бүрэн хаагаад cache хавтсыг backup нэрээр шилжүүлж дахин эхлүүл.

macOS, Linux, WSL дээр:

```bash
mv ~/.cache/mongolgpt ~/.cache/mongolgpt.backup-$(date +%Y%m%d-%H%M%S)
```

Windows PowerShell дээр:

```powershell
Rename-Item "$env:USERPROFILE\.cache\mongolgpt" "mongolgpt.backup-$(Get-Date -Format yyyyMMdd-HHmmss)"
```

Хавтас байхгүй бол энэ алхмыг алгас. Backup-аа асуудал шийдэгдсэний дараа, хэрэгтэй өгөгдөлгүйг баталгаажуулж байж устгана.

## Ширээний програм

### Нээгдэхгүй эсвэл хоосон цонх харагдана

1. Програмыг бүрэн хааж дахин нээ.
2. Task Manager эсвэл Activity Monitor-д үлдсэн MongolGPT process байгаа эсэхийг шалга.
3. Windows дээр Microsoft Edge WebView2 Runtime суусан, шинэ эсэхийг шалга.
4. Release-ийн файлыг [албан ёсны Releases хуудаснаас](https://github.com/sergei10a-rgb/mongolgpt/releases) авсан эсэхээ шалга.
5. Алдааны дэлгэц байгаа бол **Restart** дарж, нууц мэдээллээ арилгасан алдааны текстийг хадгал.

### Сервертэй холбогдохгүй

Ширээний програм локал sidecar сервер эхлүүлэх эсвэл өмнө тохируулсан серверийн URL-д холбогдож болно.

1. Сервер сонгогчоос **Default server** утгыг цэвэрлэж локал горимд дахин оролд.
2. `mongolgpt.json(c)` доторх `server.port`, `server.hostname`-ийг түр авч шалга.
3. `MONGOLGPT_PORT` орчны хувьсагч сул порт зааж байгаа эсэхийг шалга.
4. Firewall локал холболтыг хаагаагүй эсэхийг шалга.

### Plugin-оос болж гацна

Глобал config дахь `plugin` жагсаалтыг түр хоосолно.

```jsonc title="mongolgpt.jsonc"
{
  "plugin": []
}
```

Локал plugin хавтаснуудыг түр өөр нэртэй болгоод програмыг дахин эхлүүл.

- **macOS, Linux, WSL:** `~/.config/mongolgpt/plugins/`
- **Windows:** `%USERPROFILE%\.config\mongolgpt\plugins\`
- **Төсөл:** `<project>/.mongolgpt/plugins/`

Ажиллаж эхэлбэл plugin-уудыг нэг нэгээр нь буцааж идэвхжүүлж асуудалтайг ол.

### Ширээний төлөвийг reset хийх

Энэ бол хамгийн сүүлийн алхам. Эхлээд програмыг бүрэн хааж, файлуудыг устгахын оронд backup хавтас руу зөөнө.

- `mongolgpt.settings.dat`: default server URL зэрэг тохиргоо
- `mongolgpt.global.dat`: глобал UI төлөв
- `mongolgpt.workspace.*.dat`: workspace бүрийн UI төлөв

Эдгээр файл operating system-ийн application data хавтаст байрлана. Reset хийснээр recent server, tab, workspace төлөв алдагдаж болно; төслийн эх кодыг устгах ёсгүй.

## Вэб сервер

### `EADDRINUSE`

Сонгосон порт өөр process-д ашиглагдаж байна. Өөр порт сонго.

```bash
mongolgpt web --port 4097
```

### `401 Unauthorized`

`MONGOLGPT_SERVER_USERNAME`, `MONGOLGPT_SERVER_PASSWORD` утгаа серверийг эхлүүлсэн terminal session-д тохируулсан эсэхийг шалга. Нууц үгийг URL, shell history, issue-д бүү оруул.

### Сүлжээнээс холбогдохгүй

1. Сервер `--hostname 0.0.0.0` сонголттой ажиллаж байгаа эсэхийг шалга.
2. Нууц үг тохируулсан эсэхийг шалга.
3. Firewall дээр зөвхөн хэрэгтэй дотоод сүлжээний хүрээнд портоо нээ.
4. Интернетээс хандах бол HTTPS reverse proxy эсвэл VPN ашигла.

[Вэб интерфэйсийн аюулгүй тохиргоог](../web) үзнэ үү.

## Linux дээр copy/paste ажиллахгүй

X11 дээр `xclip` эсвэл `xsel`, Wayland дээр `wl-clipboard` суулгана.

```bash
# X11
sudo apt install xclip

# Wayland
sudo apt install wl-clipboard
```

Headless орчинд clipboard хэрэгтэй бол виртуал X server ашиглаж болно.

```bash
sudo apt install xvfb
Xvfb :99 -screen 0 1024x768x24 > /dev/null 2>&1 &
export DISPLAY=:99.0
```

## Windows болон WSL

Windows дээр файл унших удаан, shell command зөрөх, permission асуудал гарвал [WSL зааврыг](../windows-wsl) ашигла. Төслөө `/mnt/c/...` биш, WSL-ийн `~/projects/...` хавтаст байрлуулах нь ихэнх тохиолдолд хурдан.

## Устгах

CLI өөрийн суулгах аргыг таньж, устгах файлуудыг баталгаажуулахын өмнө харуулна.

```bash
mongolgpt uninstall
```

Тохиргоо эсвэл програмын өгөгдлөө хадгалах сонголтыг [CLI лавлагаанаас](../cli#uninstall) шалга. Ширээний програмыг үйлдлийн системийн Apps эсвэл Applications хэсгээс устгана.

## Түгээмэл асуулт

<details>
<summary>MongolGPT account заавал хэрэгтэй юу?</summary>

Free Auto, багц, quota болон төхөөрөмж хооронд хуваалцах workspace ашиглахад MongolGPT бүртгэл хэрэгтэй. Өөрийн API түлхүүр болон локал загварын нууц мэдээлэл тусдаа хадгалагдана. Hosted service идэвхтэй release дээр CLI-д `mongolgpt console login` тушаалаар нэвтэрнэ. [Бүртгэл ба багцын зааврыг](../account) үзнэ үү.

</details>

<details>
<summary>MongolGPT-ийг хаанаас татах вэ?</summary>

Зөвхөн [GitHub Releases](https://github.com/sergei10a-rgb/mongolgpt/releases) хуудас болон төслийн [GitHub санг](https://github.com/sergei10a-rgb/mongolgpt) ашигла. Танихгүй домэйнээс binary, installer, install script бүү ажиллуул.

</details>

<details>
<summary>`https://your-domain.example` ажиллахгүй байна. Яагаад?</summary>

Энэ нь `.env.example` доторх deployment placeholder бөгөөд бодит үйлчилгээ биш. Хөгжүүлэгч өөрийн Cloudflare орчны бодит URL-аар солино. Энгийн хэрэглэгч URL-ийг гараар тохируулахгүй; Desktop болон CLI нь release channel-ийн service URL-ийг сонгоно.

</details>

<details>
<summary>Free Auto, төлбөртэй багц эсвэл public share production-д нээгдсэн үү?</summary>

Нэгдсэн auth, Free Auto, quota болон QPay/Bonum төлбөрийн код dev орчинд туршигдаж байна. Release notes-д production нээлтийг албан ёсоор зарлаж, бүртгэлийн самбарт хүчинтэй нөхцөл нийтлэгдэх хүртэл бодит төлбөрийн мэдээлэл бүү оруул. Энэ хооронд CLI/TUI, local web, desktop, BYOK provider, локал model, MCP болон LSP боломжуудыг ашиглаж болно. [Одоогийн төлөвийг](../account) үзнэ үү.

</details>

<details>
<summary>Вэб интерфэйсийг интернет рүү шууд нээж болох уу?</summary>

Зөвлөдөггүй. `MONGOLGPT_SERVER_PASSWORD`, HTTPS reverse proxy, firewall, VPN эсвэл SSH tunnel ашигла. Нууц үггүй `0.0.0.0` сервер нь сесс болон төслийн мэдээллийг эрсдэлд оруулна.

</details>

<details>
<summary>Сесс болон API түлхүүр хаана хадгалагддаг вэ?</summary>

CLI-ийн локал өгөгдөл `~/.local/share/mongolgpt/` эсвэл Windows дээр `%USERPROFILE%\.local\share\mongolgpt\` дотор хадгалагдана. Prompt болон хавсаргасан контент сонгосон AI provider руу хүсэлтээр илгээгдэж болно; provider-ийн нууцлал, retention бодлогыг тусад нь шалга.

</details>

<details>
<summary>Тохиргооны аль файл хэрэглэгдэж байгааг яаж мэдэх вэ?</summary>

`mongolgpt debug config` ажиллуул. Энэ нь нийлүүлэгдсэн тохиргоог харуулж, project, global, environment утгуудын зөрчлийг оношлоход тусална.

</details>

## Тусламж авах

Шинэ issue нээхийн өмнө [одоо байгаа GitHub Issues](https://github.com/sergei10a-rgb/mongolgpt/issues)-оос алдааны текстээр хай. Шинэ issue-д дараах мэдээллийг оруулна.

- `mongolgpt --version` гаралт
- Үйлдлийн систем, архитектур
- CLI, desktop, web-ийн аль нь болох
- Давтах хамгийн богино алхам
- Хүлээсэн ба бодит үр дүн
- Нууц мэдээллээ арилгасан логийн холбогдох хэсэг

API түлхүүр, token, нууц үг, хувийн repository-ийн кодыг хэзээ ч нийтэлж болохгүй.
