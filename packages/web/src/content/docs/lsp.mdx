MongolGPT нь Language Server Protocol (LSP) серверийн diagnostic-ийг агентын контекстэд ашиглаж чадна. LSP нь локал ажиллагаа бөгөөд MongolGPT account, hosted backend эсвэл production domain шаарддаггүй.

## Түргэн эхлэх

LSP анхдагчаар идэвхгүй. Бүх суурилуулсан LSP integration-ийг идэвхжүүлэхийн тулд:

```json title="mongolgpt.json"
{
  "lsp": true
}
```

MongolGPT-г дахин эхлүүлээд дэмжигдсэн өргөтгөлтэй файл нээнэ. Тухайн language server-ийн шаардлагатай runtime эсвэл command `PATH`-д байх ёстой.

```bash
mongolgpt debug config
```

Энэ командаар нийлүүлэгдсэн тохиргоонд `lsp` идэвхтэй эсэхийг шалгана.

Автомат download-ийг хориглох бол MongolGPT-г эхлүүлэхээс өмнө:

```bash
export MONGOLGPT_DISABLE_LSP_DOWNLOAD=true
```

Windows PowerShell дээр:

```powershell
$env:MONGOLGPT_DISABLE_LSP_DOWNLOAD = "true"
```

Энэ үед шаардлагатай language server-ийг өөрөө суулгаж `PATH`-д оруулна.

## Ажиллах зарчим

LSP идэвхтэй үед MongolGPT:

1. Нээсэн файлын өргөтгөлийг идэвхтэй server бүрийн `extensions` утгатай тулгана.
2. Тохирох server ажиллаагүй бол эхлүүлнэ.
3. Diagnostic-ийг агентын дараагийн алхамд ашиглах боломжтой болгоно.

Language server их санах ой ашиглах, төслийн төлөвтэй синхрончлогдохгүй байх, эсвэл agent loop-ийг удаашруулах тохиолдол бий. Төсөлд найдвартай `lint` эсвэл `typecheck` command байвал түүнийг `AGENTS.md`-д баримтжуулах нь заримдаа илүү тогтвортой.

## Нарийвчилсан тохиргоо

Object хэлбэр ашигласнаар тодорхой server-ийг идэвхгүй болгох, built-in тохиргоог override хийх, custom server нэмэх боломжтой.

| Key | Төрөл | Утга |
| --- | --- | --- |
| `disabled` | boolean | `true` бол тухайн server-ийг идэвхгүй болгоно |
| `command` | string array | Server эхлүүлэх executable болон argument |
| `extensions` | string array | Тухайн server ажиллах файлын өргөтгөл |
| `env` | object | Server process-д дамжуулах орчны хувьсагч |
| `initialization` | object | LSP `initialize` request-д өгөх server-specific сонголт |

Нэг server-ийг идэвхгүй болгох:

```json title="mongolgpt.json"
{
  "lsp": {
    "typescript": {
      "disabled": true
    }
  }
}
```

Custom server нэмэх:

```json title="mongolgpt.json"
{
  "lsp": {
    "custom-lsp": {
      "command": ["custom-lsp-server", "--stdio"],
      "extensions": [".custom"],
      "env": {
        "LOG_LEVEL": "info"
      },
      "initialization": {
        "preferences": {
          "importModuleSpecifierPreference": "relative"
        }
      }
    }
  }
}
```

Бүх LSP server-ийг идэвхгүй болгох:

```json title="mongolgpt.json"
{
  "lsp": false
}
```

## Алдаа шийдэх

Diagnostic харагдахгүй бол:

1. `mongolgpt debug config` гаралтад `lsp` идэвхтэй эсэхийг шалгана.
2. Файлын өргөтгөл table дахь утгатай яг таарч байгаа эсэхийг шалгана.
3. Шаардлагатай runtime, dependency эсвэл language server command-ийг terminal-аас ажиллуулж үзнэ.
4. `MONGOLGPT_DISABLE_LSP_DOWNLOAD` санамсаргүй тохирсон эсэхийг шалгана.
5. Config өөрчилсний дараа MongolGPT-г дахин эхлүүлнэ.

Server тогтмол унах эсвэл их нөөц ашиглавал тухайн ID-г `disabled: true` болгож, төслийн `lint`/`typecheck` command-ийг ашиглана. Ерөнхий оношлогоог [алдаа шийдэх заавраас](../troubleshooting) харна уу.

- Windows: `%USERPROFILE%/intelephense/license.txt`
