`AGENTS.md` файл үүсгэж MongolGPT-д төсөлд зориулсан заавар өгч болно. Энэ нь Cursor rules-тэй төстэй бөгөөд LLM-ийн контекстэд орж, агент тухайн repository-д хэрхэн ажиллахыг тодорхойлно.

---

## Эхлүүлэх

Шинэ `AGENTS.md` файл үүсгэхийн тулд mongolgpt дээр `/init` командыг ажиллуулж болно.

:::tip
Та төслийнхээ `AGENTS.md` файлыг Git-д оруулах ёстой.
:::

`/init` нь таны репо дахь чухал файлуудыг сканнердаж, кодын сан тэдэнд хариулж чадахгүй үед хэд хэдэн зорилтот асуулт асууж, дараа нь төслийн тусгай зааварчилгаагаар `AGENTS.md`-г үүсгэж эсвэл шинэчилдэг.

Энэ нь ирээдүйн төлөөлөгчийн хуралдаанд хамгийн их хэрэгтэй байх зүйлсэд анхаарлаа хандуулдаг:

- бүтээх, линт, тест командууд
- Тушаалын дараалал болон чухал үед чиглэсэн баталгаажуулалтын алхамууд
- Зөвхөн файлын нэрээр тодорхойгүй архитектур ба репо бүтэц
- төслийн тусгайлсан конвенц, тохиргооны хачирхалтай байдал, үйл ажиллагааны онцлог
- Cursor эсвэл Copilot-ийн дүрэм зэрэг одоо байгаа зааврын эх сурвалжийн лавлагаа

Хэрэв `AGENTS.md` аль хэдийн байвал `/init` шууд дарж солихгүй, одоо байгаа агуулгыг сайжруулна.

---

## Жишээ

Та мөн энэ файлыг гараар үүсгэж болно. `AGENTS.md` файлд оруулж болох зарим зүйлсийн жишээ энд байна.

```markdown title="AGENTS.md"
# SST v3 monorepo төслийн дүрэм

Энэ нь TypeScript ашигладаг SST v3 monorepo. Package удирдахад Bun workspace ашиглана.

## Төслийн бүтэц

- `packages/` - Workspace-ийн бүх package (`functions`, `core`, `web` гэх мэт)
- `infra/` - Үйлчилгээгээр салгасан infrastructure тодорхойлолт (`storage.ts`, `api.ts`, `web.ts`)
- `sst.config.ts` - Dynamic import ашигладаг SST-ийн үндсэн тохиргоо

## Кодын стандарт

- TypeScript-ийн strict mode-ийг идэвхжүүл
- Хуваалцсан кодыг зөв `exports` тохиргоотой `packages/core/` дотор байрлуул
- Function-уудыг `packages/functions/` дотор байрлуул
- Infrastructure кодыг `infra/` дотор үүргээр нь тусдаа файл болго

## Monorepo дүрэм

- Хуваалцсан module-ийг workspace нэрээр import хий: `@my-app/core/example`
```

Бид энд төслийн тусгай зааварчилгааг нэмж байгаа бөгөөд үүнийг танай багт хуваалцах болно.

---

## Төрөл

mongolgpt нь `AGENTS.md` файлыг олон газраас уншихыг дэмждэг. Мөн энэ нь өөр өөр зорилготой.

### Төсөл

Төслийн тусгай дүрмийн хувьд төслийн үндэс хэсэгт `AGENTS.md` тэмдэглэнэ үү. Эдгээр нь зөвхөн таныг энэ лавлах эсвэл түүний дэд лавлах хэсэгт ажиллаж байх үед л хэрэгжинэ.

### Глобал

Та мөн `~/.config/mongolgpt/AGENTS.md` файлд глобал дүрэмтэй байж болно. Үүнийг бүх mongolgpt сессүүдэд хэрэгжүүлнэ.

Энэ нь Git-д зориулагдаагүй эсвэл танай багтай хуваалцаагүй тул LLM-ийн дагаж мөрдөх хувийн дүрмийг зааж өгөхийн тулд үүнийг ашиглахыг зөвлөж байна.

### Claude Code нийцтэй байдал

Claude Code-оос шилжиж буй хэрэглэгчдэд зориулж MongolGPT нь Claude Code-ийн файлын convention-ийг fallback байдлаар дэмждэг:

- **Төслийн дүрэм**: `CLAUDE.md` таны төслийн лавлах (хэрэв `AGENTS.md` байхгүй бол хэрэглэнэ)
- **Глобал дүрэм**: `~/.claude/CLAUDE.md` (хэрэв `~/.config/mongolgpt/AGENTS.md` байхгүй бол хэрэглэнэ)
- **Ур чадвар**: `~/.claude/skills/` — дэлгэрэнгүйг [Агентын ур чадвар](../skills/) харна уу

Claude Code нийцтэй байдлыг идэвхгүй болгохын тулд дараах орчны хувьсагчийн аль нэгийг тохируулна:

```bash
export MONGOLGPT_DISABLE_CLAUDE_CODE=1        # .claude дэмжлэгийг бүхэлд нь унтраана
export MONGOLGPT_DISABLE_CLAUDE_CODE_PROMPT=1 # зөвхөн ~/.claude/CLAUDE.md-г унтраана
export MONGOLGPT_DISABLE_CLAUDE_CODE_SKILLS=1 # зөвхөн .claude/skills-г унтраана
```

---

## Тэргүүлэх

mongolgpt эхлэх үед дүрмийн файлуудыг дараах дарааллаар хайдаг:

1. **Орон нутгийн файлуудыг** одоогийн лавлахаас дээш гүйлгэж (`AGENTS.md`, `CLAUDE.md`)
2. **Глобал файл** `~/.config/mongolgpt/AGENTS.md`
3. **Claude Code файл** `~/.claude/CLAUDE.md` (идэвхгүй болгоогүй бол)

Эхний тохирох файл нь ангилал бүрт ялна. Жишээлбэл, хэрэв танд `AGENTS.md` болон `CLAUDE.md` хоёулаа байгаа бол зөвхөн `AGENTS.md` ашигладаг. Үүний нэгэн адил `~/.config/mongolgpt/AGENTS.md` нь `~/.claude/CLAUDE.md`-ээс давуу эрхтэй.

---

## Тусгай заавар

Төслийн `mongolgpt.json` эсвэл глобал `~/.config/mongolgpt/mongolgpt.json` файлд нэмэлт зааврын файлуудыг зааж болно. Ингэснээр одоо байгаа дүрмийг `AGENTS.md` руу хуулалгүй дахин ашиглана.

Жишээ:

```json title="mongolgpt.json"
{
  "instructions": ["CONTRIBUTING.md", "docs/guidelines.md", ".cursor/rules/*.md"]
}
```

Та мөн вэбээс зааварчилгааг ачаалахын тулд алсаас URLs ашиглаж болно.

```json title="mongolgpt.json"
{
  "instructions": ["https://raw.githubusercontent.com/my-org/shared-rules/main/style.md"]
}
```

Алсын зааварчилгааг 5 секундын завсарлагатайгаар татаж авдаг.

Бүх зааврын файлууд нь таны `AGENTS.md` файлуудтай нийлдэг.

---

## Гадаад файлуудын лавлагаа

mongolgpt нь `AGENTS.md` дахь файлын лавлагааг автоматаар задалдаггүй ч та ижил төстэй ажиллагааг хоёр аргаар хийж болно:

### mongolgpt.json ашиглан

Санал болгож буй арга бол `mongolgpt.json` дахь `instructions` талбарыг ашиглах явдал юм:

```json title="mongolgpt.json"
{
  "instructions": ["docs/development-standards.md", "test/testing-guidelines.md", "packages/*/AGENTS.md"]
}
```

### Гарын авлагын заавар AGENTS.md

Та `AGENTS.md`-д тодорхой зааварчилгаа өгснөөр гадаад файлуудыг уншихыг mongolgpt-д зааж болно. Энд практик жишээ байна:

```markdown title="AGENTS.md"
# TypeScript төслийн дүрэм

## Гадаад файл ачаалах

ЧУХАЛ: Файлын лавлагаа (жишээ нь `@rules/general.md`) тааралдвал зөвхөн тухайн ажилд шаардлагатай үед `Read` хэрэгслээр ачаал. Лавлагаа бүрийг урьдчилж ачаалж болохгүй.

Заавар:

- Бүх лавлагааг урьдчилж БҮҮ ачаал; бодит хэрэгцээнд тулгуурлан lazy loading ашигла
- Ачаалсан агуулгыг өгөгдмөл дүрмээс давуу, заавал мөрдөх заавар гэж үз
- Шаардлагатай үед лавлагааг рекурсив дага

## Хөгжүүлэлтийн заавар

TypeScript кодын хэв маяг, сайн туршлага: @docs/typescript-guidelines.md
React component архитектур, hook pattern: @docs/react-patterns.md
REST API дизайн, алдаа боловсруулах: @docs/api-standards.md
Тестийн стратеги, coverage шаардлага: @test/testing-guidelines.md

## Ерөнхий заавар

Бүх workflow-д хамаарах тул дараах файлыг шууд унш: @rules/general-guidelines.md.
```

Энэ арга нь танд дараах боломжийг олгоно:

- Модульчлагдсан, дахин ашиглах боломжтой дүрмийн файлуудыг үүсгэх
- Симлинк эсвэл git дэд модулиар дамжуулан төслүүдийн дунд дүрмийг хуваалцах
- Нарийвчилсан удирдамжийг дурдахдаа AGENTS.md-г товчхон бичнэ үү
- mongolgpt файлуудыг зөвхөн тухайн ажилд шаардлагатай үед ачаалдаг эсэхийг шалгаарай

:::tip
Monorepo эсвэл нийтлэг стандарттай төсөлд `mongolgpt.json`-ийн glob pattern (`packages/*/AGENTS.md` гэх мэт) ашиглавал зааврыг гараар давтахаас хялбар арчилна.
:::
