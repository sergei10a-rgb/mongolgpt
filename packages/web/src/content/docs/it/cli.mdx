La CLI di MongolGPT, per impostazione predefinita, avvia la [TUI](/docs/tui) quando viene eseguita senza argomenti.

```bash
mongolgpt
```

Ma accetta anche i comandi documentati in questa pagina. Questo ti permette di interagire con MongolGPT in modo programmatico.

```bash
mongolgpt run "Explain how closures work in JavaScript"
```

---

### tui

Avvia l'interfaccia testuale di MongolGPT (interfaccia utente da terminale).

```bash
mongolgpt [project]
```

#### Flag

| Flag                                     | Breve | Descrizione                                                              |
| ---------------------------------------- | ----- | ------------------------------------------------------------------------ |
| <nobr><code>{"--continue"}</code></nobr> | `-c`  | Continua l'ultima sessione                                               |
| <nobr><code>{"--session"}</code></nobr>  | `-s`  | ID sessione da continuare                                                |
| <nobr><code>{"--fork"}</code></nobr>     |       | Duplica la sessione quando continui (usa con `--continue` o `--session`) |
| <nobr><code>{"--prompt"}</code></nobr>   |       | Prompt da usare                                                          |
| <nobr><code>{"--model"}</code></nobr>    | `-m`  | Modello nel formato provider/model                                       |
| <nobr><code>{"--agent"}</code></nobr>    |       | Agente da usare                                                          |
| <nobr><code>{"--port"}</code></nobr>     |       | Porta su cui mettersi in ascolto                                         |
| <nobr><code>{"--hostname"}</code></nobr> |       | Hostname su cui mettersi in ascolto                                      |

## Comandi

La CLI di MongolGPT include anche i seguenti comandi.

---

### agent

Gestisci gli agenti per MongolGPT.

```bash
mongolgpt agent [command]
```

---

### attach

Collega un terminale a un backend server di MongolGPT già in esecuzione avviato tramite i comandi `serve` o `web`.

```bash
mongolgpt attach [url]
```

Questo consente di usare la TUI con un backend MongolGPT remoto. Per esempio:

```bash
# Start the backend server for web/mobile access
mongolgpt web --port 4096 --hostname 0.0.0.0

# In un altro terminale, collega la TUI al backend in esecuzione
mongolgpt attach http://10.20.30.40:4096
```

#### Flag

| Flag                                     | Breve | Descrizione                                                                                       |
| ---------------------------------------- | ----- | ------------------------------------------------------------------------------------------------- |
| <nobr><code>{"--dir"}</code></nobr>      |       | Working directory in cui avviare la TUI                                                           |
| <nobr><code>{"--continue"}</code></nobr> | `-c`  | Continua l'ultima sessione                                                                        |
| <nobr><code>{"--session"}</code></nobr>  | `-s`  | ID sessione da continuare                                                                         |
| <nobr><code>{"--fork"}</code></nobr>     |       | Crea un fork della sessione durante la continuazione (usa con `--continue` o `--session`)         |
| <nobr><code>{"--password"}</code></nobr> | `-p`  | Password per l'autenticazione di base (predefinita: `MONGOLGPT_SERVER_PASSWORD`)                  |
| <nobr><code>{"--username"}</code></nobr> | `-u`  | Nome utente per l'autenticazione di base (predefinito: `MONGOLGPT_SERVER_USERNAME` o `mongolgpt`) |

#### create

Crea un nuovo agente con configurazione personalizzata.

```bash
mongolgpt agent create
```

Questo comando ti guida nella creazione di un nuovo agente con un system prompt personalizzato e configurazione degli strumenti.

---

#### list

Elenca tutti gli agenti disponibili.

```bash
mongolgpt agent list
```

### auth

Comando per gestire le credenziali e il login dei provider.

```bash
mongolgpt auth [command]
```

---

#### login

MongolGPT si basa sull'elenco provider di [Models.dev](https://models.dev), quindi puoi usare `mongolgpt auth login` per configurare le API key per qualunque provider tu voglia usare. Le credenziali vengono salvate in `~/.local/share/mongolgpt/auth.json`.

```bash
mongolgpt auth login
```

Quando MongolGPT si avvia, carica i provider dal file delle credenziali e, se presenti, anche eventuali key definite nell'ambiente o in un file `.env` nel progetto.

---

#### list

Elenca tutti i provider autenticati come salvati nel file delle credenziali.

```bash
mongolgpt auth list
```

Oppure la versione corta.

```bash
mongolgpt auth ls
```

---

#### logout

Esegue il logout da un provider rimuovendolo dal file delle credenziali.

```bash
mongolgpt auth logout
```

### github

Gestisci l'agente GitHub per l'automazione dei repository.

```bash
mongolgpt github [command]
```

---

#### install

Installa l'agente GitHub nel tuo repository.

```bash
mongolgpt github install
```

Questo configura il workflow GitHub Actions necessario e ti guida nel processo di configurazione. [Scopri di più](/docs/github).

---

#### run

Esegui l'agente GitHub. Tipicamente usato in GitHub Actions.

```bash
mongolgpt github run
```

##### Flag

| ------------------------------------- | -------------------------------------------- |
| <nobr><code>{"--event"}</code></nobr> | Evento GitHub mock per cui eseguire l'agente |
| <nobr><code>{"--token"}</code></nobr> | GitHub personal access token                 |

### mcp

Gestisci i server Model Context Protocol.

```bash
mongolgpt mcp [command]
```

---

#### add

Aggiungi un server MCP alla tua configurazione.

```bash
mongolgpt mcp add
```

Questo comando ti guida nell'aggiunta di un server MCP locale o remoto.

---

#### list

Elenca tutti i server MCP configurati e il loro stato di connessione.

```bash
mongolgpt mcp list
```

Oppure la versione corta.

```bash
mongolgpt mcp ls
```

---

#### auth

Autentica con un server MCP con OAuth abilitato.

```bash
mongolgpt mcp auth [name]
```

Se non fornisci un nome server, ti verrà chiesto di selezionare tra i server OAuth-capable disponibili.

Puoi anche elencare i server OAuth-capable e il loro stato di autenticazione.

```bash
mongolgpt mcp auth list
```

Oppure la versione corta.

```bash
mongolgpt mcp auth ls
```

---

#### logout

Rimuovi le credenziali OAuth per un server MCP.

```bash
mongolgpt mcp logout [name]
```

---

#### debug

Esegui debug di problemi di connessione OAuth per un server MCP.

```bash
mongolgpt mcp debug <name>
```

### models

Elenca tutti i modelli disponibili dai provider configurati.

```bash
mongolgpt models [provider]
```

Questo comando mostra tutti i modelli disponibili tra i provider configurati nel formato `provider/model`.

È utile per capire l'esatto nome del modello da usare nella [config](/docs/config/).

Puoi anche passare opzionalmente un ID provider per filtrare i modelli a quel provider.

```bash
mongolgpt models anthropic
```

#### Flag

| Flag                                    | Descrizione                                        |
| --------------------------------------- | -------------------------------------------------- |
| <nobr><code>{"--refresh"}</code></nobr> | Aggiorna la cache modelli da models.dev            |
| <nobr><code>{"--verbose"}</code></nobr> | Output più verboso (include metadati come i costi) |

Usa `--refresh` per aggiornare l'elenco modelli in cache. È utile quando nuovi modelli vengono aggiunti a un provider e vuoi vederli in MongolGPT.

```bash
mongolgpt models --refresh
```

### run

Esegui mongolgpt in modalità non interattiva passando un prompt direttamente.

```bash
mongolgpt run [message..]
```

È utile per scripting, automazione o quando vuoi una risposta rapida senza avviare la TUI completa. Per esempio:

```bash "mongolgpt run"
mongolgpt run Explain the use of context in Go
```

Puoi anche collegarti a una istanza `mongolgpt serve` già in esecuzione per evitare i cold boot dei server MCP ad ogni esecuzione:

```bash
# Avvia un server headless in un terminale
mongolgpt serve

# In un altro terminale, esegui comandi che si collegano ad esso
mongolgpt run --attach http://localhost:4096 "Explain async/await in JavaScript"
```

#### Flag

| Flag                                     | Breve | Descrizione                                                                                       |
| ---------------------------------------- | ----- | ------------------------------------------------------------------------------------------------- |
| <nobr><code>{"--command"}</code></nobr>  |       | Il comando da eseguire; usa message per gli argomenti                                             |
| <nobr><code>{"--continue"}</code></nobr> | `-c`  | Continua l'ultima sessione                                                                        |
| <nobr><code>{"--session"}</code></nobr>  | `-s`  | ID sessione da continuare                                                                         |
| <nobr><code>{"--fork"}</code></nobr>     |       | Duplica la sessione quando continui (usa con `--continue` o `--session`)                          |
| <nobr><code>{"--share"}</code></nobr>    |       | Condividi la sessione                                                                             |
| <nobr><code>{"--model"}</code></nobr>    | `-m`  | Modello nel formato provider/model                                                                |
| <nobr><code>{"--agent"}</code></nobr>    |       | Agente da usare                                                                                   |
| <nobr><code>{"--file"}</code></nobr>     | `-f`  | File da allegare al messaggio                                                                     |
| <nobr><code>{"--format"}</code></nobr>   |       | Formato: default (formattato) o json (eventi JSON grezzi)                                         |
| <nobr><code>{"--title"}</code></nobr>    |       | Titolo sessione (usa prompt troncato se non viene fornito un valore)                              |
| <nobr><code>{"--attach"}</code></nobr>   |       | Attach a un server mongolgpt in esecuzione (es. http://localhost:4096)                            |
| <nobr><code>{"--password"}</code></nobr> | `-p`  | Password per l'autenticazione di base (predefinita: `MONGOLGPT_SERVER_PASSWORD`)                  |
| <nobr><code>{"--username"}</code></nobr> | `-u`  | Nome utente per l'autenticazione di base (predefinito: `MONGOLGPT_SERVER_USERNAME` o `mongolgpt`) |
| <nobr><code>{"--dir"}</code></nobr>      |       | Directory di esecuzione, o percorso sul server remoto durante il collegamento                     |
| <nobr><code>{"--variant"}</code></nobr>  |       | Variante del modello (sforzo di ragionamento specifico del provider)                              |
| <nobr><code>{"--thinking"}</code></nobr> |       | Mostra blocchi di pensiero                                                                        |
| <nobr><code>{"--port"}</code></nobr>     |       | Porta per il server locale (di default una porta casuale)                                         |

### serve

Avvia un server MongolGPT headless per accesso via API. Vedi le [server docs](/docs/server) per l'interfaccia HTTP completa.

```bash
mongolgpt serve
```

Avvia un server HTTP che espone accesso API alle funzionalità di mongolgpt senza la TUI. Imposta `MONGOLGPT_SERVER_PASSWORD` per abilitare HTTP basic auth (username di default `mongolgpt`).

#### Flag

| Flag                                     | Descrizione                                    |
| ---------------------------------------- | ---------------------------------------------- |
| <nobr><code>{"--port"}</code></nobr>     | Porta su cui mettersi in ascolto               |
| <nobr><code>{"--hostname"}</code></nobr> | Hostname su cui mettersi in ascolto            |
| <nobr><code>{"--mdns"}</code></nobr>     | Abilita discovery mDNS                         |
| <nobr><code>{"--cors"}</code></nobr>     | Origin browser addizionali per consentire CORS |

### stats

Mostra statistiche di utilizzo token e costo per le sessioni MongolGPT.

```bash
mongolgpt stats
```

#### Flag

| Flag                                    | Descrizione                                                                           |
| --------------------------------------- | ------------------------------------------------------------------------------------- |
| <nobr><code>{"--days"}</code></nobr>    | Mostra statistiche per gli ultimi N giorni (all time)                                 |
| <nobr><code>{"--tools"}</code></nobr>   | Numero di strumenti da mostrare (all)                                                 |
| <nobr><code>{"--models"}</code></nobr>  | Mostra breakdown di utilizzo modelli (nascosto di default). Passa un numero per top N |
| <nobr><code>{"--project"}</code></nobr> | Filtra per progetto (tutti i progetti; stringa vuota: progetto corrente)              |

### export

Esporta i dati di sessione come JSON.

```bash
mongolgpt export [sessionID]
```

Se non fornisci un ID sessione, ti verrà chiesto di selezionare tra le sessioni disponibili.

---

### import

Importa i dati di sessione da un file JSON o da un URL di condivisione MongolGPT.

```bash
mongolgpt import <file>
```

Puoi importare da un file locale o da un URL di condivisione MongolGPT.

```bash
mongolgpt import session.json
mongolgpt import https://mongolgpt.duckdns.org/s/abc123
```

```bash

```

#### Flag

| Flag                                        | Breve | Descrizione                                  |
| ------------------------------------------- | ----- | -------------------------------------------- |
| <nobr><code>{"--keep-config"}</code></nobr> | `-c`  | Mantieni i file di configurazione            |

| <nobr><code>{"--dry-run"}</code></nobr>     |       | Mostra cosa verrebbe rimosso senza rimuovere |
| <nobr><code>{"--force"}</code></nobr>       | `-f`  | Salta le richieste di conferma               |

---

### upgrade

Aggiorna mongolgpt all'ultima versione o a una versione specifica.

```bash
mongolgpt upgrade [target]
```

Per aggiornare all'ultima versione:

```bash
mongolgpt upgrade
```

Per aggiornare a una versione specifica:

```bash
mongolgpt upgrade v0.1.48
```

#### Flag

| Flag                                   | Breve | Descrizione                                               |
| -------------------------------------- | ----- | --------------------------------------------------------- |
| <nobr><code>{"--method"}</code></nobr> | `-m`  | Metodo di installazione usato: curl, npm, pnpm, bun, brew |

## Variabili d'ambiente

MongolGPT può essere configurato tramite variabili d'ambiente.

| Variabile                              | Tipo    | Descrizione                                                 |
| -------------------------------------- | ------- | ----------------------------------------------------------- |
| `MONGOLGPT_AUTO_SHARE`                 | boolean | Condivide automaticamente le sessioni                       |
| `MONGOLGPT_GIT_BASH_PATH`              | string  | Percorso all'eseguibile Git Bash su Windows                 |
| `MONGOLGPT_CONFIG`                     | string  | Percorso al file di configurazione                          |
| `MONGOLGPT_TUI_CONFIG`                 | string  | Percorso al file di configurazione TUI                      |
| `MONGOLGPT_CONFIG_DIR`                 | string  | Percorso alla directory di configurazione                   |
| `MONGOLGPT_CONFIG_CONTENT`             | string  | Contenuto JSON di config inline                             |
| `MONGOLGPT_DISABLE_AUTOUPDATE`         | boolean | Disabilita i controlli automatici di aggiornamento          |
| `MONGOLGPT_DISABLE_PRUNE`              | boolean | Disabilita la potatura dei dati vecchi                      |
| `MONGOLGPT_DISABLE_TERMINAL_TITLE`     | boolean | Disabilita aggiornamenti automatici del titolo terminale    |
| `MONGOLGPT_PERMISSION`                 | string  | Config permessi JSON inline                                 |
| `MONGOLGPT_DISABLE_DEFAULT_PLUGINS`    | boolean | Disabilita i plugin di default                              |
| `MONGOLGPT_DISABLE_LSP_DOWNLOAD`       | boolean | Disabilita download automatico dei server LSP               |
| `MONGOLGPT_ENABLE_EXPERIMENTAL_MODELS` | boolean | Abilita modelli sperimentali                                |
| `MONGOLGPT_DISABLE_AUTOCOMPACT`        | boolean | Disabilita compaction automatica del contesto               |
| `MONGOLGPT_DISABLE_CLAUDE_CODE`        | boolean | Disabilita lettura da `.claude` (prompt + skill)            |
| `MONGOLGPT_DISABLE_CLAUDE_CODE_PROMPT` | boolean | Disabilita lettura di `~/.claude/CLAUDE.md`                 |
| `MONGOLGPT_DISABLE_CLAUDE_CODE_SKILLS` | boolean | Disabilita caricamento di `.claude/skills`                  |
| `MONGOLGPT_DISABLE_MODELS_FETCH`       | boolean | Disabilita fetch dei modelli da fonti remote                |
| `MONGOLGPT_FAKE_VCS`                   | string  | Provider VCS finto per scopi di test                        |
| `MONGOLGPT_CLIENT`                     | string  | Identificatore client (default `cli`)                       |
| `MONGOLGPT_ENABLE_EXA`                 | boolean | Abilita gli strumenti di web search Exa                     |
| `MONGOLGPT_SERVER_PASSWORD`            | string  | Abilita basic auth per `serve`/`web`                        |
| `MONGOLGPT_SERVER_USERNAME`            | string  | Sovrascrive lo username basic auth (default `mongolgpt`)    |
| `MONGOLGPT_MODELS_URL`                 | string  | URL personalizzato per recuperare la configurazione modelli |
