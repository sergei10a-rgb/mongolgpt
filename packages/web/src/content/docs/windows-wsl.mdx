MongolGPT CLI-г Windows дээр шууд ажиллуулж болно. Linux shell хэрэгсэл, permission, symlink, файлын гүйцэтгэлийн асуудал гарвал [Windows Subsystem for Linux](https://learn.microsoft.com/en-us/windows/wsl/install) ашиглах нь тохиромжтой.

## WSL суулгах

Administrator эрхтэй PowerShell нээгээд:

```powershell
wsl --install
```

Windows-оо restart хийж, суусан Linux distribution-оо нээгээд хэрэглэгчийн нэр, нууц үгээ үүсгэнэ. Суулгалтыг шалгах:

```powershell
wsl --status
wsl --list --verbose
```

Microsoft-ийн шинэчилсэн алхмуудыг [WSL суулгах албан ёсны заавраас](https://learn.microsoft.com/en-us/windows/wsl/install) шалгана уу.

## MongolGPT CLI суулгах

### Release binary ашиглах

1. [MongolGPT Releases](https://github.com/sergei10a-rgb/mongolgpt/releases) хуудсаас WSL-ийн архитектурт тохирох Linux архивыг татна. Ихэнх Intel/AMD төхөөрөмжид `mongolgpt-linux-x64.tar.gz` тохирно.
2. Архивыг WSL дотор задална.
3. Binary-г хэрэглэгчийн `PATH`-д байрлуулна.

```bash
mkdir -p ~/.local/bin
tar -xzf mongolgpt-linux-x64.tar.gz
install -m 755 mongolgpt ~/.local/bin/mongolgpt
export PATH="$HOME/.local/bin:$PATH"
mongolgpt --version
```

`~/.local/bin` дараагийн terminal session-д танигдахгүй бол `export PATH=...` мөрийг `~/.bashrc` эсвэл ашиглаж буй shell-ийн profile-д нэмнэ.

### Эх кодоос ажиллуулах

Bun болон Git суусан хөгжүүлэлтийн орчинд:

```bash
git clone https://github.com/sergei10a-rgb/mongolgpt
cd mongolgpt
bun install
bun run dev
```

Энэ нь repository-ийн development хувилбарыг ажиллуулна. Өдөр тутмын хэрэглээнд GitHub Release binary сонгох нь хялбар.

## Төслөө хаана байрлуулах вэ?

Хамгийн сайн гүйцэтгэл авахын тулд repository-г WSL-ийн Linux файлын системд байрлуул.

```bash
mkdir -p ~/projects
cd ~/projects
git clone https://github.com/your-org/your-project.git
cd your-project
mongolgpt
```

Windows drive-ууд WSL дотор дараах замаар харагдана.

- `C:` нь `/mnt/c/`
- `D:` нь `/mnt/d/`

```bash
cd /mnt/c/Users/YourName/Documents/project
mongolgpt
```

`/mnt/c` дээрх том repository удаан байвал `~/projects` рүү clone хийж, VS Code-ийн [WSL өргөтгөлөөр](https://code.visualstudio.com/docs/remote/wsl) нээнэ.

## Windows-ийн ширээний програмыг WSL серверт холбох

Windows desktop app ашиглаж, CLI server-ээ WSL-д ажиллуулах боломжтой.

1. WSL terminal-д урт нууц үг тохируулж серверээ эхлүүлнэ.

   ```bash
   MONGOLGPT_SERVER_PASSWORD="урт-санамсаргүй-нууц-үг" mongolgpt serve --hostname 0.0.0.0 --port 4096
   ```

2. Windows desktop app-ийн server сонгогчид `http://localhost:4096` нэмнэ.
3. Хэрэглэгчийн нэрийг `mongolgpt`, нууц үгийг дээрх утгаар оруулна.

`localhost` ажиллахгүй бол WSL IP-г олно.

```bash
hostname -I
```

Дараа нь `http://<wsl-ip>:4096` хаягийг ашиглана. WSL restart хийхэд IP өөрчлөгдөж болохыг анхаар.

:::caution
`0.0.0.0` дээр нууц үггүй сервер бүү ажиллуул. Firewall дээр портыг интернет рүү нээхгүй, зөвхөн локал эсвэл итгэмжлэгдсэн сүлжээнд ашигла.
:::

## Вэб интерфэйсийг WSL-ээс ажиллуулах

```bash
MONGOLGPT_SERVER_PASSWORD="урт-санамсаргүй-нууц-үг" mongolgpt web --hostname 0.0.0.0 --port 4096
```

Windows хөтчөөс `http://localhost:4096` хаягийг нээнэ. Аюулгүй байдлын дэлгэрэнгүйг [вэб интерфэйсийн заавраас](../web) үзнэ үү.

## Нийтлэг асуудал

- **`mongolgpt: command not found`:** `~/.local/bin` нь `PATH`-д байгаа эсэхийг шалга.
- **Permission denied:** `chmod +x mongolgpt` эсвэл `install -m 755 ...` ажиллуул.
- **Файл унших удаан:** төслөө `/mnt/c`-ээс `~/projects` рүү шилжүүлж турш.
- **Windows browser холбогдохгүй:** server `0.0.0.0` дээр сонсож байгаа эсэх, порт, Windows Firewall-аа шалга.
- **Git-ийн line ending зөрнө:** WSL доторх repository-д Linux Git ашиглаж, багийн `.gitattributes` дүрмийг дага.
- **CLI ба desktop өөр сесс харуулна:** хоёулаа ижил server URL-д холбогдсон эсэхийг шалга.

Нэмэлт оношилгоог [алдаа оношлох ба FAQ](../troubleshooting) хуудаснаас үзнэ үү.
