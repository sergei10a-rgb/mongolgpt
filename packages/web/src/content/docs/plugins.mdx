---
title: Плагинууд
description: MongolGPT-г өргөтгөхийн тулд өөрийн залгаасуудыг бичээрэй.
---

Plugins нь янз бүрийн үйл явдалд холбогдож, зан төлөвийг өөрчлөх замаар MongolGPT-г өргөтгөх боломжийг олгодог. Та шинэ функцуудыг нэмэх, гадны үйлчилгээтэй нэгтгэх эсвэл MongolGPT-ийн үндсэн үйлдлийг өөрчлөхийн тулд залгаасуудыг үүсгэж болно.

Integration төрлүүдийг [экосистемийн хуудаснаас](../ecosystem) харна уу.

---

## Plugin ашиглана уу

Плагинуудыг ачаалах хоёр арга бий.

---

### Дотоод файлуудаас

JavaScript эсвэл TypeScript файлуудыг залгаасын санд байрлуул.

- `.mongolgpt/plugins/` - Төслийн түвшний залгаасууд
- `~/.config/mongolgpt/plugins/` - Глобал залгаасууд

Эдгээр сангуудын файлуудыг эхлүүлэх үед автоматаар ачаалагддаг.

---

### npm-аас

Өөрийн тохиргооны файлд npm багцуудыг зааж өгнө үү.

```json title="mongolgpt.json"
{
  "plugin": ["./plugins/helicone-session.ts", "./plugins/wakatime.ts", "@my-org/private-plugin"]
}
```

Энгийн болон хамрах хүрээтэй npm багцуудыг хоёуланг нь дэмждэг.

Баталгаажсан integration төрлүүдийг [экосистемийн хуудаснаас](../ecosystem) харна уу.

---

### Хэрхэн залгаасуудыг суулгадаг вэ

**npm залгаасуудыг** эхлүүлэх үед Bun ашиглан автоматаар суулгадаг. Багцууд болон тэдгээрийн хамаарлыг `~/.cache/mongolgpt/node_modules/`-д хадгалдаг.

**Локал plugin**-ийг plugin directory-оос шууд ачаална. Гадаад package ашиглахын тулд config directory-д `package.json` үүсгэнэ ([Хамаарал](#хамаарал)), эсвэл package-ийг npm-д нийтэлж [тохиргоонд нэмнэ](../config#плагинууд).

---

### Ачаалах захиалга

Plugins нь бүх эх сурвалжаас ачаалагдсан бөгөөд бүх дэгээ нь дарааллаар ажилладаг. Ачаалах дараалал нь:

1. Глобал тохиргоо (`~/.config/mongolgpt/mongolgpt.json`)
2. Төслийн тохиргоо (`mongolgpt.json`)
3. Глобал залгаасуудын лавлах (`~/.config/mongolgpt/plugins/`)
4. Төслийн залгаасуудын лавлах (`.mongolgpt/plugins/`)

Давхардсан npm багцуудыг ижил нэр, хувилбараар нэг удаа ачаална. Гэсэн хэдий ч ижил төстэй нэртэй локал залгаас болон npm залгаасыг тус тусад нь ачаалдаг.

---

## Plugin үүсгэх

Plugin нь нэг буюу хэд хэдэн залгаасыг экспортлодог **JavaScript/TypeScript модуль** юм.
функцууд. Функц бүр контекст объектыг хүлээн авч, hooks объектыг буцаана.

---

### Хамаарал

Орон нутгийн залгаасууд болон тусгай хэрэгслүүд гадаад npm багцуудыг ашиглаж болно. Өөрт хэрэгтэй хамаарал бүхий тохиргооны лавлахдаа `package.json` нэмнэ үү.

```json title=".mongolgpt/package.json"
{
  "dependencies": {
    "shescape": "^2.1.0"
  }
}
```

MongolGPT эдгээрийг суулгахын тулд эхлүүлэх үед `bun install` ажиллуулдаг. Таны залгаасууд болон хэрэгслүүд тэдгээрийг импортлох боломжтой.

```ts title=".mongolgpt/plugins/my-plugin.ts"
import { escape } from "shescape"

export const MyPlugin = async (ctx) => {
  return {
    "tool.execute.before": async (input, output) => {
      if (input.tool === "bash") {
        output.args.command = escape(output.args.command)
      }
    },
  }
}
```

---

### Үндсэн бүтэц

```js title=".mongolgpt/plugins/example.js"
export const MyPlugin = async ({ project, client, $, directory, worktree }) => {
  console.log("Plugin эхэллээ!")

  return {
    // Hook-ийн хэрэгжилтийг энд бичнэ
  }
}
```

Plugin функц нь дараахь зүйлийг хүлээн авдаг.

- `project`: Төслийн одоогийн мэдээлэл.
- `directory`: Одоогийн ажлын лавлах.
- `worktree`: Гит ажлын модны зам.
- `client`: AI-тэй харилцах mongolgpt SDK үйлчлүүлэгч.
- `$`: команд ажиллуулах Bun-ийн [shell API](https://bun.com/docs/runtime/shell).

---

### TypeScript дэмжлэг

TypeScript залгаасуудын хувьд та залгаасын багцаас төрлүүдийг импортлох боломжтой:

```ts title="my-plugin.ts" {1}
import type { Plugin } from "@mongolgpt/plugin"

export const MyPlugin: Plugin = async ({ project, client, $, directory, worktree }) => {
  return {
    // Type-safe hook-ийн хэрэгжилт
  }
}
```

### Үйл явдал

Залгааснууд нь жишээ хэсэгт доороос харсны дагуу үйл явдлуудыг захиалж болно. Энд байгаа янз бүрийн арга хэмжээний жагсаалтыг энд оруулав.

#### Тушаалын үйл явдал

- `command.executed`

#### Файлын үйл явдал

- `file.edited`
- `file.watcher.updated`

#### Суулгах үйл явдлууд

- `installation.updated`

#### LSP Үйл явдал

- `lsp.client.diagnostics`
- `lsp.updated`

#### Мессежийн үйл явдал

- `message.part.removed`
- `message.part.updated`
- `message.removed`
- `message.updated`

#### Зөвшөөрлийн үйл явдал

- `permission.asked`
- `permission.replied`

#### Серверийн үйл явдал

- `server.connected`

#### хуралдааны үйл явдал

- `session.created`
- `session.compacted`
- `session.deleted`
- `session.diff`
- `session.error`
- `session.idle`
- `session.status`
- `session.updated`

#### Хийх үйл явдал

- `todo.updated`

#### Shell үйл явдлууд

- `shell.env`

#### Хэрэгслийн үйл явдал

- `tool.execute.after`
- `tool.execute.before`

#### TUI Үйл явдал

- `tui.prompt.append`
- `tui.command.execute`
- `tui.toast.show`

## Жишээ

mongolgpt-г өргөтгөхөд ашиглаж болох залгаасуудын зарим жишээ энд байна.

---

### Мэдэгдэл илгээх

Тодорхой үйл явдал тохиолдоход мэдэгдэл илгээх:

```js title=".mongolgpt/plugins/notification.js"
export const NotificationPlugin = async ({ project, client, $, directory, worktree }) => {
  return {
    event: async ({ event }) => {
      // Сесс дуусахад мэдэгдэл илгээнэ
      if (event.type === "session.idle") {
        await $`osascript -e 'display notification "Сесс дууслаа!" with title "MongolGPT"'`
      }
    },
  }
}
```

Бид AppleScript-г macOS дээр ажиллуулахын тулд `osascript` ашиглаж байна. Энд бид үүнийг мэдэгдэл илгээхэд ашиглаж байна.

:::note
Хэрэв та MongolGPT ширээний програмыг ашиглаж байгаа бол хариу бэлэн болсон эсвэл сесс алдаа гарсан үед системийн мэдэгдлийг автоматаар илгээх боломжтой.
:::

---

### .env хамгаалалт

mongolgpt файлыг `.env` уншихаас урьдчилан сэргийлэх:

```javascript title=".mongolgpt/plugins/env-protection.js"
export const EnvProtection = async ({ project, client, $, directory, worktree }) => {
  return {
    "tool.execute.before": async (input, output) => {
      if (input.tool === "read" && output.args.filePath.includes(".env")) {
        throw new Error(".env файл уншихыг хориглосон")
      }
    },
  }
}
```

---

### Орчны хувьсагчдыг оруулах

Бүх бүрхүүлийн гүйцэтгэлд орчны хувьсагчдыг оруулах (AI хэрэгслүүд болон хэрэглэгчийн терминалууд):

```javascript title=".mongolgpt/plugins/inject-env.js"
export const InjectEnvPlugin = async () => {
  return {
    "shell.env": async (input, output) => {
      output.env.MY_API_KEY = "secret"
      output.env.PROJECT_ROOT = input.cwd
    },
  }
}
```

### Тусгай хэрэгсэл

Мөн залгаасууд нь mongolgpt-д захиалгат хэрэгслийг нэмж болно:

```ts title=".mongolgpt/plugins/custom-tools.ts"
import { type Plugin, tool } from "@mongolgpt/plugin"

export const CustomToolsPlugin: Plugin = async (ctx) => {
  return {
    tool: {
      mytool: tool({
        description: "Захиалгат хэрэгслийн жишээ",
        args: {
          foo: tool.schema.string(),
        },
        async execute(args, context) {
          const { directory, worktree } = context
          return `Сайн байна уу, ${args.foo}. Хавтас: ${directory} (worktree: ${worktree})`
        },
      }),
    },
  }
}
```

`tool` туслагч нь mongolgpt дуудаж болох тусгай хэрэгсэл үүсгэдэг. Энэ нь Zod схемийн функцийг авч, багажийн тодорхойлолтыг буцаана:

- `description`: Энэ хэрэгсэл юу хийдэг
- `args`: Хэрэгслийн аргументуудын Zod схем
- `execute`: Хэрэгсэл дуудагдах үед ажилладаг функц

Таны захиалгат хэрэгслүүдийг суурилуулсан хэрэгслүүдийн хажуугаар mongolgpt хаягаар авах боломжтой.

:::note
Хэрэв залгаас хэрэгсэл нь суулгасан хэрэгсэлтэй ижил нэрийг ашигладаг бол залгаасын хэрэгсэл нь давуу эрхтэй болно.
:::

---

### Мод бэлтгэх

Зохицуулсан бүртгэлд `console.log`-ийн оронд `client.app.log()` ашиглана уу:

```ts title=".mongolgpt/plugins/my-plugin.ts"
export const MyPlugin = async ({ client }) => {
  await client.app.log({
    body: {
      service: "my-plugin",
      level: "info",
      message: "Plugin эхэллээ",
      extra: { foo: "bar" },
    },
  })
}
```

Түвшин: `debug`, `info`, `warn`, `error`. Дэлгэрэнгүйг [SDK баримт бичгээс](../sdk) үзнэ үү.

### Нягтруулах дэгээ

Сеансыг нягтруулах үед оруулсан контекстийг тохируулна уу:

```ts title=".mongolgpt/plugins/compaction.ts"
import type { Plugin } from "@mongolgpt/plugin"

export const CompactionPlugin: Plugin = async (ctx) => {
  return {
    "experimental.session.compacting": async (input, output) => {
      // Нягтруулах prompt-д нэмэлт контекст оруулна
      output.context.push(`
## Нэмэлт контекст

Нягтруулалтын дараа хадгалах шаардлагатай төлөвийг оруул:
- Одоогийн ажлын төлөв
- Гаргасан чухал шийдвэрүүд
- Идэвхтэй өөрчилж буй файлууд
`)
    },
  }
}
```

LLM үргэлжлэл хураангуйг үүсгэхээс өмнө `experimental.session.compacting` дэгээ асна. Өгөгдмөл нягтруулах сануулгыг орхигдуулах домэйны тусгай контекст оруулахын тулд үүнийг ашиглана уу.

Та `output.prompt`-г тохируулснаар нягтруулах хүсэлтийг бүхэлд нь сольж болно:

```ts title=".mongolgpt/plugins/custom-compaction.ts"
import type { Plugin } from "@mongolgpt/plugin"

export const CustomCompactionPlugin: Plugin = async (ctx) => {
  return {
    "experimental.session.compacting": async (input, output) => {
      // Нягтруулах prompt-ийг бүхэлд нь солино
      output.prompt = `
Та олон агенттай сессийг үргэлжлүүлэх prompt үүсгэж байна.

Дараахыг хураангуйл:
1. Одоогийн ажил болон төлөв
2. Аль файлыг хэн өөрчилж байгаа
3. Агентуудын хоорондын саад, хамаарал
4. Ажлыг дуусгах дараагийн алхам

Шинэ агент ажлыг үргэлжлүүлж чадах бүтэцтэй prompt болго.
`
    },
  }
}
```

`output.prompt` тохируулагдсан үед энэ нь анхдагч нягтруулах хүсэлтийг бүрэн орлоно. Энэ тохиолдолд `output.context` массивыг үл тоомсорлодог.
