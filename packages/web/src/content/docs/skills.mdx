---
title: "Агентын ур чадвар"
description: "SKILL.md тодорхойлолтоор дахин ашиглах боломжтой зан үйлийг тодорхойлох"
---

Агентын ур чадвар нь MongolGPT-д таны репо эсвэл гэрийн лавлахаас дахин ашиглах зааварчилгааг олох боломжийг олгоно.
Ур чадварыг уугуул `skill` хэрэгслээр дамжуулан хүсэлтээр ачаалдаг—агентууд боломжтой ур чадваруудыг харж, шаардлагатай үед бүрэн контентыг ачаалж болно.

---

## Файлуудыг байрлуулах

Ур чадварын нэр болгонд нэг хавтас үүсгээд дотор нь `SKILL.md` тэмдэг тавина.
MongolGPT эдгээр байршлыг хайдаг:

- Төслийн тохиргоо: `.mongolgpt/skills/<name>/SKILL.md`
- Глобал тохиргоо: `~/.config/mongolgpt/skills/<name>/SKILL.md`
- Төсөл Claude-нийцтэй: `.claude/skills/<name>/SKILL.md`
- Глобал Claude-нийцтэй: `~/.claude/skills/<name>/SKILL.md`
- Төслийн төлөөлөгчтэй нийцтэй: `.agents/skills/<name>/SKILL.md`
- Дэлхийн төлөөлөгчтэй нийцтэй: `~/.agents/skills/<name>/SKILL.md`

---

## Нээлтийг ойлгох

Төслийн орон нутгийн замуудын хувьд MongolGPT таны одоогийн ажлын лавлахаас git ажлын мод руу хүрэх хүртэл алхдаг.
Энэ нь `.mongolgpt/`-д тохирох `skills/*/SKILL.md` болон замд тохирох `.claude/skills/*/SKILL.md` эсвэл `.agents/skills/*/SKILL.md`-г ачаална.

Глобал тодорхойлолтыг мөн `~/.config/mongolgpt/skills/*/SKILL.md`, `~/.claude/skills/*/SKILL.md`, `~/.agents/skills/*/SKILL.md`-аас ачаалдаг.

---

## Урд хэсгийг бичнэ үү

`SKILL.md` бүр нь YAML урд талын зүйлээр эхлэх ёстой.
Зөвхөн эдгээр талбаруудыг хүлээн зөвшөөрдөг:

-
ame` (шаардлагатай)
- `description` (шаардлагатай)
- `license` (заавал биш)
- `compatibility` (заавал биш)
- `metadata` (заавал биш, мөр хоорондын зураглал)

Үл мэдэгдэх урд талын талбаруудыг үл тоомсорлодог.

---

## Нэрийг баталгаажуулна уу


ame` заавал байх ёстой:

- 1-64 тэмдэгттэй байх
- Ганц зураасаар ялгагч жижиг үсгээр бичнэ
- `-` гэж эхэлж эсвэл төгсгөхгүй
- Дараалсан `--` агуулаагүй
- `SKILL.md` агуулсан лавлах нэрийг тааруулна уу

Эквивалент регекс:

```text
^[a-z0-9]+(-[a-z0-9]+)*$
```

---

## Уртны дүрмийг дагаж мөрдөөрэй

`description` 1-1024 тэмдэгт байх ёстой.
Агентийг зөв сонгоход хангалттай тодорхой байлга.

---

## Жишээ ашигла

`.mongolgpt/skills/git-release/SKILL.md`-г дараах байдлаар үүсгэ:

```markdown
---
name: git-release
description: Create consistent releases and changelogs
license: MIT
compatibility: mongolgpt
metadata:
  audience: maintainers
  workflow: github
---

## What I do

- Draft release notes from merged PRs
- Propose a version bump
- Provide a copy-pasteable `gh release create` command

## When to use me

Use this when you are preparing a tagged release.
Ask clarifying questions if the target versioning scheme is unclear.
```

---

## Хэрэгслийн тайлбарыг таних

MongolGPT боломжтой ур чадваруудыг `skill` хэрэгслийн тайлбарт жагсаасан.
Бичлэг бүр ур чадварын нэр, тайлбарыг агуулна.

```xml
<available_skills>
  <skill>
    <name>git-release</name>
    <description>Create consistent releases and changelogs</description>
  </skill>
</available_skills>
```

Агент нь уг хэрэгслийг дуудаж ур чадвараа ачаалдаг:

```
skill({ name: "git-release" })
```

---

## Зөвшөөрлийг тохируулах

`mongolgpt.json` дээрх загварт суурилсан зөвшөөрлийг ашиглан агентууд ямар ур чадварт хандаж болохыг хянах:

```json
{
  "permission": {
    "skill": {
      "*": "allow",
      "pr-review": "allow",
      "internal-*": "deny",
      "experimental-*": "ask"
    }
  }
}
```

| Зөвшөөрөл | Зан төлөв |
| ---------- | ----------------------------------------- |
| `allow` | Ур чадвар нэн даруй ачаалагдана |
| `deny` | Агентаас ур чадварыг нуусан, хандалтаас татгалзсан |
| `ask` | Ачаалахын өмнө хэрэглэгчээс зөвшөөрөл хүссэн |

Загвар нь орлуулагч тэмдгийг дэмждэг: `internal-*` таарч байна `internal-docs`, `internal-tools` гэх мэт.

---

## Агент бүрт хүчингүй болгох

Тодорхой төлөөлөгчдөд дэлхийн өгөгдмөлөөс өөр зөвшөөрөл олгох.

**Захиалгат агентуудын хувьд** (агентийн нүүрэнд):

```yaml
---
permission:
  skill:
    "documents-*": "allow"
---
```

**Суурилагдсан агентуудын хувьд** (`mongolgpt.json` дээр):

```json
{
  "agent": {
    "plan": {
      "permission": {
        "skill": {
          "internal-*": "allow"
        }
      }
    }
  }
}
```

---

## Ур чадварын хэрэгслийг идэвхгүй болго

Тэдгээрийг ашиглах ёсгүй агентуудын ур чадварыг бүрэн идэвхгүй болгох:

**Захиалгат агентуудын хувьд**:

```yaml
---
tools:
  skill: false
---
```

**Суурилагдсан агентуудын хувьд**:

```json
{
  "agent": {
    "plan": {
      "tools": {
        "skill": false
      }
    }
  }
}
```

Идэвхгүй болгосон үед `<available_skills>` хэсгийг бүхэлд нь орхигдуулдаг.

---

## Ачаалах алдааг олж засварлах

Хэрэв ур чадвар гарч ирэхгүй бол:

1. `SKILL.md` гэж том үсгээр бичсэн эсэхийг шалгана уу
2. Урд талын асуудалд
ame` болон `description` орсон эсэхийг шалгана уу.
3. Ур чадварын нэрс бүх байршилд давтагдашгүй байгаа эсэхийг шалгаарай
4. Зөвшөөрөл шалгах—`deny` бүхий ур чадварууд агентуудаас нуугдаж байна
