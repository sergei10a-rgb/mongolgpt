---

Хэрэгслүүд нь LLM-д таны кодын санд үйлдэл хийхийг зөвшөөрдөг. MongolGPT суулгасан хэрэгслүүдийн хамт ирдэг боловч та үүнийг [захиалгат хэрэгслүүд](/docs/custom-tools) эсвэл [MCP серверүүд](/docs/mcp-servers)-ээр сунгаж болно.

Анхдагч байдлаар, бүх хэрэгслүүд **идэвхжсэн** бөгөөд ажиллуулах зөвшөөрөл шаардлагагүй. Та [permissions](/docs/permissions)-ээр дамжуулан багажийн ажиллагааг хянах боломжтой.

---

## Тохируулах

Багажны ажиллагааг хянахын тулд `permission` талбарыг ашиглана уу. Та хэрэгсэл тус бүрийг зөвшөөрөх, татгалзах эсвэл зөвшөөрөл шаардах боломжтой.

```json title="mongolgpt.json"
{
  "$schema": "https://mongolgpt.duckdns.org/config.json",
  "permission": {
    "edit": "deny",
    "bash": "ask",
    "webfetch": "allow"
  }
}
```

Та мөн олон хэрэгслийг нэгэн зэрэг удирдахын тулд орлуулагч тэмдгийг ашиглаж болно. Жишээлбэл, MCP серверээс бүх хэрэгсэлд зөвшөөрөл шаардахын тулд:

```json title="mongolgpt.json"
{
  "$schema": "https://mongolgpt.duckdns.org/config.json",
  "permission": {
    "mymcp_*": "ask"
  }
}
```

[Илүү ихийг мэдэж аваарай](/docs/permissions) зөвшөөрлийг тохируулах талаар.

## Баригдсан

MongolGPT-д байгаа бүх суурилуулсан хэрэгслүүд энд байна.

---

### баш

Төслийн орчинд бүрхүүлийн командуудыг гүйцэтгэнэ.

```json title="mongolgpt.json" {4}
{
  "$schema": "https://mongolgpt.duckdns.org/config.json",
  "permission": {
    "bash": "allow"
  }
}
```

Энэ хэрэгсэл нь LLM-д
pm install`, `git status` эсвэл бусад бүрхүүлийн команд зэрэг терминалын командуудыг ажиллуулах боломжийг олгодог.

---

### засвар

Яг мөр солих ашиглан одоо байгаа файлуудыг өөрчлөх.

```json title="mongolgpt.json" {4}
{
  "$schema": "https://mongolgpt.duckdns.org/config.json",
  "permission": {
    "edit": "allow"
  }
}
```

Энэ хэрэгсэл нь текстийн яг таарч байгаа зүйлийг солих замаар файлд нарийн засвар хийдэг. Энэ нь LLM кодыг өөрчлөх үндсэн арга юм.

---

### бичээрэй

Шинэ файл үүсгэх эсвэл байгаа файлуудыг дарж бичих.

```json title="mongolgpt.json" {4}
{
  "$schema": "https://mongolgpt.duckdns.org/config.json",
  "permission": {
    "edit": "allow"
  }
}
```

Үүнийг ашиглан LLM шинэ файл үүсгэхийг зөвшөөрнө үү. Энэ нь одоо байгаа файлууд байгаа бол тэдгээрийг дарж бичих болно.

:::note
`write` хэрэгсэл нь файлын бүх өөрчлөлтийг (`edit`, `write`, `apply_patch`) хамарсан `edit` зөвшөөрлөөр хянагддаг.
:::

---

### уншсан

Кодын сангаас файлын агуулгыг уншина уу.

```json title="mongolgpt.json" {4}
{
  "$schema": "https://mongolgpt.duckdns.org/config.json",
  "permission": {
    "read": "allow"
  }
}
```

Энэ хэрэгсэл нь файлуудыг уншиж, агуулгыг нь буцаадаг. Энэ нь том файлуудын тодорхой мөрийн хүрээг уншихыг дэмждэг.

---

### grep

Ердийн илэрхийлэл ашиглан файлын агуулгыг хайх.

```json title="mongolgpt.json" {4}
{
  "$schema": "https://mongolgpt.duckdns.org/config.json",
  "permission": {
    "grep": "allow"
  }
}
```

Өөрийн кодын сангаас хурдан контент хайх. Бүрэн регекс синтакс болон файлын загвар шүүлтүүрийг дэмждэг.

---

### бөмбөрцөг

Загвар тааруулах замаар файлуудыг олоорой.

```json title="mongolgpt.json" {4}
{
  "$schema": "https://mongolgpt.duckdns.org/config.json",
  "permission": {
    "glob": "allow"
  }
}
```

`**/*.js` эсвэл `src/**/*.ts` гэх мэт бөмбөрцөг загваруудыг ашиглан файл хайх. Өөрчлөлтийн хугацаагаар эрэмбэлсэн тохирох файлын замыг буцаана.

---

### lsp (туршилтын)

Тодорхойлолт, лавлагаа, хулганын мэдээлэл, дуудлагын шатлал зэрэг кодын тагнуулын функцуудыг авахын тулд өөрийн тохируулсан LSP серверүүдтэйгээ харилцаж болно.

:::note
Энэ хэрэгслийг зөвхөн `MONGOLGPT_EXPERIMENTAL_LSP_TOOL=true` (эсвэл `MONGOLGPT_EXPERIMENTAL=true`) үед ашиглах боломжтой.
:::

```json title="mongolgpt.json" {4}
{
  "$schema": "https://mongolgpt.duckdns.org/config.json",
  "permission": {
    "lsp": "allow"
  }
}
```

Дэмжигдсэн үйлдлүүд нь `goToDefinition`, `findReferences`, `hover`, `documentSymbol`, `workspaceSymbol`, `goToImplementation`, `prepareCallHierarchy`, `incomingCalls` болон `outgoingCalls`.

Танай төсөлд ямар LSP серверүүдийг ашиглах боломжтойг тохируулахын тулд [LSP серверүүд](/docs/lsp)-г үзнэ үү.

---

### нөхөөсийг хэрэглэнэ

Файлуудад засваруудыг ашиглах.

```json title="mongolgpt.json" {4}
{
  "$schema": "https://mongolgpt.duckdns.org/config.json",
  "permission": {
    "edit": "allow"
  }
}
```

Энэ хэрэгсэл нь нөхөөс файлуудыг таны кодын санд ашигладаг. Янз бүрийн эх сурвалжаас ялгаатай болон засваруудыг хэрэглэхэд хэрэгтэй.

`tool.execute.before` эсвэл `tool.execute.after` дэгээтэй харьцахдаа `input.tool === "apply_patch"` (`"patch"` биш) шалгана уу.

`apply_patch` `output.args.filePath`-ийн оронд `output.args.patchText` ашигладаг. Замууд нь `patchText` доторх тэмдэглэгээний мөрөнд суулгагдсан бөгөөд төслийн үндэстэй холбоотой байдаг (жишээ нь: `*** Add File: src/new-file.ts`, `*** Update File: src/existing.ts`, `*** Move to: src/renamed.ts`, `*** Delete File: src/obsolete.ts`).

:::note
`apply_patch` хэрэгсэл нь файлын бүх өөрчлөлтийг (`edit`, `write`, `apply_patch`) хамарсан `edit` зөвшөөрлөөр хянагддаг.
:::

---

### ур чадвар

[skill](/docs/skills) (`SKILL.md` файл) ачаалж, түүний агуулгыг харилцан ярианд буцаана уу.

```json title="mongolgpt.json" {4}
{
  "$schema": "https://mongolgpt.duckdns.org/config.json",
  "permission": {
    "skill": "allow"
  }
}
```

---

### энэ нь

Кодлох сессийн үеэр хийх зүйлсийн жагсаалтыг удирдах.

```json title="mongolgpt.json" {4}
{
  "$schema": "https://mongolgpt.duckdns.org/config.json",
  "permission": {
    "todowrite": "allow"
  }
}
```

Нарийн төвөгтэй үйл ажиллагааны явцад явцыг хянахын тулд даалгаврын жагсаалтыг үүсгэж, шинэчилдэг. LLM нь үүнийг олон алхамт ажлуудыг зохион байгуулахад ашигладаг.

:::note
Энэ хэрэгсэл нь анхдагчаар дэд агентуудад идэвхгүй болсон боловч та үүнийг гараар идэвхжүүлж болно. [Илүү ихийг мэдэж аваарай](/docs/agents/#permissions)
:::

---

### вэб татах

Вэб контентыг татах.

```json title="mongolgpt.json" {4}
{
  "$schema": "https://mongolgpt.duckdns.org/config.json",
  "permission": {
    "webfetch": "allow"
  }
}
```

LLM-д вэб хуудсыг татаж авах, унших боломжийг олгоно. Баримт бичиг хайх эсвэл онлайн эх сурвалжийг судлахад хэрэгтэй.

---

### вэб хайлт

Мэдээллийг вэбээс хайж олоорой.

:::note
Энэ хэрэгсэл нь зөвхөн MongolGPT үйлчилгээ үзүүлэгчийг ашиглах эсвэл `MONGOLGPT_ENABLE_EXA` орчны хувьсагчийг үнэн зөв утгаар тохируулсан үед л боломжтой (жишээ нь: `true` эсвэл `1`).

MongolGPT-г эхлүүлэх үед идэвхжүүлэхийн тулд:

```bash
MONGOLGPT_ENABLE_EXA=1 mongolgpt
```

:::

```json title="mongolgpt.json" {4}
{
  "$schema": "https://mongolgpt.duckdns.org/config.json",
  "permission": {
    "websearch": "allow"
  }
}
```

Холбогдох мэдээллийг онлайнаар олохын тулд Exa AI ашиглан вэб хайлт хийдэг. Сэдвийг судлах, одоогийн үйл явдлуудыг хайх, сургалтын мэдээллийн хязгаараас гадуур мэдээлэл цуглуулахад тустай.

API түлхүүр шаардлагагүй — уг хэрэгсэл нь баталгаажуулалтгүйгээр Exa AI-ийн байршуулсан MCP үйлчилгээтэй шууд холбогддог.

:::tip
Мэдээлэл (нээлт) олох шаардлагатай үед `websearch`, тодорхой URL (олж авах) -аас контент авах шаардлагатай бол `webfetch` ашиглана уу.
:::

---

### асуулт

Гүйцэтгэх явцад хэрэглэгчээс асуулт асуу.

```json title="mongolgpt.json" {4}
{
  "$schema": "https://mongolgpt.duckdns.org/config.json",
  "permission": {
    "question": "allow"
  }
}
```

Энэ хэрэгсэл нь LLM-д даалгавар гүйцэтгэх явцад хэрэглэгчээс асуулт асуух боломжийг олгодог. Энэ нь ашигтай:

- Хэрэглэгчийн сонголт, шаардлагыг цуглуулах
- Хоёрдмол утгатай зааврыг тодруулах
- Хэрэгжүүлэх сонголтын талаар шийдвэр гаргах
- Аль чиглэлийг сонгох талаар санал болгож байна

Асуулт бүр нь толгой хэсэг, асуултын текст, сонголтуудын жагсаалтыг агуулдаг. Хэрэглэгчид өгсөн сонголтуудаас сонгох эсвэл захиалгат хариулт бичих боломжтой. Олон асуулт байгаа үед хэрэглэгчид бүх хариултыг илгээхээсээ өмнө тэдгээрийн хооронд шилжих боломжтой.

---

## Тусгай хэрэгсэл

Захиалгат хэрэгслүүд нь LLM дуудаж болох өөрийн функцийг тодорхойлох боломжийг танд олгоно. Эдгээр нь таны тохиргооны файлд тодорхойлогдсон бөгөөд дурын кодыг ажиллуулж болно.

[Илүү ихийг мэдэж аваарай](/docs/custom-tools) захиалгат хэрэгсэл бий болгох талаар.

---

## MCP серверүүд

MCP (Загвар контекст протокол) серверүүд нь танд гадны хэрэгсэл, үйлчилгээг нэгтгэх боломжийг олгодог. Үүнд мэдээллийн санд нэвтрэх, API интеграцчлал, гуравдагч талын үйлчилгээ орно.

MCP серверүүдийг тохируулах талаар [Илүү ихийг мэдэж аваарай](/docs/mcp-servers).

## Дотоод

Дотооддоо `grep` болон `glob` зэрэг хэрэгслүүд бүрээсний доор [ripgrep](https://github.com/BurntSushi/ripgrep) ашигладаг. Анхдагч байдлаар, ripgrep нь `.gitignore` хэв маягийг хүндэтгэдэг бөгөөд энэ нь таны `.gitignore`-д орсон файлууд болон лавлахуудыг хайлт болон жагсаалтаас хасна гэсэн үг юм.

---

### Загварыг үл тоомсорло

Ердийн үед үл тоомсорлож болох файлуудыг оруулахын тулд төслийн үндсэн хэсэгт `.ignore` файл үүсгэнэ үү. Энэ файл нь тодорхой замыг зөвшөөрч болно.

```text title=".ignore"
!node_modules/
!dist/
!build/
```

Жишээ нь, энэ `.ignore` файл нь ripgrep-д
ode_modules/`, `dist/` болон `build/` лавлахууд нь `.gitignore`-д орсон байсан ч хайлт хийх боломжийг олгодог.
