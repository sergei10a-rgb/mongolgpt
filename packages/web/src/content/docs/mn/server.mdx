`mongolgpt serve` тушаал нь толгойгүй HTTP серверийг ажиллуулдаг бөгөөд энэ нь mongolgpt үйлчлүүлэгч ашиглаж болох НээлттэйAPI төгсгөлийн цэгийг харуулж байна.

---

### Хэрэглээ

```bash
mongolgpt serve [--port <number>] [--hostname <string>] [--cors <origin>]
```

#### Сонголтууд

| Туг             | Тодорхойлолт                           | Өгөгдмөл          |
| --------------- | -------------------------------------- | ----------------- |
| `--port`        | Сонсох порт                            | `4096`            |
| `--hostname`    | Сонсох хостын нэр                      | `127.0.0.1`       |
| `--mdns`        | mDNS илрүүлэлтийг идэвхжүүлэх          | `false`           |
| `--mdns-domain` | mDNS үйлчилгээний захиалгат домэйн нэр | `mongolgpt.local` |
| `--cors`        | Зөвшөөрөх нэмэлт хөтчийн гарал үүсэл   | `[]`              |

`--cors` хэд хэдэн удаа дамжуулж болно:

```bash
mongolgpt serve --cors http://localhost:5173 --cors https://app.example.com
```

---

### Баталгаажуулалт

Серверийг HTTP үндсэн баталгаажуулалтаар хамгаалахын тулд `MONGOLGPT_SERVER_PASSWORD`-г тохируулна уу. Хэрэглэгчийн нэр нь анхдагчаар `mongolgpt` эсвэл үүнийг хүчингүй болгохын тулд `MONGOLGPT_SERVER_USERNAME` гэж тохируулна уу. Энэ нь `mongolgpt serve` болон `mongolgpt web` хоёуланд нь хамаарна.

```bash
MONGOLGPT_SERVER_PASSWORD=your-password mongolgpt serve
```

---

### Энэ нь хэрхэн ажилладаг

`mongolgpt`-г ажиллуулахад энэ нь TUI болон серверийг эхлүүлнэ. TUI хаана байна
сервертэй ярьдаг үйлчлүүлэгч. Сервер нь OpenAPI 3.1 үзүүлэлтийг харуулж байна
эцсийн цэг. Энэ төгсгөлийн цэгийг мөн [SDK](/docs/sdk) үүсгэхэд ашигладаг.

:::tip
mongolgpt серверийг ашиглан mongolgpt программтай харьцана.
:::

Энэхүү архитектур нь mongolgpt-д олон үйлчлүүлэгчийг дэмжих боломжийг олгож, mongolgpt-тэй программын дагуу харилцах боломжийг олгодог.

Та бие даасан сервер эхлүүлэхийн тулд `mongolgpt serve` ажиллуулж болно. Хэрэв танд байгаа бол
mongolgpt TUI ажиллаж байгаа, `mongolgpt serve` шинэ сервер эхлүүлэх болно.

---

#### Одоо байгаа сервертэй холбогдоно уу

TUI-г эхлүүлэхэд энэ нь санамсаргүй байдлаар порт болон хостын нэрийг зааж өгдөг. Та үүний оронд `--hostname` болон `--port` [flags](/docs/cli)-ээр дамжуулж болно. Дараа нь сервертэй холбогдохын тулд үүнийг ашиглана уу.

[`/tui`](#tui) төгсгөлийн цэгийг TUI серверээр дамжуулан жолоодоход ашиглаж болно. Жишээлбэл, та сануулгыг урьдчилан бөглөх эсвэл ажиллуулах боломжтой. Энэ тохиргоог MongolGPT [IDE](/docs/ide) залгаасууд ашигладаг.

## Тодорхойлолт

Сервер нь OpenAPI 3.1 тодорхойлолтыг нийтэлдэг бөгөөд үүнийг дараах хаягаар үзэх боломжтой:

```
http://<hostname>:<port>/doc
```

Жишээлбэл, `http://localhost:4096/doc`. Үйлчлүүлэгч үүсгэх эсвэл хүсэлт, хариултын төрлүүдийг шалгахын тулд техникийн үзүүлэлтийг ашиглана уу. Эсвэл Swagger Explorer дээр үзэх.

### Файлууд

| арга  | Зам                      | Тодорхойлолт                      | Хариулт                                                                                        |
| ----- | ------------------------ | --------------------------------- | ---------------------------------------------------------------------------------------------- |
| `GET` | `/find?pattern=<pat>`    | Файлаас текст хайх                | `path`, `lines`, `line_number`, `absolute_offset`, `submatches`-тай тохирох объектуудын массив |

| `GET` | `/find/symbol?query=<q>` | Ажлын талбарын тэмдэгтүүдийг олох | <a href={typesUrl}><code>Тэмдэгт[]</code></a>                                                  |

| `GET` | `/file/status`           | Хянагдсан файлуудын статусыг авах | <a href={typesUrl}><code>Файл[]</code></a>                                                     |

#### `/find/file` асуулгын параметрүүд

- `query` (шаардлагатай) — хайлтын мөр (тодорхой таарахгүй)
- `type` (заавал биш) - үр дүнг `"file"` эсвэл `"directory"` гэж хязгаарлах
- `directory` (заавал биш) — хайлтын төслийн үндэсийг дарна
- `limit` (заавал биш) — хамгийн их үр дүн (1–200)
- `dirs` (заавал биш) — хуучин туг (`"false"` зөвхөн файлуудыг буцаана)
