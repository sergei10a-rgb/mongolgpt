---
title: Алдаа оношлох
description: Нийтлэг асуудлууд, тэдгээрийг хэрхэн шийдвэрлэх талаар.
---

MongolGPT-тэй холбоотой асуудлыг дибаг хийхдээ дискэн дээр хадгалсан бүртгэлүүд болон дотоод өгөгдлийг шалгаж эхэлнэ.

---

## Бүртгэл

Лог файлуудыг дараах руу бичдэг:

- **macOS/Linux**: `~/.local/share/mongolgpt/log/`
- **Windows**: `WIN+R` дарж, `%USERPROFILE%\.local\share\mongolgpt\log` дарна уу.

Лог файлуудыг цагийн тэмдэгээр нэрлэсэн (жишээ нь: `2025-01-09T123456.log`) бөгөөд хамгийн сүүлийн 10 бүртгэлийн файл хадгалагдана.

Та илүү нарийвчилсан дибаг хийх мэдээллийг авахын тулд `--log-level` тушаалын мөрийн сонголтоор бүртгэлийн түвшинг тохируулж болно. Жишээлбэл, `mongolgpt --log-level DEBUG`.

---

## Хадгалах

mongolgpt сессийн өгөгдөл болон бусад програмын өгөгдлийг дискэн дээр хадгалдаг:

- **macOS/Linux**: `~/.local/share/mongolgpt/`
- **Windows**: `WIN+R` дарж, `%USERPROFILE%\.local\share\mongolgpt` дарна уу.

Энэ лавлах нь:

- `auth.json` - API түлхүүр, OAuth жетон гэх мэт баталгаажуулалтын өгөгдөл
- `log/` - Програмын бүртгэл
- `project/` - Сеанс болон мессежийн өгөгдөл зэрэг төслийн тусгай өгөгдөл
  - Хэрэв төсөл нь Git репо дотор байгаа бол `./<project-slug>/storage/`-д хадгалагдана.
  - Хэрэв энэ нь Git репо биш бол `./global/storage/`-д хадгалагдана.

---

## Устгах

MongolGPT CLI-г устгаж, холбогдох файлуудыг устгахын тулд дараахыг ажиллуулна уу:

```bash
mongolgpt uninstall
```

Тушаал нь юу устгахыг харуулж, баталгаажуулахыг хүсэх болно. Тохиргоо эсвэл програмын өгөгдлийг хадгалах сонголтуудыг [CLI лавлагаа](/docs/cli#uninstall) харна уу.

MongolGPT Desktop-г устгахын тулд үйлдлийн системийнхээ програмын удирдлагын хэрэгслээр дамжуулан програмыг устгана уу.

---

## Ширээний програм

MongolGPT Ширээний компьютер нь орон нутгийн MongolGPT серверийг (`mongolgpt-cli` хажуугийн хайрцаг) ар талд ажиллуулдаг. Ихэнх асуудлууд залгаас, гэмтсэн кэш, серверийн тохиргоо муутай зэргээс шалтгаална.

### Шуурхай шалгалт

- Бүрэн гарч, програмаа дахин ажиллуулна уу.
- Хэрэв програмд алдаа гарсан дэлгэц гарч байвал **Дахин эхлүүлэх** дээр дарж алдааны мэдээллийг хуулна.
- Зөвхөн macOS: `MongolGPT` цэс -> **Вэб харахыг дахин ачаалах** (хэрэв UI хоосон/царцаасан тохиолдолд тусална).

---

### залгаасуудыг идэвхгүй болгох

Хэрэв ширээний програмыг эхлүүлэх үед гацаж, унжсан эсвэл хачирхалтай ажиллаж байвал залгаасуудыг идэвхгүй болго.

#### Глобал тохиргоог шалгана уу

Глобал тохиргооны файлаа нээгээд `plugin` түлхүүрийг олоорой.

- **macOS/Linux**: `~/.config/mongolgpt/mongolgpt.jsonc` (эсвэл `~/.config/mongolgpt/mongolgpt.json`)
- **macOS/Linux** (хуучин суулгацууд): `~/.local/share/mongolgpt/mongolgpt.jsonc`
- **Windows**: `WIN+R` дарж, `%USERPROFILE%\.config\mongolgpt\mongolgpt.jsonc` дарна уу.

Хэрэв танд тохируулсан залгаасууд байгаа бол түлхүүрийг арилгах эсвэл хоосон массив болгон тохируулах замаар тэдгээрийг түр идэвхгүй болго.

```jsonc
{
  "$schema": "https://mgpt.mn/config.json",
  "plugin": [],
}
```

#### Нэмэлтийн сангуудыг шалгах

MongolGPT нь мөн дискнээс локал залгаасуудыг ачаалж болно. Эдгээрийг замаас түр зуур зөөж (эсвэл фолдерын нэрийг өөрчлөх) болон ширээний програмыг дахин эхлүүлнэ үү:

- **Глобал залгаасууд**
  - **macOS/Linux**: `~/.config/mongolgpt/plugins/`
  - **Windows**: `WIN+R` дарж, `%USERPROFILE%\.config\mongolgpt\plugins` дарна уу.
- **Төслийн залгаасууд** (зөвхөн та төсөл бүрийн тохиргоог ашигладаг бол)
  - `<your-project>/.mongolgpt/plugins/`

Хэрэв програм дахин ажиллаж эхэлбэл аль нь асуудал үүсгэж байгааг олохын тулд залгаасуудыг нэг нэгээр нь дахин идэвхжүүлнэ үү.

---

### Кэшийг цэвэрлэ

Хэрэв залгаасуудыг идэвхгүй болгох нь тус болохгүй (эсвэл залгаас суулгахад гацсан бол MongolGPT үүнийг дахин бүтээхийн тулд кэшийг цэвэрлэ.

1. MongolGPT Десктопоос бүрмөсөн гарах.
2. Кэшийн лавлахыг устгана уу:

- **macOS**: Finder -> `Cmd+Shift+G` -> paste `~/.cache/mongolgpt`
- **Linux**: устгах `~/.cache/mongolgpt` (эсвэл `rm -rf ~/.cache/mongolgpt` ажиллуулах)
- **Windows**: `WIN+R` дарж, `%USERPROFILE%\.cache\mongolgpt` дарна уу.

3. MongolGPT Десктопыг дахин эхлүүлнэ үү.

---

### Серверийн холболтын асуудлыг засах

MongolGPT Ширээний компьютер нь өөрийн дотоод серверийг эхлүүлэх (анхдагч) эсвэл таны тохируулсан URL серверт холбогдох боломжтой.

Хэрэв та **"Холболт амжилтгүй болсон"** харилцах цонхыг харвал (эсвэл програм хэзээ ч дэлгэцийн өмнө гарч ирэхгүй), захиалгат сервер URL байгаа эсэхийг шалгана уу.

#### Ширээний үндсэн серверийг арилгах URL

Үндсэн дэлгэцээс серверийн нэрийг (төлөв цэгтэй) дарж Сервер сонгогчийг нээнэ үү. **Үндсэн сервер** хэсэгт **Clear** дээр дарна уу.

#### Өөрийн тохиргооноос `server.port` / `server.hostname` устгана уу

Хэрэв таны `mongolgpt.json(c)`-д `server` хэсэг байгаа бол түр устгаад ширээний програмыг дахин эхлүүлнэ үү.

#### Орчны хувьсагчдыг шалгана уу

Хэрэв таны орчинд `MONGOLGPT_PORT` тохируулагдсан бол ширээний програм нь тухайн портыг дотоод серверт ашиглахыг оролдох болно.

- `MONGOLGPT_PORT` тохиргоог болиулж (эсвэл үнэгүй порт сонгоод) дахин эхлүүлнэ үү.

---

### Linux: Wayland / X11 асуудлууд

Linux дээр Wayland-н зарим тохиргоо нь хоосон цонх эсвэл нэгтгэгчийн алдаа үүсгэж болзошгүй.

- Хэрэв та Wayland дээр байгаа бөгөөд програм хоосон/гажигтай байвал `OC_ALLOW_WAYLAND=1`-р ажиллуулж үзнэ үү.
- Хэрэв энэ нь байдлыг улам дордуулах юм бол устгаад оронд нь X11 сессийн дагуу ажиллуулж үзнэ үү.

---

### Windows: WebView2 ажиллах хугацаа

Windows, MongolGPT Ширээний компьютер дээр Microsoft Edge **WebView2 Runtime** шаардлагатай. Хэрэв програм хоосон цонхонд нээгдэх эсвэл эхлэхгүй бол WebView2-г суулгаж/шинэчлээд дахин оролдоно уу.

---

### Windows: Гүйцэтгэлийн ерөнхий асуудлууд

Хэрэв та Windows дээр гүйцэтгэл удаашрах, файлд хандах асуудал эсвэл терминалын асуудалтай тулгарвал [WSL (Linux-д зориулсан Windows дэд систем)](/docs/windows-wsl)-г ашиглана уу. WSL нь MongolGPT-ийн онцлогтой илүү жигд ажилладаг Linux орчинг бүрдүүлдэг.

---

### Мэдэгдэл харагдахгүй байна

MongolGPT Ширээний компьютер нь зөвхөн дараах тохиолдолд системийн мэдэгдлийг харуулдаг.

- Таны OS тохиргоонд MongolGPT-д мэдэгдлүүд идэвхжсэн, мөн
- програмын цонх анхаарлаа төвлөрүүлээгүй байна.

---

### Ширээний програмын санг дахин тохируулах (хамгийн сүүлийн арга)

Хэрэв програм эхлэхгүй бөгөөд та UI доторх тохиргоог арилгах боломжгүй бол ширээний програмын хадгалсан төлөвийг дахин тохируулна уу.

1. MongolGPT Десктопоос гарна.
2. Эдгээр файлуудыг олж устгана уу (тэд MongolGPT Ширээний програмын мэдээллийн санд амьдардаг):

- `mongolgpt.settings.dat` (ширээний үндсэн сервер URL)
- `mongolgpt.global.dat` болон `mongolgpt.workspace.*.dat` (UI сүүлийн үеийн серверүүд/төслүүд шиг)

Лавлахыг хурдан олохын тулд:

- **macOS**: Finder -> `Cmd+Shift+G` -> `~/Library/Application Support` (дараа нь дээрх файлын нэрийг хайна уу)
- **Linux**: дээрх файлын нэрийг `~/.local/share` доороос хайна уу
- **Windows**: `WIN+R` -> `%APPDATA%` дарна уу (дараа нь дээрх файлын нэрийг хайна уу)

## Нийтлэг асуудлууд

Зарим нийтлэг асуудлууд, тэдгээрийг хэрхэн шийдвэрлэх талаар энд оруулав.

---

### MongolGPT эхлэхгүй

1. Алдааны мэдэгдлийн бүртгэлийг шалгана уу
2. Терминал дахь гаралтыг харахын тулд `--print-logs` гэж ажиллуулж үзнэ үү
3. Та `mongolgpt upgrade` гэсэн хамгийн сүүлийн хувилбартай эсэхийг шалгаарай.

---

### Баталгаажуулалтын асуудал

1. TUI дээрх `/connect` командыг ашиглан дахин баталгаажуулахыг оролдоно уу.
2. API түлхүүрүүд хүчинтэй эсэхийг шалгана уу
3. Таны сүлжээ үйлчилгээ үзүүлэгчийн API-д холбогдохыг зөвшөөрч байгаа эсэхийг шалгаарай.

---

### Загвар байхгүй

1. Үйлчилгээ үзүүлэгчээс баталгаажуулсан эсэхээ шалгана уу
2. Өөрийн тохиргоонд байгаа загварын нэр зөв эсэхийг шалгана уу
3. Зарим загвар нь тусгай хандалт эсвэл захиалга шаарддаг

Хэрэв та `ProviderModelNotFoundError`-тэй таарвал та буруу байх магадлалтай
хаа нэгтээ загвараас лавлаж байна.
Загваруудыг дараах байдлаар дурдах ёстой: `<providerId>/<modelId>`

Examples:

- `openai/gpt-4.1`
- `openrouter/google/gemini-2.5-flash`
- `mongolgpt/kimi-k2`

Та ямар загварт хандах боломжтойг мэдэхийн тулд `mongolgpt models` гэж ажиллуулна уу.

---

### ProviderInitError

Хэрэв та ProviderInitError-тэй тулгарвал та буруу эсвэл эвдэрсэн тохиргоотой байх магадлалтай.

Үүнийг шийдэхийн тулд:

1. Эхлээд [үйлчилгээ үзүүлэгчдийн гарын авлага](/docs/providers)-г дагаж үйлчилгээ үзүүлэгчээ зөв тохируулсан эсэхийг шалгана уу.
2. Хэрэв асуудал хэвээр байвал хадгалсан тохиргоогоо устгаж үзнэ үү:

   ```bash
   rm -rf ~/.local/share/mongolgpt
   ```

   Windows дээр `WIN+R` дарж устгана уу: `%USERPROFILE%\.local\share\mongolgpt`

3. TUI дээрх `/connect` командыг ашиглан үйлчилгээ үзүүлэгчтэйгээ дахин баталгаажуулна уу.

---

### AI_APICallError болон үйлчилгээ үзүүлэгчийн багцын асуудлууд

Хэрэв та API дуудлагын алдаатай тулгарвал энэ нь үйлчилгээ үзүүлэгчийн багц хуучирсантай холбоотой байж болох юм. mongolgpt нь шаардлагатай бол үйлчилгээ үзүүлэгчийн багцуудыг (OpenAI, Anthropic, Google гэх мэт) динамикаар суулгаж, тэдгээрийг дотооддоо кэш болгодог.

Үйлчилгээ үзүүлэгчийн багцын асуудлыг шийдвэрлэхийн тулд:

1. Үйлчилгээ үзүүлэгчийн багцын кэшийг арилгах:

   ```bash
   rm -rf ~/.cache/mongolgpt
   ```

   Windows дээр `WIN+R` дарж устгана уу: `%USERPROFILE%\.cache\mongolgpt`

2. Хамгийн сүүлийн үеийн үйлчилгээ үзүүлэгчийн багцуудыг дахин суулгахын тулд mongolgpt-г дахин эхлүүлнэ үү

Энэ нь mongolgpt-г үйлчилгээ үзүүлэгчийн багцын хамгийн сүүлийн хувилбарыг татахад хүргэх бөгөөд энэ нь загварын параметрүүд болон API өөрчлөлтүүдтэй нийцтэй байдлын асуудлыг шийддэг.

---

### Хуулах/буулгах нь Linux дээр ажиллахгүй байна

Linux хэрэглэгчид хуулах/буулгах функцийг ажиллуулахын тулд дараах санах ойн хэрэгслүүдийн аль нэгийг суулгасан байх шаардлагатай:

**X11 системийн хувьд:**

```bash
apt install -y xclip
# Эсвэл
apt install -y xsel
```

** Wayland системийн хувьд:**

```bash
apt install -y wl-clipboard
```

**Толгойгүй орчинд:**

```bash
apt install -y xvfb
# Дараа нь ажиллуулах:
Xvfb :99 -screen 0 1024x768x24 > /dev/null 2>&1 &
export DISPLAY=:99.0
```

mongolgpt таныг Wayland ашиглаж байгаа эсэхийг илрүүлж, `wl-clipboard`-г илүүд үзэх болно, эс бөгөөс энэ нь санах ойн хэрэгслүүдийг дараах дарааллаар хайх болно: `xclip` болон `xsel`.
