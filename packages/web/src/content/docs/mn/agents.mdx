---
title: Агентууд
description: Мэргэшсэн агентуудыг тохируулах, ашиглах.
---

Агентууд нь тусгай даалгавар болон ажлын урсгалд тохируулж болох AI тусгай туслахууд юм. Эдгээр нь танд тусгай сануулга, загвар, хэрэгсэлд хандах хандалт бүхий төвлөрсөн хэрэгслийг бий болгох боломжийг олгодог.

:::tip
Төлөвлөгөөний төлөөлөгчийг ашиглан кодын дүн шинжилгээ хийж, ямар нэгэн кодын өөрчлөлт хийхгүйгээр саналуудыг хянана.
:::

Та сессийн үеэр төлөөлөгчдийн хооронд шилжих эсвэл `@` дурдахад тэднийг дуудаж болно.

---

## Төрөл

MongolGPT-д хоёр төрлийн агент байдаг; үндсэн бодис ба дэд агентууд.

---

### Анхдагч агентууд

Анхдагч агентууд бол таны шууд харилцдаг гол туслахууд юм. Та **Tab** товчлуур эсвэл өөрийн тохируулсан `switch_agent` товчлуурыг ашиглан тэдгээрийн дундуур эргэлдэж болно. Эдгээр агентууд таны үндсэн яриаг зохицуулдаг. Хэрэгслийн хандалтыг зөвшөөрлөөр тохируулдаг — жишээлбэл, Төлөвлөгөө хязгаарлагдсан үед Build-д бүх хэрэгслийг идэвхжүүлсэн байна.

:::tip
Та сессийн үеэр үндсэн агентуудын хооронд шилжихийн тулд **Tab** товчлуурыг ашиглаж болно.
:::

MongolGPT нь **Build** болон **Plan** гэсэн хоёр үндсэн агенттай ирдэг. Бид болно
эдгээрийг доороос харна уу.

---

### Дэд агентууд

Дэд агентууд нь үндсэн агентууд тодорхой ажлуудад дуудаж болох тусгай туслахууд юм. Та мөн тэдгээрийг мессежэндээ **@ дурдах** замаар гараар дуудаж болно.

MongolGPT нь **General**, **Explore**, **Scout** гэсэн гурван дэд агенттай ирдэг. Үүнийг бид доор авч үзэх болно.

---

## Баригдсан

MongolGPT нь хоёр үндсэн агент, гурван дэд агенттай ирдэг.

---

### Бүтэц ашиглах

_Мод_: `primary`

Build нь бүх хэрэгслийг идэвхжүүлсэн **өгөгдмөл** үндсэн агент юм. Энэ бол файлын үйлдлүүд болон системийн командуудад бүрэн хандах шаардлагатай хөгжүүлэлтийн ажлын стандарт агент юм.

---

### Төлөвлөгөө ашигла

_Мод_: `primary`

Төлөвлөлт, дүн шинжилгээ хийхэд зориулагдсан хязгаарлагдмал агент. Бид танд илүү их хяналт тавьж, хүсээгүй өөрчлөлтөөс урьдчилан сэргийлэхийн тулд зөвшөөрлийн системийг ашигладаг.
Өгөгдмөлөөр дараах бүх зүйлийг `ask` гэж тохируулсан:

- `file edits`: Бүх бичих, засварлах, засварлах
- `bash`: Бүх bash командууд

Энэ агент нь LLM кодыг шинжлэх, өөрчлөлт санал болгох эсвэл кодын санд ямар нэгэн бодит өөрчлөлт хийхгүйгээр төлөвлөгөө гаргахыг хүсвэл хэрэг болно.

---

### Ерөнхий ашиглах

_Мод_: `subagent`

Нарийн төвөгтэй асуултуудыг судлах, олон шатлалт даалгавруудыг гүйцэтгэх ерөнхий зориулалтын агент. Хэрэгслийн бүрэн хандалттай (хийхээс бусад) тул шаардлагатай үед файлд өөрчлөлт оруулах боломжтой. Үүнийг ашиглан олон нэгж ажлыг зэрэгцүүлэн ажиллуул.

---

### Explore агент ашиглах

_Мод_: `subagent`

Кодын санг судлахад зориулагдсан хурдан, зөвхөн унших боломжтой агент. Файлуудыг өөрчлөх боломжгүй. Файлуудыг хэв маягаар нь хурдан олох, түлхүүр үг хайх код эсвэл кодын сангийн талаархи асуултуудад хариулах шаардлагатай үед үүнийг ашиглана уу.

---

### Скаут ашигла

_Мод_: `subagent`

Гадны баримт бичиг болон хараат байдлын судалгаанд зориулсан зөвхөн унших боломжтой агент. Хэрэв та өөрийн ажлын талбарыг өөрчлөхгүйгээр MongolGPT-ийн удирдаж буй кэш рүү хамаарлын агуулахыг хувилах, номын сангийн эх сурвалжийг шалгах эсвэл дээд талын хэрэгжилтийн эсрэг локал кодыг хөндлөн лавлах шаардлагатай үед үүнийг ашиглаарай.

---

### Нягтруулж хэрэглэх

_Мод_: `primary`

Урт контекстийг жижиг хураангуй болгон нягтруулдаг далд системийн агент. Энэ нь шаардлагатай үед автоматаар ажилладаг бөгөөд UI-д сонгох боломжгүй.

---

### Гарчиг ашиглана уу

_Мод_: `primary`

Богино сессийн гарчиг үүсгэдэг далд системийн агент. Энэ нь автоматаар ажилладаг бөгөөд UI-д сонгох боломжгүй.

---

### Хураангуйг ашиглана уу

_Мод_: `primary`

Сешн хураангуйг үүсгэдэг далд системийн агент. Энэ нь автоматаар ажилладаг бөгөөд UI-д сонгох боломжгүй.

---

## Хэрэглээ

1. Анхдагч агентуудын хувьд **Tab** товчийг ашиглан сессийн үеэр тэдгээрийн хооронд эргэлдэнэ. Та мөн өөрийн тохируулсан `switch_agent` товчлуурыг ашиглаж болно.

2. Дэд агентуудыг дуудаж болно:
   - **Автоматаар** үндсэн агентууд нь тэдний тайлбар дээр үндэслэн тусгай даалгавар.
   - Таны зурваст дэд агентыг **@ дурдах** замаар гараар. Жишээ нь.

     ```txt frame="none"
     @general help me search for this function
     ```

3. **Сурц хоорондын навигаци**: Дэд агентууд хүүхэд сесс үүсгэх үед `session_child_first` (өгөгдмөл: **\<Leader>+Down**) ашиглан эцэг эхээс эхний хүүхэд сессийг оруулна уу.

4. Хүүхдийн хуралдаанд орсны дараа дараахыг ашиглана уу:
   - `session_child_cycle` (өгөгдмөл: **Баруун**) дараагийн хүүхдийн сесс рүү шилжих
   - `session_child_cycle_reverse` (өгөгдмөл: **Зүүн**) өмнөх хүүхэд сесс рүү шилжих
   - `session_parent` (өгөгдмөл: **Дээш**) эцэг эхийн хуралдаан руу буцна

   Энэ нь танд үндсэн яриа болон тусгай дэд агентын ажил хооронд шилжих боломжийг олгоно.

---

## Тохируулах

Та суулгасан агентуудыг тохируулах эсвэл тохиргоогоор дамжуулан өөрөө үүсгэж болно. Агентуудыг хоёр аргаар тохируулж болно:

---

### JSON

Өөрийн `mongolgpt.json` тохиргооны файл дахь агентуудыг тохируулна уу:

```json title="mongolgpt.json"
{
  "$schema": "https://mgpt.mn/config.json",
  "agent": {
    "build": {
      "mode": "primary",
      "model": "anthropic/claude-sonnet-4-20250514",
      "prompt": "{file:./prompts/build.txt}",
      "permission": {
        "edit": "allow",
        "bash": "allow"
      }
    },
    "plan": {
      "mode": "primary",
      "model": "anthropic/claude-haiku-4-20250514",
      "permission": {
        "edit": "deny",
        "bash": "deny"
      }
    },
    "code-reviewer": {
      "description": "Кодын зөв дадал болон болзошгүй асуудлыг хянана",
      "mode": "subagent",
      "model": "anthropic/claude-sonnet-4-20250514",
      "prompt": "Та код хянагч. Аюулгүй байдал, гүйцэтгэл, засварлахад хялбар байдалд төвлөр.",
      "permission": {
        "edit": "deny"
      }
    }
  }
}
```

---

### Тэмдэглэгээ

Та мөн тэмдэглэгээний файлуудыг ашиглан агентуудыг тодорхойлж болно. Тэдгээрийг байрлуул:

- Глобал: `~/.config/mongolgpt/agents/`
- Төсөл тус бүр: `.mongolgpt/agents/`

```markdown title="~/.config/mongolgpt/agents/review.md"
---
description: Кодын чанар болон зөв дадлыг хянана
mode: subagent
model: anthropic/claude-sonnet-4-20250514
temperature: 0.1
permission:
  edit: deny
  bash: deny
---

Та код хянах горимд ажиллаж байна. Дараах зүйлд төвлөр:

- Кодын чанар болон зөв дадал
- Болзошгүй алдаа болон захын тохиолдол
- Гүйцэтгэлд үзүүлэх нөлөө
- Аюулгүй байдлын асуудал

Кодод шууд өөрчлөлт хийхгүйгээр хэрэгжүүлэх боломжтой санал өг.
```

Маркdown файлын нэр нь агентын нэр болно. Жишээ нь `review.md` нь `review` агент үүсгэдэг.

---

## Сонголт

Эдгээр тохиргооны сонголтыг нарийвчлан авч үзье.

---

### Тодорхойлолт

`description` сонголтыг ашиглан төлөөлөгч юу хийдэг, хэзээ ашиглах талаар товч тайлбар өгнө үү.

```json title="mongolgpt.json"
{
  "agent": {
    "review": {
      "description": "Кодын зөв дадал болон болзошгүй асуудлыг хянана"
    }
  }
}
```

Энэ нь **шаардлагатай** тохиргооны сонголт юм.

---

### Температур

LLM-ийн хариултуудын санамсаргүй байдал, бүтээлч байдлыг `temperature` тохиргоогоор хяна.

Бага үнэ цэнэ нь хариултыг илүү төвлөрч, шийдэмгий болгодог бол өндөр утга нь бүтээлч байдал, хувьсах чадварыг нэмэгдүүлдэг.

```json title="mongolgpt.json"
{
  "agent": {
    "plan": {
      "temperature": 0.1
    },
    "creative": {
      "temperature": 0.8
    }
  }
}
```

Температурын утга нь ихэвчлэн 0.0-1.0 хооронд хэлбэлздэг:

- **0.0-0.2**: Маш төвлөрсөн, тодорхой хариултууд, кодын дүн шинжилгээ хийх, төлөвлөхөд тохиромжтой
- **0.3-0.5**: Бүтээлч байдал бүхий тэнцвэртэй хариултууд, ерөнхий хөгжлийн даалгаварт тохиромжтой
- **0.6-1.0**: Илүү бүтээлч, олон талт хариултууд нь оюуны довтолгоо, эрэл хайгуул хийхэд хэрэгтэй.

```json title="mongolgpt.json"
{
  "agent": {
    "analyze": {
      "temperature": 0.1,
      "prompt": "{file:./prompts/analysis.txt}"
    },
    "build": {
      "temperature": 0.3
    },
    "brainstorm": {
      "temperature": 0.7,
      "prompt": "{file:./prompts/creative.txt}"
    }
  }
}
```

Хэрэв температурыг заагаагүй бол MongolGPT загварт хамаарах анхдагч утгыг ашиглана; Ихэнх загваруудад ихэвчлэн 0, Qwen загваруудад 0.55 байдаг.

---

### Хамгийн их алхам

Зөвхөн текстээр хариу өгөхөөс өмнө агентын хийж чадах хамгийн их тооны төлөөлөгч давталтыг хяна. Энэ нь зардлаа хянах хүсэлтэй хэрэглэгчдэд агентын үйлдэлд хязгаарлалт тавих боломжийг олгодог.

Хэрэв үүнийг тохируулаагүй бол загвар зогсоохоор сонгох эсвэл хэрэглэгч сессийг тасалдуулах хүртэл агент давталтыг үргэлжлүүлнэ.

```json title="mongolgpt.json"
{
  "agent": {
    "quick-thinker": {
      "description": "Цөөн давталтаар хурдан дүгнэнэ",
      "prompt": "Та хурдан дүгнэгч. Асуудлыг хамгийн цөөн алхмаар шийд.",
      "steps": 5
    }
  }
}
```

Хязгаарт хүрэхэд агент нь ажлынхаа хураангуй, санал болгосон үлдсэн ажлуудын хариуг өгөхийг зааварласан тусгай системийн хүсэлтийг хүлээн авдаг.

:::caution
Хуучин `maxSteps` талбар хуучирсан. Оронд нь `steps` ашиглана уу.
:::

---

### Идэвхгүй болгох

Агентийг идэвхгүй болгохын тулд `true` гэж тохируулна уу.

```json title="mongolgpt.json"
{
  "agent": {
    "review": {
      "disable": true
    }
  }
}
```

---

### Шуурхай

`prompt` тохиргоотой энэ агентын системийн хүсэлтийн файлыг зааж өгнө үү. Мэдээллийн файл нь төлөөлөгчийн зорилгод зориулагдсан зааврыг агуулсан байх ёстой.

```json title="mongolgpt.json"
{
  "agent": {
    "review": {
      "prompt": "{file:./prompts/code-review.txt}"
    }
  }
}
```

Энэ зам нь тохиргооны файл хаана байрладагтай холбоотой юм. Тиймээс энэ нь дэлхийн MongolGPT тохиргоо болон төслийн тусгай тохиргоонд хоёуланд нь ажилладаг.

---

### Загвар

Энэ агентын загварыг хүчингүй болгохын тулд `model` тохиргоог ашиглана уу. Янз бүрийн даалгаварт тохирсон өөр өөр загваруудыг ашиглахад хэрэгтэй. Жишээлбэл, төлөвлөлтийн илүү хурдан загвар, хэрэгжүүлэхэд илүү чадвартай загвар.

:::tip
Хэрэв та загвар заагаагүй бол үндсэн агентууд [дэлхийн хэмжээнд тохируулсан загвар](/docs/config#models)-г ашигладаг бол дэд агентууд дэд агентыг дуудсан үндсэн агентын загварыг ашиглана.
:::

```json title="mongolgpt.json"
{
  "agent": {
    "plan": {
      "model": "anthropic/claude-haiku-4-20250514"
    }
  }
}
```

MongolGPT тохиргооны загварын ID нь `provider/model-id` форматтай. Жишээлбэл, OpenRouter дээрх Gemini 2.5 Flash загварыг `openrouter/google/gemini-2.5-flash` гэж заана. Free Auto ашиглах бол [бүртгэл ба багцын зааврыг](/docs/account) үзнэ үү.

---

### Хэрэгсэл (хуучирсан)

`tools` нь **хуучирсан**. Шинэ тохиргоо, шинэчлэлт болон илүү нарийн хяналтыг хийхийн тулд төлөөлөгчийн [`зөвшөөрөл`](#permissions) талбарыг илүүд үзээрэй.

Энэ агент дээр ямар хэрэгслүүд байгааг хянах боломжийг танд олгоно. Та тодорхой хэрэгслүүдийг `true` эсвэл `false` болгож идэвхжүүлж эсвэл идэвхгүй болгож болно. Агентын `tools` тохиргоонд `true` нь `{"*": "allow"}` зөвшөөрөлтэй, `false` нь `{"*": "deny"}` зөвшөөрөлтэй тэнцүү байна.

```json title="mongolgpt.json" {3-6,9-12}
{
  "$schema": "https://mgpt.mn/config.json",
  "tools": {
    "write": true,
    "bash": true
  },
  "agent": {
    "plan": {
      "tools": {
        "write": false,
        "bash": false
      }
    }
  }
}
```

:::note
Агентд зориулсан тохиргоо нь глобал тохиргоог дарна.
:::

Та мөн олон хэрэгслийг нэгэн зэрэг удирдахын тулд хуучин `tools` оруулгууд дахь орлуулагч тэмдэгтүүдийг ашиглаж болно. Жишээлбэл, MCP серверийн бүх хэрэгслийг идэвхгүй болгохын тулд:

```json title="mongolgpt.json"
{
  "$schema": "https://mgpt.mn/config.json",
  "agent": {
    "readonly": {
      "tools": {
        "mymcp_*": false,
        "write": false,
        "edit": false
      }
    }
  }
}
```

[Хэрэгслийн талаар илүү ихийг мэдэж аваарай](/docs/tools).

---

### Зөвшөөрөл

Агент ямар үйлдэл хийж болохыг удирдахын тулд та зөвшөөрлийг тохируулах боломжтой. Зөвшөөрлийн түлхүүр бүрийг дараах байдлаар тохируулж болно:

- `"ask"` — Хэрэгслийг ажиллуулахын өмнө зөвшөөрөл авах
- `"allow"` - Зөвшөөрөлгүйгээр бүх үйлдлийг зөвшөөрөх
- `"deny"` - Хэрэгслийг идэвхгүй болгох

Боломжтой зөвшөөрлийн түлхүүрүүд нь:

| Түлхүүр              | Энэ хаалганы хэрэгсэл                                            |
| -------------------- | ---------------------------------------------------------------- |
| `read`               | `read`                                                           |
| `edit`               | `write`, `edit`, `apply_patch`                                   |
| `glob`               | `glob`                                                           |
| `grep`               | `grep`                                                           |
| `list`               | `list`                                                           |
| `bash`               | `bash`                                                           |
| `task`               | `task`                                                           |
| `external_directory` | Төслийн ажлын модны гадна файлуудыг унших, бичих аливаа хэрэгсэл |
| `todowrite`          | `todowrite`, `todoread`                                          |
| `webfetch`           | `webfetch`                                                       |
| `websearch`          | `websearch`                                                      |
| `lsp`                | `lsp`                                                            |
| `skill`              | `skill`                                                          |
| `question`           | `question`                                                       |
| `doom_loop`          | Агент гацсан мэт санагдах үед сэргээх хүсэлт гаргана             |

`read`, `edit`, `glob`, `grep`, `list`, `bash`, `task`, `external_directory`, `lsp`, болон `skill` нь богино хэмжээний үйлдэл (`"allow" | "ask" | "deny"`) эсвэл бөмбөрцөг → торгуулийн үйлдэл/хяналтын объектыг хүлээн зөвшөөрдөг. Үлдсэн товчлуурууд нь зөвхөн товчилсон үйлдлийг хүлээн авдаг.

:::note
Зөвшөөрлийн түлхүүрүүд нь үндсэн хэрэгслийн нэрийн эсрэг орлуулагч тэмдэгтийн загвартай таарч байгаа тул уг синтакс нь суулгасан хэрэгсэл, захиалгат хэрэгсэл болон MCP хэрэгслүүдэд ажилладаг - жишээлбэл `"mymcp_*": "deny"` нь MCP серверийн хэрэгсэл бүрийг үгүйсгэдэг ба `"mymcp_search": "ask"` нь нэгийг онилдог.
:::

```json title="mongolgpt.json"
{
  "$schema": "https://mgpt.mn/config.json",
  "permission": {
    "edit": "deny"
  }
}
```

Та эдгээр зөвшөөрлийг агент тус бүрээр нь хүчингүй болгож болно.

```json title="mongolgpt.json" {3-5,8-10}
{
  "$schema": "https://mgpt.mn/config.json",
  "permission": {
    "edit": "deny"
  },
  "agent": {
    "build": {
      "permission": {
        "edit": "ask"
      }
    }
  }
}
```

Та мөн Markdown агентууд дээр зөвшөөрлийг тохируулах боломжтой.

```markdown title="~/.config/mongolgpt/agents/review.md"
---
description: Өөрчлөлт хийхгүйгээр код хянана
mode: subagent
permission:
  edit: deny
  bash:
    "*": ask
    "git diff": allow
    "git log*": allow
    "grep *": allow
  webfetch: deny
---

Only analyze code and suggest changes.
```

Та тодорхой bash командын зөвшөөрлийг тохируулж болно.

```json title="mongolgpt.json" {7}
{
  "$schema": "https://mgpt.mn/config.json",
  "agent": {
    "build": {
      "permission": {
        "bash": {
          "git push": "ask",
          "grep *": "allow"
        }
      }
    }
  }
}
```

Энэ нь бөмбөрцгийн загварыг авч болно.

```json title="mongolgpt.json" {7}
{
  "$schema": "https://mgpt.mn/config.json",
  "agent": {
    "build": {
      "permission": {
        "bash": {
          "git *": "ask"
        }
      }
    }
  }
}
```

Мөн та бүх тушаалын зөвшөөрлийг удирдахын тулд `*` тэмдэгтийг ашиглаж болно.
Сүүлийн тохирох дүрэм давуу эрхтэй тул `*` орлуулагч тэмдэгтийг, дараа нь тусгай дүрмүүдийг тавина.

```json title="mongolgpt.json" {8}
{
  "$schema": "https://mgpt.mn/config.json",
  "agent": {
    "build": {
      "permission": {
        "bash": {
          "*": "ask",
          "git status *": "allow"
        }
      }
    }
  }
}
```

[Зөвшөөрлийн талаар нэмэлт мэдээлэл аваарай](/docs/permissions).

---

### горим

Агентын горимыг `mode` тохиргоогоор удирдана уу. `mode` сонголтыг агентийг хэрхэн ашиглаж болохыг тодорхойлоход ашигладаг.

```json title="mongolgpt.json"
{
  "agent": {
    "review": {
      "mode": "subagent"
    }
  }
}
```

`mode` сонголтыг `primary`, `subagent` эсвэл `all` гэж тохируулж болно. Хэрэв `mode`-г заагаагүй бол энэ нь анхдагчаар `all` болно.

---

### Нуугдсан

Дэд агентийг `@` автоматаар гүйцээх цэснээс `hidden: true` ашиглан нуу. Зөвхөн Task хэрэгслээр дамжуулан бусад агентууд программчлан дуудах ёстой дотоод дэд агентуудад хэрэгтэй.

```json title="mongolgpt.json"
{
  "agent": {
    "internal-helper": {
      "mode": "subagent",
      "hidden": true
    }
  }
}
```

Энэ нь зөвхөн автоматаар бөглөх цэсэнд хэрэглэгчийн харагдах байдалд нөлөөлнө. Зөвшөөрөл зөвшөөрсөн тохиолдолд далд агентуудыг Task хэрэгслээр дамжуулан загвар дуудаж болно.

:::note
Зөвхөн `mode: subagent` агентуудад хамаарна.
:::

---

### Ажлын зөвшөөрөл

Task хэрэгслээр дамжуулан төлөөлөгч ямар дэд агентуудыг дуудаж болохыг `permission.task` ашиглан удирдана. Глоб хэв маягийг уян хатан тааруулахын тулд ашигладаг.

```json title="mongolgpt.json"
{
  "agent": {
    "orchestrator": {
      "mode": "primary",
      "permission": {
        "task": {
          "*": "deny",
          "orchestrator-*": "allow",
          "code-reviewer": "ask"
        }
      }
    }
  }
}
```

`deny` гэж тохируулсан үед дэд агент нь Даалгаврын хэрэгслийн тайлбараас бүрмөсөн хасагдах тул загвар нь түүнийг дуудахыг оролдохгүй.

:::tip
Дүрмүүдийг дарааллаар нь үнэлдэг бөгөөд **хамгийн сүүлд тохирсон дүрэм ялна**. Дээрх жишээнд `orchestrator-planner` нь `*` (үгүйсгэх) ба `orchestrator-*` (зөвшөөрөх) хоёуланд нь таарч байгаа боловч `orchestrator-*` нь `*`-ийн араас ирсэн тул үр дүн нь `allow` байна.
:::

:::tip
Агентын даалгаврын зөвшөөрөл үүнийг үгүйсгэж байсан ч хэрэглэгчид `@` автоматаар гүйцээх цэсээр дамжуулан дурын дэд агентийг үргэлж дуудаж болно.
:::

### Өнгө

`color` сонголтоор төлөөлөгчийн харагдах байдлыг UI-д тохируулна уу. Энэ нь агент интерфэйс дээр хэрхэн харагдахад нөлөөлдөг.

Хүчинтэй зургаан өнцөгт өнгө (жишээ нь: `#FF5733`) эсвэл загварын өнгө ашиглана уу: `primary`, `secondary`, `accent`, `success`, `warning`, `error`, `info`.

```json title="mongolgpt.json"
{
  "agent": {
    "creative": {
      "color": "#ff6b6b"
    },
    "code-reviewer": {
      "color": "accent"
    }
  }
}
```

### Топ П

`top_p` сонголтоор хариултын олон янз байдлыг хянах. Санамсаргүй байдлыг хянах температурын өөр хувилбар.

```json title="mongolgpt.json"
{
  "agent": {
    "brainstorm": {
      "top_p": 0.9
    }
  }
}
```

Утга нь 0.0-1.0 хооронд хэлбэлздэг. Доод утга нь илүү төвлөрч, өндөр үнэ цэнэ нь олон янз байдаг.

### Нэмэлт

Таны агентийн тохиргоонд заасан бусад сонголтуудыг загвар сонголт болгон **шууд дамжуулж** үйлчилгээ үзүүлэгч рүү илгээнэ. Энэ нь үйлчилгээ үзүүлэгчийн онцлог шинж чанар, параметрүүдийг ашиглах боломжийг танд олгоно.

Жишээлбэл, OpenAI-ийн үндэслэлийн загваруудын тусламжтайгаар та үндэслэл гаргах хүчин чармайлтыг хянах боломжтой.

```json title="mongolgpt.json" {6,7}
{
  "agent": {
    "deep-thinker": {
      "description": "Нарийн төвөгтэй асуудалд өндөр түвшний эргэцүүлэл ашигладаг агент",
      "model": "openai/gpt-5",
      "reasoningEffort": "high",
      "textVerbosity": "low"
    }
  }
}
```

Эдгээр нэмэлт сонголтууд нь загвар болон үйлчилгээ үзүүлэгчийн онцлог юм. Боломжтой параметрүүдийг үйлчилгээ үзүүлэгчийнхээ бичиг баримтаас шалгана уу.

:::tip
Боломжтой загваруудын жагсаалтыг харахын тулд `mongolgpt models` гэж ажиллуулна уу.
:::

## Агент үүсгэх

Та дараах тушаалыг ашиглан шинэ агент үүсгэж болно.

```bash
mongolgpt agent create
```

Энэхүү интерактив тушаал нь:

1. Агентийг хаана хадгалахыг асуух; дэлхийн эсвэл төсөлд зориулагдсан.
2. Төлөөлөгчийн хийх ёстой зүйлийн тодорхойлолт.
3. Тохиромжтой системийн сануулга болон танигчийг үүсгэнэ.
4. Агентд ямар зөвшөөрлийг олгохыг сонгох боломжийг танд олгоно (таны сонгоогүй бүх зүйлийг үгүйсгэнэ).
5. Эцэст нь агентын тохиргоотой тэмдэглэгээний файл үүсгэнэ.

## Хэрэглэх тохиолдол

Төрөл бүрийн агентуудын нийтлэг хэрэглээний зарим тохиолдлууд энд байна.

- **Бүтээгч агент**: Бүх хэрэгслийг идэвхжүүлсэн бүрэн хөгжүүлэлтийн ажил
- **Төлөвлөгөөний агент**: Өөрчлөлт хийхгүйгээр дүн шинжилгээ хийх, төлөвлөх
- **Хяналтын агент**: Зөвхөн уншигдах хандалт болон баримт бичгийн хэрэглүүр бүхий кодыг шалгах
- **Дбаг хийх агент**: Bash болон унших хэрэгслүүдийг идэвхжүүлснээр судалгаанд анхаарлаа хандуулсан
- **Docs agent**: Файлын үйлдлээр баримт бичгийг бичих боловч системийн команд байхгүй

### Баримт бичгийн агент

```markdown title="~/.config/mongolgpt/agents/docs-writer.md"
---
description: Төслийн баримт бичгийг бичиж, шинэчилнэ
mode: subagent
permission:
  bash: deny
---

Та техникийн баримт бичгийн зохиогч. Ойлгомжтой, бүрэн баримт бичиг бэлтгэ.

Дараах зүйлд төвлөр:

- Ойлгомжтой тайлбар
- Зөв бүтэц
- Кодын жишээ
- Хэрэглэгчид ээлтэй хэллэг
```

```markdown title="~/.config/mongolgpt/agents/security-auditor.md"
- Нэвтрэлт болон эрхийн хяналтын алдаа

- Хамаарлын эмзэг байдал
- Тохиргооны аюулгүй байдлын асуудал
```
