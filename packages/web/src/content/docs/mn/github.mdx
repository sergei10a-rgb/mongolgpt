---

MongolGPT нь таны GitHub ажлын урсгалтай нэгддэг. Сэтгэгдэл бичихдээ `/mongolgpt` гэж дурдвал MongolGPT таны GitHub Actions гүйгч доторх даалгавруудыг гүйцэтгэх болно.

---

## Онцлогууд

- **Гурвалсан асуудлууд**: MongolGPT-аас асуудлыг судалж, танд тайлбарлахыг хүс.
- **Засаж хэрэгжүүлэх**: Асуудлыг засах эсвэл функцийг хэрэгжүүлэхийг MongolGPT-аас хүс. Мөн энэ нь шинэ салбарт ажиллах бөгөөд бүх өөрчлөлтийн хамт PR илгээнэ.
- **Аюулгүй**: MongolGPT таны GitHub-н гүйгч дотор гүйдэг.

---

## Суурилуулалт

GitHub репо дахь төсөлд дараах тушаалыг ажиллуулна уу:

```bash
mongolgpt github install
```

Энэ нь таныг GitHub аппликейшнийг суулгах, ажлын урсгалыг бий болгох, нууцыг тохируулахад туслах болно.

---

### Гараар тохируулах

Эсвэл та гараар тохируулж болно.

1. **GitHub програмыг суулгана уу**

   [**github.com/apps/mongolgpt-agent**](https://github.com/apps/mongolgpt-agent) руу оч. Үүнийг зорилтот репозитор дээр суулгасан эсэхийг шалгаарай.

2. **Ажлын урсгалыг нэмэх**

   Дараах ажлын урсгалын файлыг репо дээрээ `.github/workflows/mongolgpt.yml` руу нэмнэ үү. Тохиромжтой `model` болон шаардлагатай API товчлууруудыг `env`-д тохируулахаа мартуузай.

   ```yml title=".github/workflows/mongolgpt.yml" {24,26}
   name: mongolgpt

   on:
     issue_comment:
       types: [created]
     pull_request_review_comment:
       types: [created]

   jobs:
     mongolgpt:
       if: |
         contains(github.event.comment.body, '/mongolgpt')
       runs-on: ubuntu-latest
       permissions:
         id-token: write
       steps:
          - name: Checkout repository
            uses: actions/checkout@v6
            with:
              fetch-depth: 1
              persist-credentials: false

          - name: Run MongolGPT
           uses: sergei10a-rgb/mongolgpt/github@latest
           env:
             ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
           with:
             model: anthropic/claude-sonnet-4-20250514
             # share: true
             # github_token: xxxx
   ```

3. ** API түлхүүрүүдийг нууцаар хадгалах**

   Байгууллага эсвэл төслийн **тохиргоо** дотроос зүүн талд байгаа **Нууц ба хувьсагч**-г өргөтгөж, **Үйлдэл**-ийг сонгоно уу. Мөн шаардлагатай API товчлууруудыг нэмнэ үү.

---

## Тохиргоо

- `model`: MongolGPT-тай ашиглах загвар. `provider/model` форматыг авна. Энэ **шаардлагатай**.
- `agent`: Ашиглах агент. Үндсэн төлөөлөгч байх ёстой. Тохиргооноос `default_agent` эсвэл олдохгүй бол `"build"` руу буцна.
- `share`: MongolGPT сессийг хуваалцах эсэх. Нийтийн агуулахын хувьд өгөгдмөл нь **үнэн**.
- `prompt`: Өгөгдмөл зан төлөвийг хүчингүй болгох нэмэлт захиалгат сануулга. MongolGPT хүсэлтийг хэрхэн боловсруулахыг тохируулахын тулд үүнийг ашиглана уу.
- `token`: Сэтгэгдэл бичих, өөрчлөлт хийх, татах хүсэлтийг нээх зэрэг үйлдлүүдийг гүйцэтгэх нэмэлт GitHub хандалтын токен. Анхдагчаар MongolGPT нь MongolGPT GitHub Апп-аас суулгах хандалтын тэмдгийг ашигладаг тул амлалт, сэтгэгдэл, татах хүсэлтүүд апп-аас ирсэн мэт харагдана.

  Эсвэл та GitHub Action runner-ийн [суулгасан `GITHUB_TOKEN`](https://docs.github.com/en/actions/tutorials/authenticate-with-github_token)-г MongolGPT GitHub програм суулгахгүйгээр ашиглаж болно. Ажлын урсгалдаа шаардлагатай зөвшөөрлийг олгохоо мартуузай:

  ```yaml
  permissions:
    id-token: write
    contents: write
    pull-requests: write
    issues: write
  ```

  Хэрэв хүсвэл [хувийн хандалтын токенууд](https://docs.github.com/en/authentication/keeping-your-account-and-data-secure/managing-your-personal-access-tokens)(PAT) ашиглаж болно.

## Дэмжигдсэн үйл явдал

MongolGPT дараах GitHub үйл явдлуудаар өдөөгдөж болно:

| Үйл явдлын төрөл              | өдөөгдсөн                                                                                                                                                      | Дэлгэрэнгүй                                                                                                                                                                        |
| ----------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `issue_comment`               | Асуудлын талаар сэтгэгдэл бичих эсвэл PR                                                                                                                       | Сэтгэгдэл бичихдээ `/mongolgpt` гэж бичээрэй. MongolGPT нь контекстийг уншиж, салбар үүсгэх, PR нээх эсвэл хариу бичих боломжтой.                                      |
| `pull_request_review_comment` | PR                                                                                                                                                             | доторх тодорхой кодын мөрөнд тайлбар бичнэ үү Кодтой танилцахдаа `/mongolgpt` гэж тэмдэглэ. MongolGPT нь файлын зам, мөрийн дугаар, ялгаатай контекстийг хүлээн авдаг. |
| `issues`                      | Асуудлыг нээсэн эсвэл зассан                                                                                                                                   | Асуудал үүсгэх эсвэл өөрчлөх үед автоматаар MongolGPT-г идэвхжүүлнэ. `prompt` оруулах шаардлагатай.                                                                                |
| `pull_request`                | PR нээгдсэн эсвэл шинэчлэгдсэн                                                                                                                                 | PR-г нээх, синхрончлох эсвэл дахин нээх үед автоматаар MongolGPT-г идэвхжүүлнэ. Автомат үнэлгээ хийхэд хэрэгтэй.                                                                   |
| `schedule`                    | Крон дээр суурилсан хуваарь                                                                                                                                    | MongolGPT хуваарийн дагуу ажиллуул. `prompt` оруулах шаардлагатай. Гаралт нь лог болон PR-д ордог (тайлбар бичих асуудал байхгүй).                                                 |
| `workflow_dispatch`           | GitHub UI -аас гар аргаар гох Үйлдлүүд табаар дамжуулан хүсэлтээр MongolGPT-г идэвхжүүлнэ үү. `prompt` оруулах шаардлагатай. Гаралт нь лог болон PR руу ордог. |                                                                                                                                                                                    |

### Хуваарийн жишээ

Автоматжуулсан ажлуудыг гүйцэтгэхийн тулд MongolGPT-г хуваарийн дагуу ажиллуулна уу:

```yaml title=".github/workflows/mongolgpt-scheduled.yml"
name: Scheduled MongolGPT Task

on:
  schedule:
    - cron: "0 9 * * 1" # Every Monday at 9am UTC

jobs:
  mongolgpt:
    runs-on: ubuntu-latest
    permissions:
      id-token: write
      contents: write
      pull-requests: write
      issues: write
    steps:
      - name: Checkout repository
        uses: actions/checkout@v6
        with:
          persist-credentials: false

      - name: Run MongolGPT
        uses: sergei10a-rgb/mongolgpt/github@latest
        env:
          ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
        with:
          model: anthropic/claude-sonnet-4-20250514
          prompt: |
            Review the codebase for any TODO comments and create a summary.
            If you find issues worth addressing, open an issue to track them.
```

Хуваарьт үйл явдлын хувьд заавар авах тайлбар байхгүй тул `prompt` оруулах **шаардлагатай**. Хуваарьт ажлын урсгалууд нь хэрэглэгчийн контекст зөвшөөрөлгүйгээр ажилладаг тул хэрэв та MongolGPT салбар эсвэл PR үүсгэхийг хүлээж байгаа бол ажлын урсгал нь `contents: write` болон `pull-requests: write` өгөх ёстой.

### Татах хүсэлтийн жишээ

PR-г нээх эсвэл шинэчлэх үед автоматаар хянана:

```yaml title=".github/workflows/mongolgpt-review.yml"
name: mongolgpt-review

on:
  pull_request:
    types: [opened, synchronize, reopened, ready_for_review]

jobs:
  review:
    runs-on: ubuntu-latest
    permissions:
      id-token: write
      contents: read
      pull-requests: read
      issues: read
    steps:
      - uses: actions/checkout@v6
        with:
          persist-credentials: false
      - uses: sergei10a-rgb/mongolgpt/github@latest
        env:
          ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
        with:
          model: anthropic/claude-sonnet-4-20250514
          use_github_token: true
          prompt: |
            Review this pull request:
            - Check for code quality issues
            - Look for potential bugs
            - Suggest improvements
```

`pull_request` үйл явдлуудын хувьд, хэрэв `prompt` өгөөгүй бол MongolGPT татах хүсэлтийг анхдагчаар хянана.

### Асуудлыг ангилах жишээ

Шинэ асуудлуудыг автоматаар шалгана. Энэ жишээ нь спамыг багасгахын тулд 30-аас дээш хоногтой бүртгэлүүдийг шүүж байна:

```yaml title=".github/workflows/mongolgpt-triage.yml"
name: Issue Triage

on:
  issues:
    types: [opened]

jobs:
  triage:
    runs-on: ubuntu-latest
    permissions:
      id-token: write
      contents: write
      pull-requests: write
      issues: write
    steps:
      - name: Check account age
        id: check
        uses: actions/github-script@v7
        with:
          script: |
            const user = await github.rest.users.getByUsername({
              username: context.payload.issue.user.login
            });
            const created = new Date(user.data.created_at);
            const days = (Date.now() - created) / (1000 * 60 * 60 * 24);
            return days >= 30;
          result-encoding: string

      - uses: actions/checkout@v6
        if: steps.check.outputs.result == 'true'
        with:
          persist-credentials: false

      - uses: sergei10a-rgb/mongolgpt/github@latest
        if: steps.check.outputs.result == 'true'
        env:
          ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
        with:
          model: anthropic/claude-sonnet-4-20250514
          prompt: |
            Review this issue. If there's a clear fix or relevant docs:
            - Provide documentation links
            - Add error handling guidance for code examples
            Otherwise, do not comment.
```

`issues` үйл явдлын хувьд заавар авах тайлбар байхгүй тул `prompt` оролт **заавал ** шаардлагатай.

## Тусгай сануулга

Өөрийн ажлын урсгалд MongolGPT-н зан төлөвийг өөрчлөхийн тулд өгөгдмөл сануулгыг хүчингүй болго.

```yaml title=".github/workflows/mongolgpt.yml"
- uses: sergei10a-rgb/mongolgpt/github@latest
  with:
    model: anthropic/claude-sonnet-4-5
    prompt: |
      Review this pull request:
      - Check for code quality issues
      - Look for potential bugs
      - Suggest improvements
```

Энэ нь таны төсөлтэй холбоотой тодорхой хянан шалгах шалгуур, кодчиллын стандарт эсвэл анхаарлаа төвлөрүүлэхэд тустай.

## Жишээ

GitHub-д MongolGPT-г хэрхэн ашиглаж болох зарим жишээ энд байна.

- **Асуудлыг тайлбарлана уу**

  Энэ сэтгэгдлийг GitHub дугаарт нэмнэ үү.

  ```
  /mongolgpt explain this issue
  ```

  MongolGPT бүх сэтгэгдлийг оруулаад сэдвийг бүхэлд нь уншиж, тодорхой тайлбартайгаар хариулах болно.

- **Асуудлыг засах**

  GitHub дугаарт:

  ```
  /mongolgpt fix this
  ```

  Мөн MongolGPT нь шинэ салбар үүсгэж, өөрчлөлтийг хэрэгжүүлж, өөрчлөлттэй хамт PR нээх болно.

- **PR-г хянаж, өөрчлөлт оруулах**

  Дараах сэтгэгдэлийг GitHub PR дээр үлдээнэ үү.

  ```
  Delete the attachment from S3 when the note is removed /mongolgpt
  ```

  MongolGPT нь хүссэн өөрчлөлтийг хэрэгжүүлж, ижил PR-д хариуцна.

- **Тусгай кодын мөрүүдийг шалгах**

  PR-н "Файл" таб дахь кодын мөрөнд шууд сэтгэгдэл үлдээнэ үү. MongolGPT нь автоматаар файл, мөрийн дугаар, ялгавартай контекстийг илрүүлж, тодорхой хариулт өгдөг.

  ```
  [Comment on specific lines in Files tab]
  /mongolgpt add error handling here
  ```

  Тодорхой мөрөнд сэтгэгдэл бичихдээ MongolGPT хүлээн авна:
  - Яг файлыг шалгаж байна
  - Кодын тусгай мөрүүд
  - Эргэн тойрон дахь ялгаатай нөхцөл байдал
  - Шугамын дугаарын мэдээлэл

  Энэ нь файлын зам эсвэл мөрийн дугаарыг гараар зааж өгөх шаардлагагүйгээр илүү зорилтот хүсэлт гаргах боломжийг олгодог.
