MongolGPT нь **75+ LLM үйлчилгээ үзүүлэгчдийг** дэмжихийн тулд [AI SDK](https://ai-sdk.dev/) болон [Models.dev](https://models.dev) ашигладаг бөгөөд орон нутгийн загваруудыг ажиллуулахыг дэмждэг.

Үйлчилгээ үзүүлэгч нэмэхийн тулд та:

1. `/connect` командыг ашиглан үйлчилгээ үзүүлэгчийн API товчлууруудыг нэмнэ үү.
2. Өөрийн MongolGPT тохиргоонд үйлчилгээ үзүүлэгчийг тохируулна уу.

---

### Итгэмжлэл

Та үйлчилгээ үзүүлэгчийн API товчлууруудыг `/connect` тушаалаар нэмэхэд тэдгээр нь хадгалагдана.
`~/.local/share/mongolgpt/auth.json`-д.

---

### Тохиргоо

Та өөрийн MongolGPT хэсэгт байгаа `provider` хэсгээс үйлчилгээ үзүүлэгчдийг өөрчлөх боломжтой.
config.

---

#### Үндсэн URL

Та `baseURL` сонголтыг тохируулснаар ямар ч үйлчилгээ үзүүлэгчийн URL суурийг өөрчлөх боломжтой. Энэ нь прокси үйлчилгээ эсвэл захиалгат төгсгөлийн цэгүүдийг ашиглахад хэрэгтэй.

```json title="mongolgpt.json" {6}
{
  "$schema": "https://mongolgpt.duckdns.org/config.json",
  "provider": {
    "anthropic": {
      "options": {
        "baseURL": "https://api.anthropic.com/v1"
      }
    }
  }
}
```

---

#### Загваруудыг нууж байна

Та `blacklist` сонголтыг ашиглан үйлчилгээ үзүүлэгчийн `/models` сонгогчоос тодорхой загваруудыг нууж болно. Энэ нь үйлчилгээ үзүүлэгч таны ашиглахыг эсвэл сонгохыг хүсэхгүй байгаа загваруудыг ил гаргахад хэрэгтэй.

```json title="mongolgpt.json" {6}
{
  "$schema": "https://mongolgpt.duckdns.org/config.json",
  "provider": {
    "anthropic": {
      "blacklist": ["claude-opus-4-20250514"]
    }
  }
}
```

Урвуу `whitelist` сонголт нь жагсаасан загваруудаас бусад бүх загварыг нуудаг.

```json title="mongolgpt.json" {6}
{
  "$schema": "https://mongolgpt.duckdns.org/config.json",
  "provider": {
    "anthropic": {
      "whitelist": ["claude-sonnet-4-20250514"]
    }
  }
}
```

Хоёр сонголт хоёулаа `/models` сонгогч дээр харуулсан ижил ID-н загваруудын массивыг авдаг.

- `blacklist` жагсаасан загваруудыг сонгогчоос хасна.
- `whitelist` зөвхөн жагсаасан загваруудыг хадгалж үлдсэнийг нь нуудаг.
- Та тэдгээрийг нэгтгэж болно: `whitelist` багцыг нарийсгаж, дараа нь `blacklist` түүнээс оруулгуудыг устгана.

### Amazon Bedrock

Amazon Bedrock-г MongolGPT-тэй ашиглахын тулд:

1. Amazon Bedrock консол дахь **Загварын каталог** руу орж хүсэлт гаргана уу.
   хүссэн загварууд руугаа нэвтрэх.

   :::зөвлөгөө
   Та Amazon Bedrock хаягаар хүссэн загвартаа хандах эрхтэй байх шаардлагатай.
   :::

2. Дараах аргуудын аль нэгийг ашиглан **Баталгаажуулалтыг тохируулах**:

   ***

   #### Хүрээлэн буй орчны хувьсагчид (хурдан эхлүүлэх)

   mongolgpt ажиллуулж байх үед эдгээр орчны хувьсагчийн аль нэгийг тохируулна уу:

   ```bash
   # Option 1: Using AWS access keys
   AWS_ACCESS_KEY_ID=XXX AWS_SECRET_ACCESS_KEY=YYY mongolgpt

   # Option 2: Using named AWS profile
   AWS_PROFILE=my-profile mongolgpt

   # Option 3: Using Bedrock bearer token
   AWS_BEARER_TOKEN_BEDROCK=XXX mongolgpt
   ```

   Эсвэл тэдгээрийг bash профайлдаа нэмнэ үү:

   ```bash title="~/.bash_profile"
   export AWS_PROFILE=my-dev-profile
   export AWS_REGION=us-east-1
   ```

   ***

   #### Тохируулгын файл (санал болгосон)

   Төсөлд зориулсан эсвэл байнгын тохиргооны хувьд `mongolgpt.json` ашиглана уу:

   ```json title="mongolgpt.json"
   {
     "$schema": "https://mongolgpt.duckdns.org/config.json",
     "provider": {
       "amazon-bedrock": {
         "options": {
           "region": "us-east-1",
           "profile": "my-aws-profile"
         }
       }
     }
   }
   ```

   ** Боломжтой сонголтууд:**
   - `region` - AWS бүс (жишээ нь: `us-east-1`, `eu-west-1`)
   - `profile` - AWS нэртэй профайлыг `~/.aws/credentials`
- `endpoint` - VPC төгсгөлийн цэгүүдийн захиалгат төгсгөлийн цэг URL (ерөнхий `baseURL` сонголтын бусад нэр)

   :::зөвлөгөө
   Тохируулгын файлын сонголтууд нь орчны хувьсагчдаас давуу эрхтэй.
   :::

   ***

   #### Нарийвчилсан: VPC Төгсгөлийн цэгүүд

   Хэрэв та Berock-д VPC төгсгөлийн цэгүүдийг ашиглаж байгаа бол:

   ```json title="mongolgpt.json"
   {
     "$schema": "https://mongolgpt.duckdns.org/config.json",
     "provider": {
       "amazon-bedrock": {
         "options": {
           "region": "us-east-1",
           "profile": "production",
           "endpoint": "https://bedrock-runtime.us-east-1.vpce-xxxxx.amazonaws.com"
         }
       }
     }
   }
   ```

   ::: тэмдэглэл
   `endpoint` сонголт нь AWS- тусгай нэр томъёог ашиглан ерөнхий `baseURL` сонголтын нэр юм. Хэрэв `endpoint` болон `baseURL` хоёуланг нь зааж өгсөн бол `endpoint` давуу эрхтэй болно.
   :::

   ***

   #### Баталгаажуулах аргууд
   - **`AWS_ACCESS_KEY_ID` / `AWS_SECRET_ACCESS_KEY`**: IAM хэрэглэгч үүсгэж, AWS Консол дээр хандалтын түлхүүрүүдийг үүсгэнэ үү.
   - **`AWS_PROFILE`**: `~/.aws/credentials`-аас нэрлэсэн профайлыг ашиглана уу. Эхлээд `aws configure --profile my-profile` эсвэл `aws sso login` тохируулна уу.
   - **`AWS_BEARER_TOKEN_BEDROCK`**: Amazon Bedrock консолоос урт хугацааны API түлхүүрүүдийг үүсгэх
   - **`AWS_WEB_IDENTITY_TOKEN_FILE` / `AWS_ROLE_ARN`**: EKS IRSA (IAM Үйлчилгээний дансны үүрэг) эсвэл OIDC холбоо бүхий Kubernetes-ийн бусад орчинд. Үйлчилгээний дансны тэмдэглэгээг ашиглах үед эдгээр орчны хувьсагчдыг Кубернетес автоматаар оруулдаг.

   ***

   #### Баталгаажуулалтын давуу эрх

   Amazon Bedrock дараах баталгаажуулалтын тэргүүлэх чиглэлийг ашигладаг:
   1. **Bearer Token** - `AWS_BEARER_TOKEN_BEDROCK` орчны хувьсагч эсвэл `/connect` командын токен
   2. **AWS Credential Chain** - Профайл, хандалтын түлхүүр, хуваалцсан итгэмжлэл, IAM үүрэг, Вэб таних тэмдэг (EKS IRSA), жишээ мета өгөгдөл

   ::: тэмдэглэл
   Тэмдэглэгээний токеныг тохируулах үед (`/connect` эсвэл `AWS_BEARER_TOKEN_BEDROCK`-ээр) энэ нь тохируулсан профайл зэрэг бүх AWS итгэмжлэлийн аргуудаас давуу эрхтэй болно.
   :::

3. Хүссэн загвараа сонгохын тулд `/models` командыг ажиллуулна.

   ```txt
   /models
   ```

:::note
Захиалгат дүгнэлт гаргахын тулд түлхүүр дэх загвар болон үйлчилгээ үзүүлэгчийн нэрийг ашиглан `id` шинж чанарыг arn болгон тохируулна уу. Энэ нь зөв кэшийг баталгаажуулдаг.
:::

```json title="mongolgpt.json"
{
  "$schema": "https://mongolgpt.duckdns.org/config.json",
  "provider": {
    "amazon-bedrock": {
      // ...
      "models": {
        "anthropic-claude-sonnet-4.5": {
          "id": "arn:aws:bedrock:us-east-1:xxx:application-inference-profile/yyy"
        }
      }
    }
  }
}
```

### Атомын чат

Та mongolgpt-г [Atomic Chat](https://atomic.chat)-ээр дамжуулан локал загвар ашиглахаар тохируулж болно. Atomic Chat нь локал LLM-үүдийг OpenAI нийцтэй API серверийн ард ажиллуулдаг ширээний програм юм (анхдагч endpoint `http://127.0.0.1:1337/v1`).

```json title="mongolgpt.json" "atomic-chat" {5, 6, 8, 10-14}
{
  "$schema": "https://mongolgpt.duckdns.org/config.json",
  "provider": {
    "atomic-chat": {
      "npm": "@ai-sdk/openai-compatible",
      "name": "Atomic Chat (local)",
      "options": {
        "baseURL": "http://127.0.0.1:1337/v1"
      },
      "models": {
        "<your-model-id>": {
          "name": "<your-model-name>"
        }
      }
    }
  }
}
```

Энэ жишээнд:

- `atomic-chat` нь захиалгат үйлчилгээ үзүүлэгч ID юм. Энэ нь таны хүссэн ямар ч мөр байж болно.
-
pm` нь энэ үйлчилгээ үзүүлэгчийн ашиглах багцыг тодорхойлдог. Энд `@ai-sdk/openai-compatible` нь ямар ч OpenAI нийцтэй API-д ашиглагддаг.
-
ame` нь UI дахь үйлчилгээ үзүүлэгчийн дэлгэцийн нэр юм.
- `options.baseURL` нь дотоод серверийн төгсгөлийн цэг юм. Atomic Chat-н тохиргоонд тохируулан хост болон портоо өөрчил.
- `models` нь загвар ID-уудын дэлгэцийн нэрийн газрын зураг юм. ID тус бүр нь `GET /v1/models`-ийн буцаасан `id`-тэй таарч байх ёстой - одоо Atomic Chat-д ачаалагдсан id-уудыг жагсаахын тулд `curl http://127.0.0.1:1337/v1/models` ажиллуулна уу.

:::tip
Хэрэгслийн дуудлагууд сайн ажиллахгүй байвал хэрэгсэл дуудах хүчтэй дэмжлэгтэй ачаалагдсан загварыг сонгоорой (жишээ нь Qwen-Coder эсвэл DeepSeek-Coder хувилбар).
:::

   ::: тэмдэглэл

   ```txt
   /connect
   ```

4. API түлхүүрээ оруулна уу.

   ```txt
   │
   └ enter
   ```

5. Өөрийн нөөцийн нэрийг орчны хувьсагч болгон тохируулна уу:



4. API түлхүүрээ оруулна уу.

MongolGPT нь баталгаажуулалтын хоёр аргыг дэмждэг:

- **OAuth (Санал болгосон)** — DigitalOcean бүртгэлдээ нэвтэрнэ үү; MongolGPT нь таны DigitalOcean API жетоныг шууд дүгнэлт хийхэд ашиглаж, таны Inference Routers-ийг илрүүлдэг.
- **Загвар нэвтрэх түлхүүр** — DigitalOcean консолоос байгаа түлхүүрийг буулгана уу.

#### OAuth (Санал болгосон)

1. `/connect` командыг ажиллуулаад **DigitalOcean** гэж хайна уу.

   ```txt

   ```

2. **DigitalOcean-р нэвтрэх**-г сонгоно уу.

   ```txt
   ┌ Select auth method
   │
   │ Login with DigitalOcean
   │ Paste Model Access Key
   └
   ```

3. Таны хөтөч MongolGPT-д зөвшөөрөл олгохоор нээгдэнэ. Нэвтэрч, зөвшөөрнө үү.

   ::: тэмдэглэл
   MongolGPT хүсэлт `genai:read` болон `inference:query` OAuth хамрах хүрээ. Таны DigitalOcean API токеныг дүгнэлт хийхэд шууд ашигладаг - тусдаа Загварын хандалтын түлхүүр үүсгэгдээгүй.
   :::

   ::: тэмдэглэл
   Дүгнэлт чиглүүлэгчид зөвхөн OAuth-ийн дараа загвар сонгогч дээр гарч ирнэ. Загварын хандалтын түлхүүрийг гараар буулгах нь чиглүүлэгчийг илрүүлэхгүй.
   :::

4. `/models` командыг ажиллуулна. Таны дүгнэлт чиглүүлэгчид загвар сонгоход `router:` форматаар харагдана.

   ```txt

   ```

5. Шинээр үүсгэсэн Дүгнэлт чиглүүлэгчийг авахын тулд `/connect`-г дахин ажиллуулаад **DigitalOcean**-г дахин сонгоно уу.

   ```txt
   │
   │
   └ enter
   ```

   ::: тэмдэглэл
   Дүгнэлт чиглүүлэгчийг энэ аргаар автоматаар илрүүлдэггүй. Тэдгээрийг загвар сонгогч дээр харуулахын тулд оронд нь OAuth-ээр нэвтэрнэ үү.

##### OAuth Өөрөө байршуулсан тохиолдлуудад

Oauth-г өөрийн эзэмшдэг жишээнд ажиллуулахын тулд та үүсгэх хэрэгтэй
бүхий шинэ програм (Тохиргоо → Програмууд).
буцаан залгах URL `http://127.0.0.1:8080/callback` болон дараах хамрах хүрээ:

- api (Таны өмнөөс API руу хандана уу)
- read_user (Хувийн мэдээллээ уншина уу)
- read_repository (Repository руу зөвхөн унших боломжтой)

Дараа нь ID програмыг орчны хувьсагч болгон харуулна:

```bash
export GITLAB_OAUTH_CLIENT_ID=your_application_id_here
```

Дэлгэрэнгүй баримт бичгийг MongolGPT GitLab auth adapter нүүр хуудаснаас авна уу.

##### GitLab Duo Agent Platform (DAP) Ажлын урсгалын загварууд

DAP ажлын урсгалын загварууд нь хэрэгслийн дуудлагыг чиглүүлэх өөр гүйцэтгэх замыг өгдөг
Стандарт агент чатын оронд GitLab-ийн Duo Workflow үйлчилгээ (DWS)-ээр дамжуулан.
`duo-workflow-*` загварыг сонгоход MongolGPT:

1. Өөрийн GitLab нэрийн талбараас боломжтой загваруудыг олж мэдээрэй
2. Олон загвар байгаа бол сонголт сонгогчийг танилцуулна уу
3. Дараа нь хурдан эхлүүлэхийн тулд сонгосон загвараа дискэнд хадгал
4. Хэрэгслийн гүйцэтгэлийн хүсэлтийг MongolGPT-н зөвшөөрөлтэй хэрэгслийн системээр дамжуулан чиглүүлэх

Боломжтой DAP ажлын урсгалын загварууд нь `duo-workflow-*` нэрлэх конвенцийг дагаж мөрддөг.
таны GitLab инстансаас динамикаар нээсэн.

#### Нэмэлт тохиргоо

Хэрэв та Helicone-ээс mongolgpt-р автоматаар тохируулагдаагүй функц эсвэл загварыг олж харвал та үүнийг хүссэн үедээ өөрөө тохируулах боломжтой.

Энд [Helicone загварын лавлах](https://helicone.ai/models), энэ нь нэмэхийг хүссэн загварынхаа ID-г авахын тулд танд хэрэгтэй болно.

```jsonc title="~/.config/mongolgpt/mongolgpt.jsonc"
{
  "$schema": "https://mongolgpt.duckdns.org/config.json",
  "provider": {
    "helicone": {
      "npm": "@ai-sdk/openai-compatible",
      "name": "Helicone",
      "options": {
        "baseURL": "https://ai-gateway.helicone.ai",
      },
      "models": {
        "gpt-4o": {
          // Model ID (from Helicone's model directory page)
          "name": "GPT-4o", // Your own custom name for the model
        },
        "claude-sonnet-4-20250514": {
          "name": "Claude Sonnet 4",
        },
      },
    },
  },
}
```

#### Тусгай толгой

Helicone нь кэш хийх, хэрэглэгчийг хянах, сессийг удирдах зэрэг функцүүдийн захиалгат толгой хэсгийг дэмждэг. Тэдгээрийг `options.headers` ашиглан үйлчилгээ үзүүлэгчийн тохиргоонд нэмнэ үү:

```jsonc title="~/.config/mongolgpt/mongolgpt.jsonc"
{
  "$schema": "https://mongolgpt.duckdns.org/config.json",
  "provider": {
    "helicone": {
      "npm": "@ai-sdk/openai-compatible",
      "name": "Helicone",
      "options": {
        "baseURL": "https://ai-gateway.helicone.ai",
        "headers": {
          "Helicone-Cache-Enabled": "true",
          "Helicone-User-Id": "mongolgpt",
        },
      },
    },
  },
}
```

##### Сеанс хянах

Helicone-ийн [Sessions](https://docs.helicone.ai/features/sessions) функц нь танд холбогдох LLM хүсэлтүүдийг хамтад нь бүлэглэх боломжийг олгодог. MongolGPT яриа бүрийг Helicone-д сесс болгон автоматаар бүртгэхийн тулд local Helicone session plugin-ийг ашиглана уу.

```bash
# Dedicated MongolGPT npm plugin is not published yet.
# Use a local plugin file or import an adapter with:
mongolgpt compat import plan ./plugins/helicone-session.ts
```

Үүнийг өөрийн тохиргоонд нэмнэ үү.

```json title="mongolgpt.json"
{
  "plugin": ["./plugins/helicone-session.ts"]
}
```

Энэхүү залгаас нь таны хүсэлтэд `Helicone-Session-Id` болон `Helicone-Session-Name` толгойг оруулдаг. Helicone's Sessions хуудсанд та MongolGPT харилцан яриа бүрийг тусдаа сесс хэлбэрээр жагсаасан харагдана.

##### Энгийн Helicone толгой

| Толгой | Тодорхойлолт |
| -------------------------- | ------------------------------------------------------------- |
| `Helicone-Cache-Enabled` | Хариултын кэшийг идэвхжүүлэх (`true`/`false`) |
| `Helicone-User-Id` | Хэрэглэгчийн хэмжүүрийг хянах |
| `Helicone-Property-[Name]` | Захиалгат шинж чанаруудыг нэмэх (жишээ нь: `Helicone-Property-Environment`) |
| `Helicone-Prompt-Id` | Хүсэлтийг шуурхай хувилбаруудтай холбох |

Боломжтой бүх толгой хэсгийг [Helicone толгойн лавлах](https://docs.helicone.ai/helicone-headers/header-directory) харна уу.

Та Ollama-аар дамжуулан орон нутгийн загваруудыг ашиглахын тулд mongolgpt-г тохируулах боломжтой.

:::tip
Оллама өөрийгөө автоматаар MongolGPT-д тохируулж болно. Дэлгэрэнгүйг [Оллама нэгтгэх баримтууд](https://docs.ollama.com/integrations/mongolgpt) харна уу.
:::

```json title="mongolgpt.json" "ollama" {5, 6, 8, 10-14}
{
  "$schema": "https://mongolgpt.duckdns.org/config.json",
  "provider": {
    "ollama": {
      "npm": "@ai-sdk/openai-compatible",
      "name": "Ollama (local)",
      "options": {
        "baseURL": "http://localhost:11434/v1"
      },
      "models": {
        "llama2": {
          "name": "Llama 2"
        }
      }
    }
  }
}
```

Энэ жишээнд:

- `ollama` нь захиалгат үйлчилгээ үзүүлэгч ID юм. Энэ нь таны хүссэн ямар ч мөр байж болно.
-
pm` нь энэ үйлчилгээ үзүүлэгчийн ашиглах багцыг тодорхойлдог. Энд `@ai-sdk/openai-compatible` нь ямар ч OpenAI нийцтэй API-д ашиглагддаг.
-
ame` нь UI дахь үйлчилгээ үзүүлэгчийн дэлгэцийн нэр юм.
- `options.baseURL` нь дотоод серверийн төгсгөлийн цэг юм.
- `models` нь загвар ID-уудын тохиргооны газрын зураг юм. Загварын нэр нь загвар сонгох жагсаалтад харагдах болно.

:::tip
Хэрэгслийн дуудлагууд ажиллахгүй байвал Ollama-д
um_ctx`-г нэмэгдүүлж үзнэ үү. Ойролцоогоор 16k - 32k эхэлнэ.
:::

### Цасан ширхгийн Cortex

[Цасан ширхгийн Cortex](https://docs.snowflake.com/en/user-guide/snowflake-cortex/cortex-rest-api) нь OpenAI нийцтэй API-ээр дамжуулан хилийн загварт (Claude, OpenAI GPT-5 болон бусад) хандах боломжийг олгоно. Бүх дүгнэлт нь Цасан ширхгийн периметрийн хүрээнд явагдах ба Цасан ширхгийн кредитээр тооцогдоно. Загвар бүрийн үнийг [Цасан ширхгийн үйлчилгээний хэрэглээний хүснэгт](https://www.snowflake.com/legal-files/CreditConsumptionTable.pdf) харна уу.

Танд Snowflake данс байхгүй юу? [Үнэгүй туршилтанд бүртгүүлнэ үү](https://signup.snowflake.com/?utm_source=mongolgpt&utm_medium=docs&utm_campaign=cortex-provider).

mongolgpt-ийн кодлох, файл засварлах, командуудыг ажиллуулах үндсэн ажлын урсгал нь багажны дуудлагад тулгуурладаг. Зөвхөн Snowflake Cortex доторх Claude болон OpenAI гэр бүлүүд үүнийг дэмждэг. Үйлчилгээ үзүүлэгч нь үндсэн ажлын урсгалыг дэмжихийн тулд тэдгээр гэр бүлээр хязгаарлагддаг.

MongolGPT нь баталгаажуулалтын хоёр аргыг дэмждэг:

- **Хөтөч OAuth (Санал болгосон)** — өөрийн IdP/SSO-ээр нэвтэрнэ үү; удирдах нууц байхгүй, токенууд автоматаар шинэчлэгддэг.
- **Гараар зөөгч токен** — Snowflake консолоос PAT эсвэл JWT-г буулгана уу.

#### Хөтөч OAuth (Санал болгосон)

1. `/connect` командыг ажиллуулаад **Snowflake Cortex** гэж хайна.

   ```txt
   /connect
   ```

2. **Snowflake (External Browser) ашиглан нэвтрэх**-г сонгоно.

   ```txt
   ┌ Select auth method
   │
   │ Login with Snowflake (External Browser)
   │ Paste PAT or bearer token manually
   └
   ```

3. Сануулахад [данс танигч](https://docs.snowflake.com/en/user-guide/admin-account-identifier)-г оруулна уу, жишээ нь `myorg-myaccount` эсвэл `xy12345.us-east-1`.

   ```txt
   ┌ Snowflake Account Identifier
   │
   │
   └ enter
   ```

4. Сонголтоор хичээлийг хамрахын тулд Цасан ширхгийн дүрийг оруулна уу (жишээ нь: `SYSADMIN`). Өгөгдмөл дүрээ ашиглахын тулд хоосон орхино уу.

5. Нээгдэх вэб хөтчөөр нэвтэрч дуусгана уу. MongolGPT OAuth буцаан дуудлагыг автоматаар барьж, токеныг хадгалдаг - хуулах/буулгах шаардлагагүй.

6. Загвар сонгохын тулд `/models` командыг ажиллуулна.

   ```txt
   /models
   ```

:::note
OAuth хөтөч нь Snowflake-ийн суурилуулсан `SNOWFLAKE$LOCAL_APPLICATION` аюулгүй байдлын интеграцийг ([docs](https://docs.snowflake.com/en/user-guide/oauth-local-applications)) ашигладаг бөгөөд энэ нь бүх бүртгэлд нэвтэрч байна. Бүртгэлдээ байгаа эсэхийг шалгахын тулд:

```sql
SHOW SECURITY INTEGRATIONS LIKE 'SNOWFLAKE$LOCAL_APPLICATION';
```

Үр дүн нь хоосон байвал доорх **Гараар зөөвөрлөгч токен** аргыг ашиглана уу.
:::

#### Гарын авлагын тэмдэг

Хэрэв та токеныг шууд буулгахыг хүсч байвал эсвэл таны бүртгэлд `SNOWFLAKE$LOCAL_APPLICATION` байхгүй бол:

1. Snowflake дансандаа [Програмчлалын хандалтын токен (PAT)](https://docs.snowflake.com/en/user-guide/programmatic-access-tokens) үүсгэнэ үү.

2. `/connect` командыг ажиллуулж, **Snowflake Cortex** гэж хайгаад **Paste PAT эсвэл зөөгч токеныг гараар** сонгоно уу.

3. Сануулахад [данс танигч](https://docs.snowflake.com/en/user-guide/admin-account-identifier)-г оруулна уу.

4. PAT-г наа.

5. Загвар сонгохын тулд `/models` командыг ажиллуулна.

   ```txt
   /models
   ```

#### Хүрээлэн буй орчны хувьсагч

CI эсвэл толгойгүй орчинд mongolgpt эхлэхээс өмнө PAT эсвэл JWT-г тохируулна уу:

```bash
export SNOWFLAKE_ACCOUNT=myorg-myaccount
export SNOWFLAKE_CORTEX_TOKEN=your-pat-or-jwt
```

:::note
`SNOWFLAKE_CORTEX_TOKEN` зөвхөн PAT эсвэл JWT-г хүлээн зөвшөөрдөг - хөтчийн OAuth урсгалыг зөвхөн `/connect`-р ашиглах боломжтой бөгөөд орчны хувьсагчаар дамжуулан тохируулах боломжгүй. `SNOWFLAKE_CORTEX_PAT` нь буцаад нийцтэй байхаар дэмжигдсэн хэвээр байна.
:::

Загварын каталогийг автоматаар өгдөг. Хамгийн багадаа `mongolgpt.json` байхад л хангалттай:

```json title="mongolgpt.json"
{
  "$schema": "https://mongolgpt.duckdns.org/config.json",
  "model": "snowflake-cortex/claude-sonnet-4-6",
  "small_model": "snowflake-cortex/claude-haiku-4-5"
}
```

### Vercel AI Гарц

Vercel AI Gateway нь нэгдсэн төгсгөлийн цэгээр дамжуулан OpenAI, Anthropic, Google, xAI болон бусад зүйлсийн загварт хандах боломжийг танд олгоно. Загваруудыг жагсаалтын үнээр ямар ч нэмэлтгүйгээр санал болгож байна.

1. [Vercel хяналтын самбар](https://vercel.com/) руу очиж, **AI Gateway** таб руу шилжиж, **API товчлуурууд** дээр дарж шинэ API түлхүүр үүсгэнэ үү.

2. `/connect` командыг ажиллуулаад **Vercel AI Gateway** гэж хайна уу.

   ```txt
   /connect
   ```

3. Vercel AI Gateway API түлхүүрээ оруулна уу.

   ```txt
   ┌ API key
   │
   │
   └ enter
   ```

4. Загвар сонгохын тулд `/models` командыг ажиллуулна.

   ```txt
   /models
   ```

Та мөн mongolgpt тохиргоогоор дамжуулан загвараа өөрчлөх боломжтой. Үйлчилгээ үзүүлэгчийн чиглүүлэлтийн дарааллыг зааж өгөх жишээ энд байна.

```json title="mongolgpt.json"
{
  "$schema": "https://mongolgpt.duckdns.org/config.json",
  "provider": {
    "vercel": {
      "models": {
        "anthropic/claude-sonnet-4": {
          "options": {
            "order": ["anthropic", "vertex"]
          }
        }
      }
    }
  }
}
```

Зарим ашигтай чиглүүлэлтийн сонголтууд:

| Сонголт | Тодорхойлолт |
| ------------------- | ---------------------------------------------------- |
| `order` | Оролдох үйлчилгээ үзүүлэгчийн дараалал |
| `only` | Тодорхой үйлчилгээ үзүүлэгчдийг хязгаарлах |
| `zeroDataRetention` | Зөвхөн өгөгдөл хадгалах бодлогогүй үйлчилгээ үзүүлэгчдийг ашиглана уу |

### xAI

Баталгаажуулах гурван арга: OAuth хөтчөөр дамжуулан SuperGrok-ийн захиалга, толгойгүй төхөөрөмжийн кодын урсгалаар дамжуулан ижил SuperGrok захиалга (VPS / SSH / Docker) эсвэл xAI консолоос төлбөр төлдөг API түлхүүр.

#### Сонголт А — SuperGrok OAuth (хөтөч рүү нэвтрэх)

1. `/connect` командыг ажиллуулаад **xAI** гэж хайна уу.

   ```txt
   /connect
   ```

2. **xAI Grok OAuth (SuperGrok Subscription)**-г сонгоно уу. MongolGPT таны хөтөч дээр xAI-ийн зөвшөөрлийн дэлгэцийг нээж, `http://127.0.0.1:56121/callback` дээр дахин залгахыг хүлээнэ.

3. `/models` командыг ажиллуулж Grok загварыг сонгоно.

   ```txt
   /models
   ```

MongolGPT OAuth хандалтын тэмдгийг автоматаар шинэчилнэ. Grok API хандалтыг багтаасан аливаа Grok эсвэл X Premium төлөвлөгөө; танд тусдаа `XAI_API_KEY` хэрэггүй.

#### Сонголт B — SuperGrok төхөөрөмжийн код (толгойгүй / алсын сервер / VPS)

Үүнийг MongolGPT хөтөч loopback redirect хүрч чадахгүй орчинд ажиллаж байх үед ашиглана уу: VPS, SSH-ээр холбогдсон алсын dev box, Docker дотор, CI гэх мэт. MongolGPT ажиллаж буй хост дээр callback порт нээгдэхгүй. Харин xAI CLI-д богино код өгч, та тэр кодыг өөр төхөөрөмжийн (зөөврийн компьютер, утас гэх мэт) хөтөчид оруулна.

1. Алсын хост дээр `/connect` командыг ажиллуулаад **xAI** гэж хайна уу.

   ```txt
   /connect
   ```

2. **xAI Grok OAuth (Толгойгүй / Алсын / VPS)**-г сонгоно уу. MongolGPT баталгаажуулах URL болон хэрэглэгчийн богино кодыг хэвлэдэг.

   ```txt
   Open https://x.ai/device on any device and enter code: ABCD-1234
   ```

3. Хөтөч (зөөврийн компьютер эсвэл утас) бүхий төхөөрөмж дээр URL-г нээж, кодыг оруулаад зөвшөөрлийн дэлгэцийг зөвшөөрнө үү. MongolGPT xAI-ийн токен төгсгөлийн цэгийг санал асуулга авч, таныг зөвшөөрсний дараа үүссэн OAuth жетонуудыг хадгална. Токен шинэчлэх нь сонголт А-тай адил ажилладаг.

#### Сонголт C — API товчлуур

1. [xAI консол](https://console.x.ai/) руу орж, бүртгэл үүсгээд API түлхүүр үүсгэнэ үү.

2. `/connect` командыг ажиллуулаад **xAI** гэж хайна уу.

   ```txt
   /connect
   ```

3. **Гараар API Түлхүүрийг** сонгоод xAI API түлхүүрээ оруулна уу.

   ```txt
   ┌ API key
   │
   │
   └ enter
   ```

4. `/models` командыг ажиллуулж _Grok Beta_ шиг загварыг сонгоно.

   ```txt
   /models
   ```

## Тусгай үйлчилгээ үзүүлэгч

`/connect` командын жагсаалтад ороогүй **OpenAI нийцтэй** үйлчилгээ үзүүлэгчийг нэмэхийн тулд:

:::tip
Та mongolgpt-тэй ямар ч OpenAI нийцтэй үйлчилгээ үзүүлэгчийг ашиглаж болно. Ихэнх орчин үеийн AI үйлчилгээ үзүүлэгчид OpenAI нийцтэй APIs санал болгодог.
:::

1. `/connect` командыг ажиллуулаад **Бусад** руу доош гүйлгэ.

   ```bash
   $ /connect

   ┌  Add credential
   │
   ◆  Select provider
   │  ...
   │  ● Other
   └
   ```

2. Үйлчилгээ үзүүлэгчийн хувьд өвөрмөц ID оруулна уу.

   ```bash
   $ /connect

   ┌  Add credential
   │
   ◇  Enter provider id
   │  myprovider
   └
   ```

   ::: тэмдэглэл
   Мартагдашгүй ID-г сонго, та үүнийг тохиргооны файлдаа ашиглах болно.
   :::

3. Үйлчилгээ үзүүлэгчийн хувьд API түлхүүрээ оруулна уу.

   ```bash
   $ /connect

   ┌  Add credential
   │
   ▲  This only stores a credential for myprovider - you will need to configure it in mongolgpt.json, check the docs for examples.
   │
   ◇  Enter your API key
   │  sk-...
   └
   ```

4. Төслийн лавлахдаа `mongolgpt.json` файлаа үүсгэх буюу шинэчлэх:

   ```json title="mongolgpt.json" ""myprovider"" {5-15}
   {
     "$schema": "https://mongolgpt.duckdns.org/config.json",
     "provider": {
       "myprovider": {
         "npm": "@ai-sdk/openai-compatible",
         "name": "My AI ProviderDisplay Name",
         "options": {
           "baseURL": "https://api.myprovider.com/v1"
         },
         "models": {
           "my-model-name": {
             "name": "My Model Display Name"
           }
         }
       }
     }
   }
   ```

   Энд тохиргооны сонголтууд байна:
   - **npm**: AI SDK ашиглах багц, OpenAI нийцтэй үйлчилгээ үзүүлэгчийн хувьд `@ai-sdk/openai-compatible` (`/v1/chat/completions` хувьд). Хэрэв таны үйлчилгээ үзүүлэгч/загвар `/v1/responses` ашигладаг бол `@ai-sdk/openai` ашиглана уу.
   - **нэр**: UI-д нэрийг харуулна.
   - **загвар**: Боломжтой загварууд.
   - **options.baseURL**: API төгсгөлийн цэг URL.
   - **options.apiKey**: Зөвшөөрөл ашиглаагүй тохиолдолд API товчлуурыг тохируулна уу.
   - **options.headers**: Сонголтоор захиалгат толгой хэсгийг тохируулна уу.

   Доорх жишээн дээрх дэвшилтэт сонголтуудын талаар дэлгэрэнгүй үзнэ үү.

5. `/models` командыг ажиллуулснаар таны захиалгат үйлчилгээ үзүүлэгч болон загварууд сонголтын жагсаалтад гарч ирнэ.

---

##### Жишээ

`apiKey`, `headers`, загвар `limit` сонголтуудыг тохируулах жишээ энд байна.

```json title="mongolgpt.json" {9,11,17-20}
{
  "$schema": "https://mongolgpt.duckdns.org/config.json",
  "provider": {
    "myprovider": {
      "npm": "@ai-sdk/openai-compatible",
      "name": "My AI ProviderDisplay Name",
      "options": {
        "baseURL": "https://api.myprovider.com/v1",
        "apiKey": "{env:ANTHROPIC_API_KEY}",
        "headers": {
          "Authorization": "Bearer custom-token"
        }
      },
      "models": {
        "my-model-name": {
          "name": "My Model Display Name",
          "limit": {
            "context": 200000,
            "output": 65536
          }
        }
      }
    }
  }
}
```

Тохиргооны дэлгэрэнгүй:

- **apiKey**: `env` хувьсагчийн синтакс, [илүү ихийг мэдэж аваарай](/docs/config#env-vars) ашиглан тохируулна.
- **толгой**: Хүсэлт болгонд захиалгат толгойг илгээдэг.
- **хязгаар.контекст**: Загвар хүлээн авах хамгийн их оролтын токенууд.
- **limit.output**: Загвар үүсгэж болох хамгийн их жетон.

`limit` талбарууд нь MongolGPT танд хэр их контекст үлдээснийг ойлгох боломжийг олгодог. Стандарт үйлчилгээ үзүүлэгчид эдгээрийг models.dev-ээс автоматаар татдаг.

## Алдааг олж засварлах

Хэрэв та үйлчилгээ үзүүлэгчийг тохируулахад асуудалтай байгаа бол дараахь зүйлийг шалгана уу.

1. **Autth тохиргоог шалгана уу**: `mongolgpt auth list`-г ажиллуулж итгэмжлэл байгаа эсэхийг шалгана уу.
   үйлчилгээ үзүүлэгч нь таны тохиргоонд нэмэгдсэн байна.

   Энэ нь баталгаажуулалтдаа орчны хувьсагчдаас хамаардаг Amazon Bedrock зэрэг үйлчилгээ үзүүлэгчдэд хамаарахгүй.

2. Захиалгат үйлчилгээ үзүүлэгчийн хувьд mongolgpt тохиргоог шалгаад:
   - `/connect` тушаалд ашигласан ID үйлчилгээ үзүүлэгч нь mongolgpt тохиргооны ID-тэй таарч байгаа эсэхийг шалгаарай.
- Үйлчилгээ үзүүлэгчийн хувьд зөв npm багцыг ашигладаг. Жишээлбэл, `@ai-sdk/cerebras`-г Cerebras-д ашиглана уу. Мөн бусад бүх OpenAI нийцтэй үйлчилгээ үзүүлэгчийн хувьд `@ai-sdk/openai-compatible` (`/v1/chat/completions` хувьд); Хэрэв загвар `/v1/responses` ашигладаг бол `@ai-sdk/openai`-г ашиглана уу. Нэг үйлчилгээ үзүүлэгчийн доорх холимог тохиргооны хувьд та `provider.npm`-ээр дамжуулан загвар тус бүрийг хүчингүй болгож болно.
   - `options.baseURL` талбарт API төгсгөлийн цэг ашигласан эсэхийг шалгана уу.
