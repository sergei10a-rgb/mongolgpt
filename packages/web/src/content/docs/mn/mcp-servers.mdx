---

Та _Загвар контекст протокол_, эсвэл MCP ашиглан MongolGPT-д гадаад хэрэгслийг нэмж болно. MongolGPT нь дотоод болон алсын серверүүдийг дэмждэг.

Нэгэнт нэмсэн бол MCP хэрэгслүүд нь суурилуулсан хэрэгслүүдийн хажуугаар LLM дээр автоматаар боломжтой болно.

---

#### Анхааруулга

MCP сервер ашиглах үед энэ нь контекстийг нэмдэг. Хэрэв танд олон хэрэгсэл байгаа бол энэ нь хурдан нэмэгдэх болно. Тиймээс бид аль MCP серверийг ашиглахдаа болгоомжтой байхыг зөвлөж байна.

:::tip
MCP серверүүд нь таны контекстийг нэмдэг тул та алийг нь идэвхжүүлэхдээ болгоомжтой хандахыг хүсч байна.
:::

GitHub MCP сервер гэх мэт зарим MCP серверүүд нь маш олон жетон нэмэх хандлагатай байдаг бөгөөд контекстийн хязгаараас амархан давж чаддаг.

---

## Идэвхжүүлэх

Та MCP серверүүдийг өөрийн [MongolGPT тохиргоо](https://mgpt.mn/docs/config/) дотор `mcp` доор тодорхойлж болно. MCP бүрийг өвөрмөц нэрээр нэмнэ үү. Та LLM гэж асуухдаа MCP-г нэрээр нь дурдаж болно.

```jsonc title="mongolgpt.jsonc" {6}
{
  "$schema": "https://mgpt.mn/config.json",
  "mcp": {
    "name-of-mcp-server": {
      // ...
      "enabled": true,
    },
    "name-of-other-mcp-server": {
      // ...
    },
  },
}
```

Мөн `enabled`-г `false` болгож серверийг идэвхгүй болгож болно. Хэрэв та серверийг тохиргооноосоо хасалгүйгээр түр идэвхгүй болгохыг хүсвэл энэ нь хэрэг болно.

---

### Алсын өгөгдмөлүүдийг дарж байна

Байгууллага нь `.well-known/mongolgpt` төгсгөлийн цэгээр дамжуулан анхдагч MCP серверүүдийг өгч болно. Эдгээр серверүүд нь анхдагчаар идэвхгүй болсон байж болох бөгөөд энэ нь хэрэглэгчдэд хэрэгцээтэй серверүүдээ сонгох боломжийг олгоно.

Байгууллагынхаа алсын тохиргооноос тодорхой серверийг идэвхжүүлэхийн тулд үүнийг өөрийн дотоод тохиргоонд `enabled: true`-аар нэмнэ үү:

```json title="mongolgpt.json"
{
  "$schema": "https://mgpt.mn/config.json",
  "mcp": {
    "jira": {
      "type": "remote",
      "url": "https://jira.example.com/mcp",
      "enabled": true
    }
  }
}
```

Таны локал тохиргооны утгууд алсын өгөгдмөл утгыг дарна. Дэлгэрэнгүй мэдээллийг [тохиргооны давуу эрх](/docs/config#precedence-order) харна уу.

---

## Орон нутгийн

MCP объект дотор `type` ашиглан орон нутгийн MCP серверүүдийг `"local"` руу нэмнэ.

```jsonc title="mongolgpt.jsonc" {15}
{
  "$schema": "https://mgpt.mn/config.json",
  "mcp": {
    "my-local-mcp-server": {
      "type": "local",
      // Эсвэл ["bun", "x", "my-mcp-command"]
      "command": ["npx", "-y", "my-mcp-command"],
      "enabled": true,
      "environment": {
        "MY_ENV_VAR": "my_env_var_value",
      },
    },
  },
}
```

Энэ тушаал нь локал MCP серверийг хэрхэн эхлүүлэхийг хэлнэ. Та мөн хүрээлэн буй орчны хувьсагчдын жагсаалтад орж болно.

Жишээлбэл, та тестийн [`@modelcontextprotocol/server-бүх зүйл`](https://www.npmjs.com/package/@modelcontextprotocol/server-everything) MCP серверийг хэрхэн нэмэх талаар эндээс үзнэ үү.

```jsonc title="mongolgpt.jsonc"
{
  "$schema": "https://mgpt.mn/config.json",
  "mcp": {
    "mcp_everything": {
      "type": "local",
      "command": ["npx", "-y", "@modelcontextprotocol/server-everything"],
    },
  },
}
```

Үүнийг ашиглахын тулд би өөрийн сануулгад `use the mcp_everything tool` нэмж болно.

```txt "mcp_everything"
use the mcp_everything tool to add the number 3 and 4
```

---

#### Сонголтууд

Орон нутгийн MCP серверийг тохируулах бүх сонголт энд байна.

| Сонголт       | Төрөл   | Шаардлагатай | Тодорхойлолт                                                                           |
| ------------- | ------- | ------------ | -------------------------------------------------------------------------------------- |
| `type`        | Мөр     | Y            | MCP серверийн холболтын төрөл нь `"local"` байх ёстой.                                 |
| `command`     | Массив  | Y            | MCP серверийг ажиллуулах тушаал ба аргументууд.                                        |
| `cwd`         | Мөр     |              | MCP серверийн процессын ажлын лавлах. Харьцангуй замууд нь ажлын талбараас шийдэгддэг. |
| `environment` | Объект  |              | Серверийг ажиллуулах үед тохируулах орчны хувьсагч.                                    |
| `enabled`     | Boolean |              | Эхлэх үед MCP серверийг идэвхжүүлэх эсвэл идэвхгүй болгох.                             |
| `timeout`     | Тоо     |              | MCP серверээс хэрэглүүр татахад ms-ээр хугацаа хэтэрсэн. Анхдагчаар 5000 (5 секунд).   |

---

## Алсын удирдлага

`type`-г `"remote"` болгож алсын MCP серверүүдийг нэмнэ үү.

```json title="mongolgpt.json"
{
  "$schema": "https://mgpt.mn/config.json",
  "mcp": {
    "my-remote-mcp": {
      "type": "remote",
      "url": "https://my-mcp-server.com",
      "enabled": true,
      "headers": {
        "Authorization": "Bearer MY_API_KEY"
      }
    }
  }
}
```

`url` нь алсын удирдлагатай MCP серверийн URL бөгөөд `headers` сонголтоор та толгойн жагсаалтад дамжуулж болно.

---

#### Сонголтууд

| Сонголт   | Төрөл   | Шаардлагатай | Тодорхойлолт                                                                         |
| --------- | ------- | ------------ | ------------------------------------------------------------------------------------ |
| `type`    | Мөр     | Y            | MCP серверийн холболтын төрөл нь `"remote"` байх ёстой.                              |
| `url`     | Мөр     | Y            | Алсын удирдлагатай MCP серверийн URL.                                                |
| `enabled` | Boolean |              | Эхлэх үед MCP серверийг идэвхжүүлэх эсвэл идэвхгүй болгох.                           |
| `headers` | Объект  |              | Хүсэлтийн хамт илгээх толгой хэсэг.                                                  |
| `oauth`   | Объект  |              | OAuth баталгаажуулалтын тохиргоо. Доорх [OAuth](#oauth) хэсгийг үзнэ үү.             |
| `timeout` | Тоо     |              | MCP серверээс хэрэглүүр татахад ms-ээр хугацаа хэтэрсэн. Анхдагчаар 5000 (5 секунд). |

## OAuth

MongolGPT алсын MCP серверт зориулсан OAuth баталгаажуулалтыг автоматаар зохицуулдаг. Серверт баталгаажуулалт шаардлагатай үед MongolGPT:

1. 401 хариултыг илрүүлж, OAuth урсгалыг эхлүүлнэ
2. Хэрэв сервер дэмждэг бол **Динамик Клиент Бүртгэл (RFC 7591)** ашиглана уу.
3. Ирээдүйн хүсэлтэд зориулж жетонуудыг найдвартай хадгал

---

### Автомат

Ихэнх OAuth-идэвхжүүлсэн MCP серверүүдийн хувьд тусгай тохиргоо хийх шаардлагагүй. Зүгээр л алсын серверийг тохируулна уу:

```json title="mongolgpt.json"
{
  "$schema": "https://mgpt.mn/config.json",
  "mcp": {
    "my-oauth-server": {
      "type": "remote",
      "url": "https://mcp.example.com/mcp"
    }
  }
}
```

Хэрэв сервер нэвтрэлт танилт шаардлагатай бол MongolGPT таныг анх ашиглах гэж оролдох үед таныг баталгаажуулахыг сануулна. Үгүй бол та [урсгалыг гараар өдөөх](#authenticating)-г `mongolgpt mcp auth <server-name>`-тэй хамт хийж болно.

---

### Урьдчилан бүртгүүлсэн

Хэрэв танд MCP сервер үйлчилгээ үзүүлэгчээс үйлчлүүлэгчийн итгэмжлэл байгаа бол та тэдгээрийг тохируулж болно:

```json title="mongolgpt.json" {7-11}
{
  "$schema": "https://mgpt.mn/config.json",
  "mcp": {
    "my-oauth-server": {
      "type": "remote",
      "url": "https://mcp.example.com/mcp",
      "oauth": {
        "clientId": "{env:MY_MCP_CLIENT_ID}",
        "clientSecret": "{env:MY_MCP_CLIENT_SECRET}",
        "scope": "tools:read tools:execute"
      }
    }
  }
}
```

---

### Баталгаажуулж байна

Та баталгаажуулалтыг гараар эхлүүлэх эсвэл итгэмжлэлийг удирдах боломжтой.

Тодорхой MCP серверээр баталгаажуулах:

```bash
mongolgpt mcp auth my-oauth-server
```

Бүх MCP серверүүд болон тэдгээрийн баталгаажуулалтын статусыг жагсаана уу:

```bash
mongolgpt mcp list
```

Хадгалагдсан итгэмжлэлүүдийг устгах:

```bash
mongolgpt mcp logout my-oauth-server
```

`mcp auth` тушаал нь таны хөтөчийг зөвшөөрөл авахын тулд нээх болно. Таныг зөвшөөрсний дараа MongolGPT тэмдэгтүүдийг `~/.local/share/mongolgpt/mcp-auth.json`-д найдвартай хадгална.

---

#### Идэвхгүй болгож байна OAuth

Хэрэв та серверийн автомат OAuth-г идэвхгүй болгохыг хүсвэл (жишээ нь, оронд нь API товчлуурыг ашигладаг серверүүдэд) `oauth`-г `false` гэж тохируулна уу:

```json title="mongolgpt.json" {7}
{
  "$schema": "https://mgpt.mn/config.json",
  "mcp": {
    "my-api-key-server": {
      "type": "remote",
      "url": "https://mcp.example.com/mcp",
      "oauth": false,
      "headers": {
        "Authorization": "Bearer {env:MY_API_KEY}"
      }
    }
  }
}
```

---

#### OAuth Сонголтууд

| Сонголт        | Төрөл           | Тодорхойлолт                                                                                |
| -------------- | --------------- | ------------------------------------------------------------------------------------------- |
| `oauth`        | Объект \| худал | OAuth тохиргооны объект эсвэл OAuth автомат илрүүлэлтийг идэвхгүй болгохын тулд `false`.    |
| `clientId`     | Мөр             | OAuth үйлчлүүлэгч ID. Хэрэв оруулаагүй бол динамик үйлчлүүлэгчийн бүртгэлийг оролдох болно. |
| `clientSecret` | Мөр             | Зөвшөөрлийн сервер шаардлагатай бол OAuth үйлчлүүлэгчийн нууц.                              |
| `scope`        | Мөр             | OAuth зөвшөөрлийн үед хүсэлт гаргах боломжтой.                                              |

#### Дибаг хийх

Хэрэв алсын MCP сервер баталгаажуулах боломжгүй бол та дараах асуудлыг оношлох боломжтой.

```bash
# OAuth дэмждэг бүх серверийн нэвтрэлтийн төлөвийг харах
mongolgpt mcp auth list

# Сонгосон серверийн холболт болон OAuth урсгалыг оношлох
mongolgpt mcp debug my-oauth-server
```

`mcp debug` тушаал нь одоогийн баталгаажуулалтын статусыг харуулж, HTTP холболтыг шалгаж, OAuth илрүүлэх урсгалыг оролддог.

## Удирдах

Таны MCP нь суурилуулсан хэрэгслүүдийн хажуугаар MongolGPT-д хэрэгсэл хэлбэрээр байдаг. Тиймээс та бусад хэрэгслийн нэгэн адил MongolGPT тохиргоогоор дамжуулан тэдгээрийг удирдах боломжтой.

---

### Дэлхийн

Энэ нь та тэдгээрийг дэлхий даяар идэвхжүүлэх эсвэл идэвхгүй болгох боломжтой гэсэн үг юм.

```json title="mongolgpt.json" {14}
{
  "$schema": "https://mgpt.mn/config.json",
  "mcp": {
    "my-mcp-foo": {
      "type": "local",
      "command": ["bun", "x", "my-mcp-command-foo"]
    },
    "my-mcp-bar": {
      "type": "local",
      "command": ["bun", "x", "my-mcp-command-bar"]
    }
  },
  "tools": {
    "my-mcp-foo": false
  }
}
```

Бид мөн бүх тохирох MCPs-г идэвхгүй болгохын тулд бөмбөрцгийн загварыг ашиглаж болно.

```json title="mongolgpt.json" {14}
{
  "$schema": "https://mgpt.mn/config.json",
  "mcp": {
    "my-mcp-foo": {
      "type": "local",
      "command": ["bun", "x", "my-mcp-command-foo"]
    },
    "my-mcp-bar": {
      "type": "local",
      "command": ["bun", "x", "my-mcp-command-bar"]
    }
  },
  "tools": {
    "my-mcp*": false
  }
}
```

Энд бид бүх MCP-г идэвхгүй болгохын тулд `my-mcp*` глоб загварыг ашиглаж байна.

---

### Агент бүрт

Хэрэв танд олон тооны MCP сервер байгаа бол тэдгээрийг зөвхөн нэг агент болгон идэвхжүүлж, дэлхий даяар идэвхгүй болгохыг хүсэж болно. Үүнийг хийхийн тулд:

1. Үүнийг дэлхий даяар хэрэгсэл болгон идэвхгүй болго.
2. [агентын тохиргоо](/docs/agents#tools)-д MCP серверийг хэрэгсэл болгон идэвхжүүл.

```json title="mongolgpt.json" {11, 14-18}
{
  "$schema": "https://mgpt.mn/config.json",
  "mcp": {
    "my-mcp": {
      "type": "local",
      "command": ["bun", "x", "my-mcp-command"],
      "enabled": true
    }
  },
  "tools": {
    "my-mcp*": false
  },
  "agent": {
    "my-agent": {
      "tools": {
        "my-mcp*": true
      }
    }
  }
}
```

---

#### Бөмбөрцөг хээ

Глоб загвар нь энгийн regex globbing загварыг ашигладаг:

- `*` нь тэг буюу түүнээс дээш тэмдэгттэй таарч байна (жишээ нь, `"my-mcp*"` нь `my-mcp_search`, `my-mcp_list` гэх мэт)
- `?` яг нэг тэмдэгттэй таарч байна
- Бусад бүх дүрүүд шууд утгаараа таарч байна

:::note
MCP серверийн хэрэгслүүд нь серверийн нэрээр угтвар хэлбэрээр бүртгэгдсэн тул серверийн бүх хэрэгслийг идэвхгүй болгохын тулд дараахыг ашиглана уу:

```
"mymcpservername_*": false
```

:::

## Жишээ

Зарим нийтлэг MCP серверүүдийн жишээг доор харуулав. Хэрэв та бусад серверүүдийг баримтжуулахыг хүсвэл PR илгээх боломжтой.

---

### Харуул

Таны Sentry төсөл болон асуудлуудтай харилцахын тулд [Sentry MCP сервер](https://mcp.sentry.dev) нэмнэ үү.

```json title="mongolgpt.json" {4-8}
{
  "$schema": "https://mgpt.mn/config.json",
  "mcp": {
    "sentry": {
      "type": "remote",
      "url": "https://mcp.sentry.dev/mcp",
      "oauth": {}
    }
  }
}
```

Тохиргоог нэмсний дараа Sentry ашиглан баталгаажуулна уу:

```bash
mongolgpt mcp auth sentry
```

Энэ нь OAuth урсгалыг дуусгах хөтчийн цонхыг нээж, MongolGPT-г таны Sentry бүртгэлтэй холбох болно.

Баталгаажуулсны дараа та асуудал, төсөл болон алдааны өгөгдлийг асуухын тулд өөрийн сануулгад Sentry хэрэгслийг ашиглаж болно.

```txt "use sentry"
Show me the latest unresolved issues in my project. use sentry
```

---

### Контекст7

Доксоор хайхын тулд [Context7 MCP сервер](https://github.com/upstash/context7) нэмнэ үү.

```json title="mongolgpt.json" {4-7}
{
  "$schema": "https://mgpt.mn/config.json",
  "mcp": {
    "context7": {
      "type": "remote",
      "url": "https://mcp.context7.com/mcp"
    }
  }
}
```

Хэрэв та үнэгүй дансанд бүртгүүлсэн бол API түлхүүрээ ашиглан илүү өндөр тарифын хязгаарлалт авах боломжтой.

```json title="mongolgpt.json" {7-9}
{
  "$schema": "https://mgpt.mn/config.json",
  "mcp": {
    "context7": {
      "type": "remote",
      "url": "https://mcp.context7.com/mcp",
      "headers": {
        "CONTEXT7_API_KEY": "{env:CONTEXT7_API_KEY}"
      }
    }
  }
}
```

Энд бид таныг `CONTEXT7_API_KEY` орчны хувьсагчтай гэж таамаглаж байна.

Context7 MCP серверийг ашиглахын тулд сануулгууддаа `use context7` нэмнэ.

```txt "use context7"
Configure a Cloudflare Worker script to cache JSON API responses for five minutes. use context7
```

Эсвэл та [AGENTS.md](/docs/rules/)-д ийм зүйл нэмж болно.

```md title="AGENTS.md"
When you need to search docs, use `context7` tools.
```

### Греп Версел

[Греп, Версел](https://grep.app) MCP серверийг нэмээд GitHub дээрх кодын хэсгүүдээр хайлт хийнэ үү.

```json title="mongolgpt.json" {4-7}
{
  "$schema": "https://mgpt.mn/config.json",
  "mcp": {
    "gh_grep": {
      "type": "remote",
      "url": "https://mcp.grep.app"
    }
  }
}
```

Бид MCP серверээ `gh_grep` гэж нэрлэсэн тул та төлөөлөгчдөө ашиглахын тулд өөрийн сануулгад `use the gh_grep tool`-г нэмж болно.
