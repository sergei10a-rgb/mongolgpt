---
title: Permissions
description: Ямар үйлдлийг гүйцэтгэхийн тулд зөвшөөрөл шаардлагатайг хянах.
---

MongolGPT нь `permission` тохиргоог ашиглан өгөгдсөн үйлдлийг автоматаар гүйцэтгэх, танд сануулах эсвэл хаагдах эсэхийг шийддэг.

`v1.1.1`-ийн байдлаар, хуучин `tools` логикийн тохиргоо хуучирсан бөгөөд `permission`-д нэгтгэгдсэн. Хуучин `tools` тохиргоог буцаан нийцүүлэхийн тулд дэмжсэн хэвээр байна.

---

## Үйлдэл

Зөвшөөрлийн дүрэм бүр нь дараахь зүйлийг шийддэг.

- `"allow"` - зөвшөөрөлгүйгээр ажиллуулах
- `"ask"` — зөвшөөрөл авах хүсэлт
- `"deny"` — үйлдлийг хаах

---

## Тохиргоо

Та зөвшөөрлийг дэлхий даяар (`*`-тай) тохируулж, тусгай хэрэгслийг хүчингүй болгож болно.

```json title="mongolgpt.json"
{
  "$schema": "https://mongolgpt.duckdns.org/config.json",
  "permission": {
    "*": "ask",
    "bash": "allow",
    "edit": "deny"
  }
}
```

Та мөн бүх зөвшөөрлийг нэг дор тохируулж болно:

```json title="mongolgpt.json"
{
  "$schema": "https://mongolgpt.duckdns.org/config.json",
  "permission": "allow"
}
```

---

## Мөхлөгт дүрмүүд (Объектийн синтакс)

Ихэнх зөвшөөрлийн хувьд та багажийн оролт дээр тулгуурлан өөр өөр үйлдэл хийх объектыг ашиглаж болно.

```json title="mongolgpt.json"
{
  "$schema": "https://mongolgpt.duckdns.org/config.json",
  "permission": {
    "bash": {
      "*": "ask",
      "git *": "allow",
      "npm *": "allow",
      "rm *": "deny",
      "grep *": "allow"
    },
    "edit": {
      "*": "deny",
      "packages/web/src/content/docs/*.mdx": "allow"
    }
  }
}
```

Дүрмүүдийг загварын тохирлоор үнэлдэг ба **хамгийн сүүлд тохирсон дүрэм ялна**. Нийтлэг загвар бол catch-all `"*"` дүрмийг эхлээд, дараа нь илүү тодорхой дүрмийг тавих явдал юм.

### Орлогч тэмдэг

Зөвшөөрлийн загвар нь энгийн орлуулагдах тэмдэгтийг ашиглана:

- `*` нь ямар ч тэмдэгтийн тэг буюу түүнээс дээш хэмжээтэй таарч байна
- `?` яг нэг тэмдэгттэй таарч байна
- Бусад бүх дүрүүд шууд утгаараа таарч байна

### Нүүр хуудасны лавлах өргөтгөл

Та хээний эхэнд `~` эсвэл `$HOME`-г ашиглан гэрийн лавлах лавлах боломжтой. Энэ нь ялангуяа [`гадаад_заах'](#external-directories) дүрэмд хэрэгтэй.

- `~/projects/*` -> `/Users/username/projects/*`
- `$HOME/projects/*` -> `/Users/username/projects/*`
- `~` -> `/Users/username`

### Гадаад лавлахууд

`external_directory`-г эхлүүлсэн ажлын лавлахын гаднах замд хүрэх хэрэгслийн дуудлагыг зөвшөөрөхийн тулд `external_directory` ашиглана уу. Энэ нь оролт болгон замыг авдаг аливаа хэрэгсэлд хамаарна (жишээ нь `read`, `edit`, `glob`, `grep` болон олон `bash` тушаалууд).

Гэрийн өргөтгөл (`~/...` гэх мэт) нь зөвхөн загварыг хэрхэн бичихэд нөлөөлдөг. Энэ нь одоогийн ажлын талбарын гадаад замын хэсэг болгодоггүй тул ажлын лавлахаас гадуурх замыг `external_directory`-ээр дамжуулан зөвшөөрөх ёстой.

Жишээлбэл, энэ нь `~/projects/personal/` доорх бүх зүйлд хандах боломжийг олгоно:

```json title="mongolgpt.json"
{
  "$schema": "https://mongolgpt.duckdns.org/config.json",
  "permission": {
    "external_directory": {
      "~/projects/personal/**": "allow"
    }
  }
}
```

Энд зөвшөөрөгдсөн дурын лавлах нь одоогийн ажлын талбартай ижил өгөгдмөлүүдийг өвлөнө. [`унших` нь анхдагчаар `зөвшөөрөх` байна](#defaults) тул, хүчингүй болгоогүй бол `external_directory` доорх бичилтүүдийг уншихыг зөвшөөрдөг. Уншсан мэдээллийг хадгалах явцад засварыг блоклох гэх мэт эдгээр замд хэрэгслийг хязгаарлах шаардлагатай тодорхой дүрмийг нэмнэ үү:

```json title="mongolgpt.json"
{
  "$schema": "https://mongolgpt.duckdns.org/config.json",
  "permission": {
    "external_directory": {
      "~/projects/personal/**": "allow"
    },
    "edit": {
      "~/projects/personal/**": "deny"
    }
  }
}
```

Жагсаалтыг итгэмжлэгдсэн замууд дээр төвлөрүүлж, бусад хэрэгсэлд шаардлагатай бол нэмэлт дүрмүүдийг зөвшөөрөх эсвэл үгүйсгэх (жишээ нь `bash`).

---

## Боломжтой зөвшөөрөл

MongolGPT зөвшөөрлүүд нь багаж хэрэгслийн нэр, мөн хэд хэдэн хамгаалалтын хамгаалалтаар тодорхойлогддог:

- `read` — файл унших (файлын замд таарч байна)
- `edit` — файлын бүх өөрчлөлт (`edit`, `write`, `patch` хамарна)
- `glob` — файлын бөмбөрцөг (глоб загвартай таарч байна)
- `grep` — агуулгын хайлт (regex загварт таарч байна)
- `bash` — бүрхүүлийн командуудыг ажиллуулж байна (`git status --porcelain` гэх мэт задалсан командуудтай таарч байна)
- `task` — дэд агентуудыг ажиллуулж байна (дэд агентын төрөлд таарч байна)
- `skill` — ур чадвар ачаалж байна (ур чадварын нэрэнд таарч байна)
- `lsp` — LSP асуулга ажиллуулж байна (одоогоор нарийн биш)
- `question` - гүйцэтгэлийн явцад хэрэглэгчээс асуулт асуух
- `webfetch` — URL дуудаж байна (URL-тэй таарч байна)
- `websearch` — вэб хайлт (асуулгатай тохирч байна)
- `external_directory` - хэрэгсэл нь төслийн ажлын лавлахаас гадна замд хүрэх үед идэвхждэг
- `doom_loop` — нэг багажны дуудлагыг ижил оролтоор 3 удаа давтах үед идэвхждэг.

---

## Өгөгдмөл

Хэрэв та юу ч заагаагүй бол MongolGPT зөвшөөрөгдсөн өгөгдмөлөөс эхэлнэ:

- Ихэнх зөвшөөрлүүд өгөгдмөлөөр `"allow"` байна.
- `doom_loop` болон `external_directory` анхдагч нь `"ask"`.
- `read` нь `"allow"` боловч `.env` файлууд нь анхдагчаар татгалзсан байна:

```json title="mongolgpt.json"
{
  "permission": {
    "read": {
      "*": "allow",
      "*.env": "deny",
      "*.env.*": "deny",
      "*.env.example": "allow"
    }
  }
}
```

## “Асуу” юу хийдэг вэ

MongolGPT зөвшөөрөл хүсэхэд UI гурван үр дүнг санал болгодог:

- `once` - зөвхөн энэ хүсэлтийг зөвшөөрнө үү
- `always` - санал болгож буй загварт тохирсон ирээдүйн хүсэлтийг батлах (одоогийн MongolGPT сессийн үлдсэн хугацаанд)
- `reject` - хүсэлтийг үгүйсгэх

`always`-ийн зөвшөөрөх загваруудын багцыг хэрэгслээр хангадаг (жишээ нь, bash зөвшөөрөл нь `git status*` гэх мэт аюулгүй командын угтварыг цагаан жагсаалтад оруулах).

## Агентууд

Та төлөөлөгч бүрийн зөвшөөрлийг хүчингүй болгож болно. Агентийн зөвшөөрлийг глобал тохиргоотой нэгтгэсэн бөгөөд агентын дүрмүүдийг давуу эрхтэй болгоно. [Илүү ихийг мэдэж аваарай](/docs/agents#permissions) төлөөлөгчийн зөвшөөрлийн талаар.

:::note
Дэлгэрэнгүй загвар тохирох жишээг дээрх [Нарийн дүрмүүд (Объектийн синтакс)](#granular-rules-object-syntax) хэсгээс харна уу.
:::

```json title="mongolgpt.json"
{
  "$schema": "https://mongolgpt.duckdns.org/config.json",
  "permission": {
    "bash": {
      "*": "ask",
      "git *": "allow",
      "git commit *": "deny",
      "git push *": "deny",
      "grep *": "allow"
    }
  },
  "agent": {
    "build": {
      "permission": {
        "bash": {
          "*": "ask",
          "git *": "allow",
          "git commit *": "ask",
          "git push *": "deny",
          "grep *": "allow"
        }
      }
    }
  }
}
```

Та мөн Markdown-д агентийн зөвшөөрлийг тохируулах боломжтой:

```markdown title="~/.config/mongolgpt/agents/review.md"
---
description: Code review without edits
mode: subagent
permission:
  edit: deny
  bash: ask
  webfetch: deny
---

Only analyze code and suggest changes.
```

:::tip
Аргументтай командуудад тохирох загвар ашиглана уу. `"grep *"` нь `grep pattern file.txt`-г зөвшөөрдөг бол `"grep"` ганцаараа үүнийг хориглох болно. `git status` гэх мэт тушаалууд нь анхдагч үйлдлийн хувьд ажилладаг боловч аргумент дамжуулагдах үед тодорхой зөвшөөрөл (`"git status *"` гэх мэт) шаарддаг.
:::
