## Одоогийн төлөв

**Нийтийн share link одоогоор дэмжигдэхгүй.** `/share` болон `/unshare` командын client талын интерфэйс байж болох ч тэдгээрт шаардлагатай MongolGPT-ийн өөрийн account/share backend бэлэн болоогүй. Иймээс launch profile-д public URL үүсгэх боломж гэж үзэж болохгүй.

:::caution
Танихгүй share домэйнд сесс, source code, API key, нэвтрэх эсвэл төлбөрийн мэдээлэл бүү илгээ. CLI, TUI, local web, desktop болон BYOK provider ашиглахад share service хэрэггүй.
:::

Launch тохиргоонд share-ийг илэрхий унтраана:

```json title="mongolgpt.json"
{
  "share": "disabled"
}
```

## Одоогоор сесс дамжуулах

Нийтийн холбоосын оронд сессийг локал файл болгон экспортлоно.

TUI-д Markdown экспорт нээх:

```txt
/export
```

CLI-аас JSON экспортлох:

```bash
mongolgpt export [sessionID] > session.json
```

Файлыг бусдад өгөхийн өмнө prompt, tool output, diff, файлын зам, secret, token болон хэрэглэгчийн өгөгдөл байгаа эсэхийг шалгаж цэвэрлэнэ. Дараа нь байгууллагын баталсан файл дамжуулах сувгийг ашиглана.

## Ирээдүйн hosted байршуулалт

Төслийн өөрийн share/account backend бүрэн бэлэн болж, release notes-д public share дэмжлэгийг идэвхжүүлсэн үед л repository-ийн `.env.example` доторх `MONGOLGPT_PUBLIC_URL`, `MONGOLGPT_API_URL`, `MONGOLGPT_SHARE_URL` зэрэг утгыг өөрийн эзэмшлийн production URL-аар тохируулна. Одоогоор:

```dotenv
MONGOLGPT_ENABLE_HOSTED_SERVICES=false
```

`.env.example` доторх `https://your-domain.example` нь placeholder бөгөөд ажилладаг service биш. Backend-ийг идэвхжүүлэхээс өмнө authentication, authorization, TLS, retention, deletion, abuse protection, audit logging болон incident response-ийг баталгаажуулах шаардлагатай.

Байгууллагын үндсэн тохиргоог [байгууллагын хэрэглээ](../enterprise), локал export-ийн командуудыг [CLI](../cli#export) болон [TUI](../tui)-ийн заавраас харна уу.
