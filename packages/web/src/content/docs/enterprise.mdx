MongolGPT-ийг байгууллагад нэвтрүүлэхдээ эхлээд жижиг баг, туршилтын repository, батлагдсан provider-оор турш. Энэ хуудас нийтэд зарлаагүй managed enterprise үйлчилгээ, үнэ, SLA амлахгүй; одоогийн open-source хувилбарыг хянаж байршуулж буй багт зориулсан суурь заавар юм.

## Эхлэх шалгах хуудас

1. [GitHub Releases](https://github.com/sergei10a-rgb/mongolgpt/releases)-ээс баталсан хувилбараа сонго.
2. Binary болон desktop installer-ийг дотоод software distribution сувгаараа түгээ.
3. Ашиглах AI provider, model, region, retention бодлогыг аюулгүй байдлын багаараа батлуул.
4. Төслийн `mongolgpt.json`-д provider, permission, policy, plugin, MCP хязгаарлалтаа commit хий.
5. `/share`-ийг анхдагчаар идэвхгүй болго.
6. Нууц түлхүүрийг config эсвэл Git-д хадгалахгүй, байгууллагын secret manager ашигла.
7. Лог, сессийн хадгалалт, устгал, incident response-ийн дотоод журмаа тогтоо.

```json title="mongolgpt.json"
{
  "share": "disabled",
  "permission": {
    "bash": "ask",
    "edit": "ask",
    "external_directory": "deny"
  }
}
```

## Өгөгдлийн урсгал

MongolGPT CLI сесс, лог, provider-ийн эрхийн мэдээллийг хэрэглэгчийн машинд хадгалж болно. Prompt, сонгосон файлын контент, tool output нь ашиглаж буй AI provider руу хүсэлтээр илгээгдэнэ. Plugin болон MCP server нэмбэл тэдгээр нь тусдаа өгөгдлийн урсгал үүсгэж болно.

Байгууллагын үнэлгээнд дараах талуудыг тус бүр шалгана.

- AI provider-ийн data retention, training, region, subprocessors
- OAuth болон API key-ийн scope, rotation, revoke ажиллагаа
- Plugin болон MCP server-ийн эх код, publisher, network access
- Локал лог, сессийн хадгалах хугацаа болон backup
- Permission, policy, external directory дүрэм
- Хуваалцах болон алсын web server-ийн тохиргоо

:::caution
"Локал клиент" гэдэг нь prompt зөвхөн локалд үлдэнэ гэсэн үг биш. Сонгосон provider-ийн API руу илгээх контентыг provider-ийн нөхцөл, байгууллагын бодлоготой тулгаж шалга.
:::

## Хуваалцахыг хязгаарлах

Нийтийн share болон MongolGPT account backend одоогоор дэмжигдэхгүй. Байгууллагын launch profile-д share-ийг идэвхгүй байлгана.

```json title="mongolgpt.json"
{
  "share": "disabled"
}
```

CLI/TUI, локал web, desktop болон BYOK provider ашиглахад production domain эсвэл hosting шаардлагагүй. Ирээдүйд төслийн өөрийн hosted backend бэлэн болж release notes-д дэмжлэгийг идэвхжүүлсний дараа repository-ийн `.env.example` доторх URL-уудыг зөвхөн байгууллагын өөрийн эзэмшлийн production URL-аар тохируулна. Тэр хүртэл `MONGOLGPT_ENABLE_HOSTED_SERVICES=false` хэвээр байна. [Share-ийн төлөв](../share)-ийг тусад нь шалга.

## Төвлөрсөн тохиргоо

Төслийн түвшний `mongolgpt.json`-ийг repository-д commit хийснээр багийн provider, permission, policy, MCP, plugin тохиргоог ижил байлгаж болно. Нууц утгыг `{env:VARIABLE_NAME}` хэлбэрийн орчны хувьсагчаар өгнө.

```json title="mongolgpt.json"
{
  "provider": {
    "openai": {
      "options": {
        "apiKey": "{env:OPENAI_API_KEY}"
      }
    }
  },
  "share": "disabled"
}
```

Тохиргооны хүчин төгөлдөр утгыг хэрэглэгчийн машин дээр шалгана.

```bash
mongolgpt debug config
```

Тохиргооны precedence болон боломжтой key-үүдийг [тохиргооны лавлагаанаас](../config) үзнэ үү.

## Дотоод AI gateway

Байгууллагын gateway нь OpenAI-compatible API өгдөг бол [custom provider](../providers#тусгай-үйлчилгээ-үзүүлэгч) хэлбэрээр тохируулж болно. `baseURL`, model ID, header, certificate, proxy-г gateway-ийн албан ёсны утгаар өгнө.

Бусад provider-ийг хязгаарлахдаа зөвхөн UI жагсаалт нуухад найдахгүй. Egress firewall, proxy allowlist, provider policy, secret distribution-ийг хамтад нь ашигла.

## SSO ба алсын хандалт

Open-source CLI нь provider бүрийн дэмжсэн OAuth эсвэл API key аргыг ашиглана. Байгууллагын SSO-г өөрийн web/reverse proxy давхаргад нэмэх бол:

1. HTTPS болон байгууллагын IdP authentication тохируул.
2. Upstream MongolGPT серверт зөвхөн reverse proxy-оос хандахыг зөвшөөр.
3. `MONGOLGPT_SERVER_PASSWORD`-г нэмэлт хамгаалалт болгон ашигла.
4. Session timeout, group authorization, audit log-ийг турш.
5. Нэвтрэх тойрч гарах локал болон network route байхгүйг шалга.

Managed SSO боломж байгаа мэтээр нийтэд зарлахын өмнө бодит deployment, support owner, SLA-г тусад нь баталгаажуулна.

## Хувийн npm registry

Хэрэв байгууллага npm dependency-г JFrog Artifactory, Nexus зэрэг хувийн registry-ээр дамжуулдаг бол эхлээд npm эрхээ тохируулна.

```bash
npm login --registry=https://your-company.jfrog.io/api/npm/npm-virtual/
```

Эсвэл `~/.npmrc`:

```ini title="~/.npmrc"
registry=https://your-company.jfrog.io/api/npm/npm-virtual/
//your-company.jfrog.io/api/npm/npm-virtual/:_authToken=${NPM_AUTH_TOKEN}
```

`NPM_AUTH_TOKEN`-г source control-д бүү commit хий. Token-ийн scope болон хугацааг хамгийн бага шаардлагатай хэмжээнд тохируул.

## Түгээмэл асуулт

<details>
<summary>MongolGPT код эсвэл prompt-ийг хаашаа илгээдэг вэ?</summary>

Локал client өөрөө сесс, лог хадгалж болно. AI хүсэлтэд багтсан prompt, файл, tool output нь сонгосон provider руу илгээгдэнэ. Plugin, MCP, share ашиглавал нэмэлт үйлчилгээ рүү өгөгдөл дамжиж болно.

</details>

<details>
<summary>Байгууллагын бүх хэрэглэгчид ижил тохиргоо ашиглаж болох уу?</summary>

Төслийн `mongolgpt.json`-ийг commit хийж суурь тохиргоог ижил байлгаж болно. Нууц утгыг орчны хувьсагч эсвэл secret manager-аас өгнө. OS-level managed configuration ашиглах бол тухайн платформ дээр туршиж, `mongolgpt debug config`-оор үр дүнг баталгаажуул.

</details>

<details>
<summary>Managed enterprise үнэ, SLA хаана байна?</summary>

Энэ баримт бичиг managed offering, үнэ, SLA зарлаагүй. Одоогийн боломжийг source code болон release notes-оор шалгаж, шаардлагатай feature эсвэл deployment асуултыг [GitHub Issues](https://github.com/sergei10a-rgb/mongolgpt/issues)-д нууц мэдээлэлгүйгээр оруулна.

</details>
