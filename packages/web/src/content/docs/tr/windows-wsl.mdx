mongolgpt Windows'ta dogrudan calisabilir, ancak en iyi deneyim icin [Windows Subsystem for Linux (WSL)](https://learn.microsoft.com/en-us/windows/wsl/install) kullanmanizi oneriyoruz. WSL, mongolgpt ozellikleriyle sorunsuz calisan bir Linux ortami saglar.

:::tip[Neden WSL?]
WSL, daha iyi dosya sistemi performansi, tam terminal destegi ve mongolgpt'un dayandigi gelistirme araclariyla uyumluluk sunar.
:::

---

## Kurulum

<Steps>

1. **WSL'i yukleyin**

   Henuz yuklemediyseniz resmi Microsoft rehberini kullanarak [WSL kurulumunu](https://learn.microsoft.com/en-us/windows/wsl/install) yapin.

2. **mongolgpt'u WSL icine yukleyin**

   WSL hazir olduktan sonra WSL terminalini acin ve mongolgpt'u [kurulum yontemlerinden](/docs/) biriyle yukleyin.

   ```bash
   git clone https://github.com/sergei10a-rgb/mongolgpt && cd mongolgpt && bun install
   ```

3. **mongolgpt'u WSL icinden kullanin**

   Proje dizininize gidin (`/mnt/c/`, `/mnt/d/` vb. ile Windows dosyalarina erisebilirsiniz) ve mongolgpt'u calistirin.

   ```bash
   cd /mnt/c/Users/YourName/project
   mongolgpt
   ```

</Steps>

## Masaüstü uygulaması + WSL sunucusu

mongolgpt Desktop kullanmak isteyip sunucuyu WSL'de calistirmak istiyorsaniz:

1. **Sunucuyu WSL'de baslatin** ve dis baglantilara izin vermek icin `--hostname 0.0.0.0` kullanin:

   ```bash
   mongolgpt serve --hostname 0.0.0.0 --port 4096
   ```

2. **Desktop uygulamasini** `http://localhost:4096` adresine baglayin

:::note
Kurulumunuzda `localhost` calismiyorsa bunun yerine WSL IP adresini kullanin (WSL icinde: `hostname -I`) ve `http://<wsl-ip>:4096` adresine baglanin.
:::

:::caution
`--hostname 0.0.0.0` kullaniyorsaniz sunucuyu guvene almak icin `MONGOLGPT_SERVER_PASSWORD` ayarlayin.

```bash
MONGOLGPT_SERVER_PASSWORD=your-password mongolgpt serve --hostname 0.0.0.0
```

:::

## Web istemcisi + WSL

Windows'ta en iyi web deneyimi icin:

1. **`mongolgpt web` komutunu PowerShell yerine WSL terminalinde calistirin**:

   ```bash
   mongolgpt web --hostname 0.0.0.0
   ```

2. **Windows tarayicinizdan** `http://localhost:<port>` adresini acin (mongolgpt URL'yi yazdirir)

`mongolgpt web` komutunu WSL'den calistirmak, dosya sistemi erisimi ve terminal entegrasyonunu daha tutarli hale getirirken tarayicidan erisimi korur.

## Windows dosyalarına erişin

WSL, tum Windows dosyalariniza `/mnt/` dizini uzerinden erisebilir:

- `C:` surucusu -> `/mnt/c/`
- `D:` surucusu -> `/mnt/d/`
- Ve digerleri...

Ornek:

```bash
cd /mnt/c/Users/YourName/Documents/project
mongolgpt
```

:::tip
En sorunsuz deneyim icin deponuzu WSL dosya sistemine (ornegin `~/code/` altina) klonlayip mongolgpt'u orada calistirmayi dusunun.
:::

---

## İpuçları

- Windows suruculerinde duran projelerde mongolgpt'u WSL icinde calistirin - dosya erisimi daha sorunsuz olur
- Butunlesik bir gelistirme akisi icin mongolgpt ile birlikte VS Code [WSL eklentisini](https://code.visualstudio.com/docs/remote/wsl) kullanin
- mongolgpt konfigurasyonunuz ve oturumlariniz WSL ortaminda `~/.local/share/mongolgpt/` konumunda saklanir
