---
title: Захиалгат командууд
description: Дахин давтагдах даалгавруудад зориулсан захиалгат тушаалуудыг үүсгэ.
---

Захиалгат тушаалууд нь TUI-д тухайн тушаалыг гүйцэтгэх үед ажиллуулахыг хүссэн хүсэлтийг зааж өгөх боломжийг танд олгоно.

```bash frame="none"
/my-command
```

Custom command нь `/init`, `/undo`, `/redo`, `/share`, `/help` зэрэг built-in command-аас тусдаа. [TUI командуудыг](../tui#командууд) харна уу.

---

## Тушаалын файл үүсгэх

Захиалгат тушаалуудыг тодорхойлохын тулд `commands/` директорт бууруулах файлуудыг үүсгэ.

`.mongolgpt/commands/test.md` үүсгэх:

```md title=".mongolgpt/commands/test.md"
---
description: Тестийг coverage-тай ажиллуулах
agent: build
model: anthropic/claude-3-5-sonnet-20241022
---

Бүх тестийг coverage тайлантай ажиллуулж, унасан тестүүдийг харуул.
Унасан тестүүдэд төвлөрч засвар санал болго.
```

Frontmatter нь тушаалын шинж чанарыг тодорхойлдог. Агуулга нь загвар болж хувирдаг.

`/` гэж бичээд дараа нь тушаалын нэрийг бичнэ үү.

```bash frame="none"
"/test"
```

---

## Тохируулах

Та захиалгат тушаалуудыг MongolGPT тохиргоогоор эсвэл `commands/` директорт бууруулах файлуудыг үүсгэх замаар нэмж болно.

---

### JSON

Өөрийн MongolGPT [config](../config) дээрх `command` сонголтыг ашиглана уу:

```json title="mongolgpt.jsonc" {4-12}
{
  "command": {
    // Энэ нь командын нэр болно
    "test": {
      // LLM рүү илгээх prompt
      "template": "Бүх тестийг coverage тайлантай ажиллуулж, унасан тестүүдийг харуул.\nУнасан тестүүдэд төвлөрч засвар санал болго.",
      // TUI-д харагдах тайлбар
      "description": "Тестийг coverage-тай ажиллуулах",
      "agent": "build",
      "model": "anthropic/claude-3-5-sonnet-20241022"
    }
  }
}
```

Одоо та энэ тушаалыг TUI дээр ажиллуулж болно:

```bash frame="none"
/test
```

---

### Тэмдэглэгээ

Та мөн markdown файлуудыг ашиглан тушаалуудыг тодорхойлж болно. Тэдгээрийг байрлуул:

- Глобал: `~/.config/mongolgpt/commands/`
- Төсөл тус бүр: `.mongolgpt/commands/`

```markdown title="~/.config/mongolgpt/commands/test.md"
---
description: Тестийг coverage-тай ажиллуулах
agent: build
model: anthropic/claude-3-5-sonnet-20241022
---

Бүх тестийг coverage тайлантай ажиллуулж, унасан тестүүдийг харуул.
Унасан тестүүдэд төвлөрч засвар санал болго.
```

Маркdown файлын нэр нь тушаалын нэр болно. Жишээлбэл, `test.md` зөвшөөрнө
чи гүйж:

```bash frame="none"
/test
```

## Шуурхай тохиргоо

Захиалгат тушаалын сануулгууд нь хэд хэдэн тусгай орлуулагч болон синтаксийг дэмждэг.

---

### Аргументууд

`$ARGUMENTS` орлуулагчийг ашиглан командуудад аргумент дамжуулна уу.

```md title=".mongolgpt/commands/component.md"
---
description: Шинэ component үүсгэх
---

TypeScript дэмжлэгтэй `$ARGUMENTS` нэртэй React component үүсгэ.
Зөв type болон үндсэн бүтцийг оруул.
```

Аргументтай командыг ажиллуулна уу:

```bash frame="none"
/component Button
```

Мөн `$ARGUMENTS`-г `Button` гэж солино.

Та мөн байрлалын параметрүүдийг ашиглан бие даасан аргументуудад хандах боломжтой:

- `$1` - Эхний аргумент
- `$2` - Хоёр дахь аргумент
- `$3` - Гурав дахь аргумент
- Гэх мэт...

Жишээ нь:

```md title=".mongolgpt/commands/create-file.md"
---
description: Агуулгатай шинэ файл үүсгэх
---

`$2` хавтаст `$1` нэртэй файл үүсгэ.
Дараах агуулгыг бич: `$3`
```

Командыг ажиллуулна уу:

```bash frame="none"
/create-file config.json src "{ \"key\": \"value\" }"
```

Энэ нь орлоно:

- `$1`, `config.json`
- `$2`, `src`
- `$3`, `{ "key": "value" }`

---

### Бүрхүүлийн гаралт

_!`command`_ ашиглан [Bash command](../tui#bash-командууд)-ийн гаралтыг prompt-д оруулна.

Жишээлбэл, тестийн хамрах хүрээг шинжилдэг захиалгат командыг үүсгэхийн тулд:

```md title=".mongolgpt/commands/analyze-coverage.md"
---
description: Тестийн coverage шинжлэх
---

Одоогийн тестийн үр дүн:
!
pm test`

Эдгээр үр дүнд тулгуурлан coverage нэмэх сайжруулалт санал болго.
```

Эсвэл сүүлийн үеийн өөрчлөлтүүдийг шалгахын тулд:

```md title=".mongolgpt/commands/review-changes.md"
---
description: Сүүлийн өөрчлөлтийг шалгах
---

Сүүлийн Git commit-ууд:
!`git log --oneline -10`

Эдгээр өөрчлөлтийг шалгаж сайжруулалт санал болго.
```

Командууд нь таны төслийн үндсэн директорт ажилладаг бөгөөд тэдгээрийн гаралт нь сануулгын нэг хэсэг болно.

---

### Файлын лавлагаа

Файлуудыг `@`, дараа нь файлын нэрийг ашиглан тушаалдаа оруулна уу.

```md title=".mongolgpt/commands/review-component.md"
---
description: Component шалгах
---

`@src/components/Button.tsx` дахь component-ийг шалга.
Гүйцэтгэлийн асуудал байгаа эсэхийг шалгаж сайжруулалт санал болго.
```

Файлын агуулгыг сануулгад автоматаар оруулна.

## Сонголт

Тохиргооны сонголтуудыг нарийвчлан авч үзье.

---

### Загвар

`template` сонголт нь тушаалыг гүйцэтгэх үед LLM руу илгээгдэх хүсэлтийг тодорхойлдог.

```json title="mongolgpt.json"
{
  "command": {
    "test": {
      "template": "Бүх тестийг coverage тайлантай ажиллуулаад алдааг харуул.\nУнасан тестүүдэд төвлөрч, засах санал өг."
    }
  }
}
```

Энэ нь **шаардлагатай** тохиргооны сонголт юм.

---

### Тодорхойлолт

`description` сонголтыг ашиглан команд юу хийдэг талаар товч тайлбар өгнө үү.

```json title="mongolgpt.json"
{
  "command": {
    "test": {
      "description": "Run tests with coverage"
    }
  }
}
```

Энэ нь таныг тушаалыг бичих үед TUI-д тайлбар хэлбэрээр харагдана.

---

### Агент

`agent` тохиргоог ашиглан аль [agent](../agents) энэ командыг гүйцэтгэх ёстойг тохируулна уу.
Хэрэв сонгосон agent нь [дэд агент](../agents/#дэд-агентууд) бол command анхдагчаар subagent call үүсгэнэ.
Энэ үйлдлийг идэвхгүй болгохын тулд `subtask`-г `false` гэж тохируулна уу.

```json title="mongolgpt.json"
{
  "command": {
    "review": {
      "agent": "plan"
    }
  }
}
```

Энэ нь **заавал биш** тохиргооны сонголт юм. Хэрэв заагаагүй бол таны одоогийн төлөөлөгчийн өгөгдмөл.

---

### Дэд даалгавар

[Дэд агент](../agents/#дэд-агентууд) дуудахыг албадах бол `subtask` boolean ашиглана.
Хэрэв та команд нь таны үндсэн контекстийг бохирдуулахгүй байхыг хүсэж байгаа бөгөөд агентыг дэд агентын үүрэг гүйцэтгэхийг **хүчлэх** бол энэ нь ашигтай.
`mode` тохиргоонд `primary` гэж тохируулсан ч гэсэн.

```json title="mongolgpt.json"
{
  "command": {
    "analyze": {
      "subtask": true
    }
  }
}
```

Энэ нь **заавал биш** тохиргооны сонголт юм.

---

### Загвар

Энэ тушаалын өгөгдмөл загварыг хүчингүй болгохын тулд `model` тохиргоог ашиглана уу.

```json title="mongolgpt.json"
{
  "command": {
    "analyze": {
      "model": "anthropic/claude-3-5-sonnet-20241022"
    }
  }
}
```

Энэ нь **заавал биш** тохиргооны сонголт юм.

---

## Баригдсан

MongolGPT нь `/init`, `/undo`, `/redo`, `/share`, `/help` зэрэг built-in command-тай. [TUI командуудыг](../tui#командууд) харна уу.

:::note
Захиалгат тушаалууд нь суулгасан командуудыг дарж болно.
:::

Хэрэв та ижил нэртэй захиалгат тушаалыг тодорхойлсон бол энэ нь суулгасан тушаалыг хүчингүй болгоно.
